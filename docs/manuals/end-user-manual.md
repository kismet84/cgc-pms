# v1.5 终端用户手册（最小版）

## 1. 适用范围

本手册面向试点终端用户，优先覆盖以下当前已放行角色：
- 项目经理
- 商务经理
- 采购经理
- 生产经理

本手册适用于 `develop/1.5`。实际菜单、账号和数据范围以当前角色授权及交付环境验证为准。

## 2. 登录与进入系统

### 2.1 登录入口

- 前端入口通常为本地或交付环境的管理后台首页。
- 登录失败、账号发放方式、初始密码规则属于实施现场信息，当前需要确认；确认口径统一以 [试点账号与审批模板确认清单](../training/pilot-account-and-approval-checklist.md) 为准。

### 2.2 首次进入后的判断顺序

1. 先确认能否正常进入首页。
2. 再确认左侧菜单是否符合自己的角色范围。
3. 最后进入对应业务域完成本角色的最小主路径。

如果出现以下情况，优先找系统管理员处理：
- 页面提示 `401` 或登录失效：通常表示需要重新登录。
- 页面提示 `403` 或无按钮：通常表示权限不足。
- 登录成功但看不到目标菜单：通常是角色或菜单权限未开通。

## 3. 菜单定位

当前业务主入口以 [03 业务模块说明](../standards/03-业务模块说明.md) 为准：

| 业务域 | 常见菜单入口 |
|------|------|
| 工作台与预警 | `/dashboard`、`/alert` |
| 合同与签证 | `/contract`、`/variation` |
| 成本与目标 | `/cost`、`/cost-subject`、`/cost-target` |
| 采购与库存 | `/purchase`、`/receipt`、`/inventory`、`/requisition` |
| 分包与结算 | `/subcontract`、`/settlement` |
| 付款与发票 | `/payment`、`/invoice` |
| 审批中心 | `/approval` |

使用原则：
- 不要求终端用户记住全部路径，只需记住自己角色的主路径。
- 若菜单存在但无数据，先确认是否已切换到正确项目，再判断是否属于演示数据不足或业务数据为空。

## 4. 角色边界

### 4.1 当前建议培训主清单

- 项目经理：执行协同、待办推进、进度预警、合同履约。
- 商务经理：合同、变更、成本控制、结算、付款相关台账。
- 采购经理：采购执行、供应商、交货、验收、库存联动。
- 生产经理：材料验收、领料、库存、分包计量等现场执行近似数据。

### 4.2 当前不纳入正式试点培训主清单

- 总工程师：当前仍冻结，不纳入本轮正式培训。

### 4.3 需要确认

- 生产经理若需要展示完整施工进度、劳务、机械、产值能力，当前需要确认，不能按已上线能力培训；确认边界详见 [生产经理能力边界确认单](production-manager-capability-boundary.md)。

## 5. 核心主路径

以下只写最小主路径，不展开所有支线操作。

### 5.1 项目经理主路径

1. 登录系统。
2. 进入工作台或驾驶舱，查看待办、预警、临期事项。
3. 进入项目相关台账或审批中心，处理待审批事项。
4. 对照项目执行情况推进合同履约或协同事项。

### 5.2 商务经理主路径

1. 进入驾驶舱查看经营结果、成本偏差、待付款和预警。
2. 进入合同、签证、成本、付款相关模块查看台账。
3. 对需要审批的业务提交或跟踪审批状态。

### 5.3 采购经理主路径

1. 进入采购、验收、库存相关菜单。
2. 查看采购订单状态、未入库情况、供应商与库存预警。
3. 根据业务需要联动验收、库存或领料信息。

### 5.4 生产经理主路径

1. 进入验收、领料、库存、分包计量相关菜单。
2. 查看现场材料验收状态、领料出库、库存变化。
3. 对需要协同的问题回到审批或项目相关入口处理。

## 6. 常见问题

### 6.1 权限不足

现象：
- 能登录，但看不到菜单或按钮。
- 页面返回 `403`。

处理：
- 联系系统管理员核对角色、菜单权限、按钮权限。
- 前端隐藏按钮不等于授权，仍以后端权限结果为准。

### 6.2 审批未配置

现象：
- 单据可填写，但提交审批失败。
- 审批中心没有预期待办。

处理：
- 联系系统管理员检查审批模板、节点配置、审批人角色是否已开通。
- 审批入口当前固定在 `/approval/todo`、`/approval/done`、`/approval/cc`。

### 6.3 列表无数据

现象：
- 页面可进入，但表格为空。

优先排查顺序：
1. 是否切换到正确项目。
2. 是否属于当前账号可见的数据范围。
3. 是否当前演示环境尚无业务数据。
4. 若角色是生产经理且期待完整施工主数据，当前需要确认。
5. 账号、密码、审批模板等交付口径若现场说法不一致，以 [试点账号与审批模板确认清单](../training/pilot-account-and-approval-checklist.md) 为准。
