# v1.5 管理员手册（最小版）

## 1. 适用范围

本手册面向系统管理员、实施管理员，目标是支撑试点交付前的最小开通、权限配置、审批配置和交付前检查。

本手册适用于 `develop/1.5`。历史验收结论不能替代交付环境的账号、权限、审批、健康检查和备份恢复验证。

## 2. 用户角色

当前角色边界以 [03 业务模块说明](../standards/03-业务模块说明.md) 为准：

- 已放行：商务经理、项目经理、采购经理、生产经理。
- 已冻结：总工程师。

管理员交付动作：
1. 明确试点用户属于哪个角色。
2. 不给冻结角色安排正式演示或正式权限包。
3. 若业务方要求超出当前边界的角色能力，标记为“需要确认”，不要现场口头放行。

## 3. 菜单权限

权限模型以 [08 权限与审批流程](../standards/08-权限与审批流程.md) 为准，当前遵循：

```text
用户 -> 角色 -> 菜单权限 / 按钮权限 / API 权限 -> 项目数据范围
```

管理员至少需要核对：
- 用户是否绑定正确角色。
- 角色是否勾选对应菜单权限。
- 关键操作按钮是否具备按钮权限。
- 项目数据范围是否符合试点要求。

交付提醒：
- 前端看得见按钮不等于后端已授权。
- 导出、统计、驾驶舱同样受数据权限约束。

## 4. 审批配置入口

当前审批中心和配置口径以 [08 权限与审批流程](../standards/08-权限与审批流程.md) 为准。

管理员应确认：
- 审批中心入口可用：`/approval/todo`、`/approval/done`、`/approval/cc`。
- 业务单据提交审批后，实例状态与任务状态符合当前文档约束。
- 审批模板、节点、处理角色已与试点角色清单对齐。

最小检查点：
1. 业务类型是否存在对应审批模板。
2. 模板是否启用。
3. 审批节点是否存在实际处理角色或处理人。
4. 重复提交、驳回、撤回等常见流转是否已有明确处理口径。

需要确认：
- 试点租户是否已经固化审批模板命名规范与审批人映射方式；统一收口到 [试点账号与审批模板确认清单](../training/pilot-account-and-approval-checklist.md)。

## 5. 交付前检查

### 5.1 账号与角色

- 试点账号是否已创建。
- 每个账号是否绑定正确角色。
- 冻结角色是否误开通。
- 账号清单、初始密码发放方式、密码重置口径未确认前，不得作为已完成交付项；详见 [试点账号与审批模板确认清单](../training/pilot-account-and-approval-checklist.md)。

### 5.2 菜单权限

- 关键角色能看到自己的主菜单。
- 不属于本角色的高风险入口未误开放。
- 进入页面后无大面积 `403`。

### 5.3 审批配置

- 审批模板已启用。
- 审批节点角色已配置。
- 审批中心待办、已办、抄送入口可访问。

### 5.4 环境与运维跳转

- 如需部署、备份、恢复、健康检查、JAR 重打等动作，统一跳转 [10 部署运维手册](../standards/10-部署运维手册.md)。
- 本手册不复制部署细节，避免双维护。

## 6. 常见问题

### 6.1 用户看不到菜单

优先检查：
1. 用户是否绑定正确角色。
2. 角色是否勾选菜单权限。
3. 是否存在项目数据范围限制。

### 6.2 用户能进页面但操作失败

优先检查：
1. 是否缺少按钮权限或 API 权限。
2. 是否业务状态不允许提交/审批。
3. 是否审批模板未配置或审批人缺失。

### 6.3 交付现场问到旧手册步骤

处理原则：
- 旧手册只作为历史参考骨架。
- 若当前 `docs/` 没有证据支持，统一标记“需要确认”，不要把旧步骤直接当现状。
- 涉及生产经理完整施工进度、劳务、机械、产值能力时，统一跳转 [生产经理能力边界确认单](production-manager-capability-boundary.md)。
