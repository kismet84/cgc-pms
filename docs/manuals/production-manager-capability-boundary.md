# 生产经理能力边界确认单

## 1. 文档定位

本文件是生产经理培训与试点交付使用的“能力边界确认单”，不是能力已全部确认的结论文件。

使用边界：
- 只定义当前可按现有手册培训的范围、当前不能承诺的范围、仍需确认的范围。
- 不把完整施工进度、劳务、机械、产值类能力写成既成事实。
- 确认完成后，应按本文件指定位置回写手册；确认前不得在培训或交付现场口头放大承诺。
- 现场确认过程应同步记录到 `docs/training/pilot-confirmation-record.md`，最终是否允许回写以 `docs/training/pending-items-signoff.md` 为准。

## 2. 状态说明

| 状态 | 说明 |
|------|------|
| 已确认 | 当前已有手册和有效文档支撑，可直接用于培训 |
| 待确认 | 当前文档明确没有足够证据支撑，只能作为确认项存在 |
| 不适用 | 当前试点范围不要求培训或演示 |

## 5. 需要确认的能力项

| 能力项 | 当前状态 | 需要确认内容 | 确认责任人 | 证据方式 | 通过标准 | 不通过标准 |
|------|------|------|------|------|------|------|
| 完整施工进度 | 待确认 | 是否存在可稳定演示的菜单、流程、数据口径和角色权限 | 产品负责人 / 实施负责人 / 系统管理员 | 现网或试点环境截图、实操记录、角色权限核对记录 | 能明确入口、角色、最小流程和培训口径 | 只有历史认知或口头说法，没有当前证据 |
| 劳务 | 待确认 | 是否纳入本轮试点范围，是否有可培训的最小路径 | 产品负责人 / 实施负责人 | 范围确认记录、环境截图 | 能说明是否纳入试点及对应最小路径 | 范围不清，或无法证明当前可用 |
| 机械 | 待确认 | 是否纳入本轮试点范围，是否有稳定入口与数据 | 产品负责人 / 实施负责人 | 范围确认记录、环境截图 | 能说明入口、角色和最小操作口径 | 现场无法说明或与手册冲突 |
| 产值 | 待确认 | 是否有当前可交付的功能范围、权限与数据口径 | 产品负责人 / 实施负责人 / 系统管理员 | 功能演示记录、权限核对截图 | 能说明功能范围、角色权限、培训边界 | 只有概念性描述，不能落到现有环境 |

## 6. 确认后回写位置

| 确认结果类型 | 回写位置 |
|------|------|
| 生产经理新增可培训能力 | `docs/manuals/end-user-manual.md` 的“角色边界”“生产经理主路径” |
| 管理员侧角色开通或权限口径变化 | `docs/manuals/admin-manual.md` 的“用户角色”“菜单权限”“交付前检查” |
| 培训组织或现场答疑口径变化 | `docs/training/README.md` 或角色化培训脚本 |

## 7. 培训时禁止承诺的话术

以下话术当前禁止使用：
- “生产经理这块完整施工进度已经全部能用了。”
- “劳务、机械、产值模块都已经按正式上线范围交付。”
- “这些能力手册没写，但现场肯定都有，只是还没整理。”
- “先按旧手册讲，后面系统再补齐就行。”

允许使用的最小口径：
- “当前已确认可培训范围以现有手册为准，超出部分需要确认后再回写。”
- “完整施工进度、劳务、机械、产值目前属于待确认项，本轮不作为已确认能力承诺。”

## 8. 使用限制

- 本确认单未变更为“已确认”的能力项，不得写回正式培训结论。
- 不得把 `.archive` 或历史认知中的生产经理能力直接转写为当前事实。
- 如现场需要扩大生产经理培训范围，必须先补证据、补确认，再更新手册。
