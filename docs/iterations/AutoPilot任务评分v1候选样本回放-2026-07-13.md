# AutoPilot 任务评分 v1 候选样本回放

## 结论

`autopilot-task-score/v1` 的公式可确定性计算，但现有三份历史正式验收报告不足以生成可审计的正式评分。该历史结论继续禁止回算旧任务；用户已于 2026-07-13 明确批准 v1 权重，新版本只从批准配置提交后启动的下一项新实施型 Ready 起生效。

## 样本与方法

样本来自三份已通过的正式报告：

- `docs/quality/ISSUE-040-019-系统菜单新建管理员入口验收报告.md`
- `docs/quality/ISSUE-040-020-系统菜单删除管理员入口验收报告.md`
- `docs/quality/ISSUE-040-021-系统菜单详情管理员入口验收报告.md`

本次只回放正式字段可用性，不用模型主观补分，也不把报告中的叙述性推断写成正式结构化事实。

| 维度/前置 | 可稳定计算样本 | 结果 |
| --- | ---: | --- |
| 交付正确性 | 3/3 | 均有验收标准、专项验证、范围核对和通过结论 |
| 问题零悬空 | 3/3 | 均有新增、关闭、净变化及剩余风险处置 |
| 首次验收通过率 | 0/3 | 有首次失败与补修叙述，但缺少统一的结构化 `attempt`；工具/环境复跑与正式验收补修不能稳定机器区分 |
| 周期效率 | 0/3 | 缺少结构化 `cycleSeconds`、`firstPassSuccess` 和 `avoidableReworkCount` |
| 存量问题下降速度 | 3/3 | 均明确关闭一个 A-01 存量问题且净变化为 -1 |
| 两阶段提交绑定 | 0/3 | 历史报告没有分别记录 `implementationCommit` 与 `closeoutCommit`，不能生成正式评分幂等键 |

## 分歧样本

三份报告都包含工具配置或环境前置复跑，也都包含 Reviewer 首次阻塞后的本轮修正。若只从自然语言统计“首次失败”，会把命令前置、环境恢复和正式验收补修混为一类；因此 v1 不允许从历史文本推断 `attempt` 或返工次数。

## 调整与批准建议

1. 保持当前建议权重不变，先由新 runner 结构化记录 `attempt`、`firstPassSuccess`、`cycleSeconds`、`avoidableReworkCount` 和两个提交 SHA。
2. 至少积累一组由新结构化链路产生的正式样本后，再审阅权重是否区分度合理。
3. 本报告不是权重批准证据；正式批准证据见 `docs/iterations/AutoPilot任务评分v1批准决策-2026-07-13.md`。
