### 2.1 颜色语义标准

| 角色           | Token                                                                  | 使用边界                                                                     |
| -------------- | ---------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| 页面与容器     | `canvas`、`surface`、`surface-subtle`、`surface-hover`                 | 页面背景、卡片、弱分区和悬停；不表达业务状态                                 |
| 文字           | `text-strong`、`text`、`text-secondary`、`text-muted`、`text-disabled` | 按信息层级使用；禁用态不得只靠降低透明度                                     |
| 品牌与交互     | `primary`、`primary-hover`、`primary-active`、`primary-soft`           | 主操作、链接、焦点和常规导航激活态                                           |
| 信息           | `info`、`info-text`、`info-soft`                                       | 普通说明、建设/迁移状态，不表示风险                                          |
| 成功           | `success`、`success-text`、`success-soft`                              | 已完成、已通过、运行正常                                                     |
| 预警           | `warning`、`warning-text`、`warning-soft`                              | 需要关注但尚未失败的业务风险                                                 |
| 危险           | `danger`、`danger-text`、`danger-hover`、`danger-soft`                 | 错误、失败、高风险和破坏性操作                                               |
| 工作区标签例外 | `workspace-tab-accent`、`workspace-tab-accent-soft`                    | 仅用于用户确认的斜切标签激活线与激活态；不得代替危险色，也不得扩散到其他导航 |

同一信息只使用一种状态语义；颜色必须配合文字、图标或状态标签，不能作为唯一识别手段。业务组件禁止直接引用十六进制颜色，也不得借用 `danger` 表示普通选中态。

### 2.2 文字层级标准

| 层级       | 字号 / 字重 / 行高                                   | 典型用途                         |
| ---------- | ---------------------------------------------------- | -------------------------------- |
| 页面标题   | `--v2-font-size-21` / `bold` / `tight`               | 页面唯一语义 `h1`                |
| 区块标题   | `--v2-font-size-15` / `semibold` / `tight`           | 卡片组、主要区块标题             |
| 详情重点   | `--v2-font-size-14` / `semibold` / `ui`              | 对象名称、关键事实               |
| 正文与控件 | `--v2-font-size-13` / `regular` 或 `medium` / `body` | 说明、长文本、表单控件正文       |
| 事实文本   | `--v2-font-size-12` / `regular` / `ui`               | 表格、详情事实、分页、下拉项     |
| 辅助文字   | `--v2-font-size-11` 或 `12` / `regular` / `ui`       | 时间、提示、元数据；按密度二选一 |

页面最多同时呈现三级标题；同一层级不得靠临时颜色或任意加粗制造新层级。正文默认使用 `text` 或 `text-secondary`，辅助文字使用 `text-muted`，禁用内容使用 `text-disabled`。业务数字仅在需要等宽对齐时启用 tabular nums。

## 3. 本切片组件

- 操作：`V2Button`，覆盖 primary/secondary/ghost/danger、small/medium/touch、loading、disabled 和 focus-visible。
- 表单：`V2Input`、`V2Select`，覆盖标签、必填、提示、错误、禁用、加载和 `aria-describedby`。
- 容器：`V2Card`，覆盖标题、说明、操作、正文、页脚和可交互状态。
- 状态：`V2Badge`、`V2Alert`、`V2Skeleton`，覆盖 neutral/info/success/warning/danger、可关闭反馈与加载语义。
- 浮层：`V2Dialog`，覆盖焦点进入/恢复、Escape、背景关闭、Tab 循环、标题/说明关联和移动底部形态。
- 布局：`V2Stack`、`V2Cluster`、`V2Grid`，只暴露 token 化间距、对齐和最小列宽。

非目标：认证、权限、导航、真实驾驶舱、业务表格/图表、Toast 队列、复杂表单适配、第三方 UI 库和正式入口切换。

### 7.2 字号、颜色与密度

| 场景                         | 标准                                             |
| ---------------------------- | ------------------------------------------------ |
| 页面标题                     | `--v2-font-size-21`、bold；每页只有一个语义 `h1` |
| 卡片或区块标题               | `--v2-font-size-15`、semibold                    |
| 详情重点信息                 | `--v2-font-size-14`、semibold                    |
| 表格、详情事实、分页、下拉项 | `--v2-font-size-12`                              |
| 表单控件正文                 | `--v2-font-size-13`                              |
| 辅助说明、元数据             | `--v2-font-size-11` 或 `--v2-font-size-12`       |

- 颜色只能使用 `--v2-color-*` 语义 token；状态色必须同时配文字或图标，不得只靠颜色表达。
- 页面标题通过 `V2Card.headingLevel/titleId` 或原生语义标题实现；不可用普通 `div` 模拟标题。
- 文本、搜索和下拉字段统一使用 `V2Input`、`V2Select`；日期、数字、文件、复选、单选和多行文本可保留原生语义控件。详情/编辑弹窗内的输入框、文本域和下拉框使用透明背景，轮廓使用 `primary 22% + surface` 的浅蓝混色；focus、error、disabled 继续使用公共语义状态。

### 7.3 标准弹窗

- 确认操作统一使用 `V2ConfirmDialog`，禁止 `window.confirm`；破坏性确认使用 danger 动作，loading 时禁止重复提交和关闭。
- 新建、编辑和上下文追溯统一使用 `V2Dialog`。面板使用 `v2-dialog-standard`；只读追溯再叠加 `v2-detail-dialog`。需要深链、刷新恢复、前进后退或承载完整对象工作流的详情默认使用独立详情路由；审批工作台完整详情与快速预览均复用公共 `V2Dialog`，以 `v2-detail-dialog` 毛玻璃悬浮窗覆盖列表。
- 标准宽度为 `32rem`；视口不小于 `64rem` 时为 `46rem`；移动端保持底部弹层形态。
- 弹窗内部允许滚轮、触控和键盘滚动，但隐藏可视滚动条。下拉菜单向控件下方锚定，可独立滚动，不得撑高或撑宽弹窗。
- 弹窗正文统一使用 `--v2-font-size-12` 和正文行高，长编号、长名称及说明必须自动换行；业务页不得依赖页面根字号继承或用单行裁切隐藏事实。
- 只读详情允许背景关闭；新建、编辑背景点击不得关闭，必须保留内容并提示用户保存或主动关闭。
- 标题区、内容区、操作区使用 `--v2-dialog-divider` 分隔；禁止页面重新定义另一套弹窗壳。

### 7.4 详情区与操作区

- 详情结构统一使用 `v2-detail-dialog__section`、`v2-detail-dialog__facts`、`v2-detail-dialog__actions` 和 `v2-detail-dialog__form-row`。
- 详情内嵌 `V2Card` 取消独立底色、圆角、阴影和四周边框，只保留分区所需的浅蓝分割线。
- 详情区与操作区之间的分割线上下各留 `10px`；不得用空卡片或背景色代替分区。
- 事实信息桌面端两列，窄视口单列。快速操作桌面端保持一行弹性布局，空间不足时换行，`400px` 以下按钮占满行宽。
- 详情表格正文、事实值、分页和卡片详情文字统一为 `12px` token；业务页不得单独放大。

### 7.5 按钮

- 页面标题区、筛选区、表格行、分页和正文中的主操作、次操作、幽灵操作及危险操作使用 `V2Button` 对应 variant，不使用玻璃材质。
- 表格行内“更多”操作使用 `V2ActionMenu`，由公共组件统一处理菜单材质、外部点击关闭、Escape 关闭和焦点返回；页面不得使用原生 `details` 自建菜单壳或把 `.v2-button` 类复制到原生元素。
- `V2Dialog`、`v2-detail-dialog` 内的上下文操作和表单页脚动作可使用 `V2GlassButton`，接口固定为 `text`、`loading`、`disabled`、`onClick`、`className`。
- `V2ConfirmDialog` 继续使用语义明确的普通或危险按钮，不为视觉统一改成玻璃材质。
- 玻璃按钮使用 50% 透明白灰渐变、白色高透明描边、顶部弧形高光、内部高光、底部环境反光和柔和内外阴影；不得使用蓝色实心填充。
- 同时使用 `backdrop-filter: blur(16px) saturate(160%)` 与 WebKit 写法。hover 轻微上浮，active 下压并缩小，focus-visible 显示蓝色柔和焦点环；disabled 必须有文字、背景和交互三重区分。
- reduced-motion 下取消位移和过渡，不得牺牲键盘焦点或可读性。

### 7.6 分页、下拉与反馈

- 分页右对齐，固定顺序为“上一页 — 第 N 页 — 下一页”；页码位于两个按钮中间并使用 `--v2-font-size-12`。
- 分页列表固定请求 `pageSize=10`；表格数据行保持单行，超出宽度由表格容器受控横向滚动，不以卡片换行制造双行记录。
- 施工内容、备注、问题描述等长叙述字段使用 `v2-table-cell--wrap`，允许断词换行；编号、日期、金额、状态等结构化事实仍保持单行。
- 顶层列表的数据表格与分页属于同一复合数据区，统一置于一个 `V2Card`；筛选可独立成卡片，详情弹窗和驾驶舱可视化表格不适用本条。每条记录仍保持语义表格行，禁止改成记录卡片。
- 同一主对象的计划、执行、检查、整改、审批、结算、质保和归档等阶段数据，使用一张主表或同一 `V2Card` 内的具名语义表格区；禁止用并列卡片模拟多册账簿。相互独立的业务链最多各占一个顶层卡片。
- “全部项目”等聚合视图只展示带项目身份的主表；跨项目子记录不得在概览下失去项目归属后直接铺开，完整链路通过项目行或公共详情对话框进入。
- `V2Select` 选择后立即提交页面筛选状态；空值、禁用、键盘焦点和 Escape 行为由公共组件处理，页面不得另造下拉层。
- 搜索输入和筛选下拉统一隐藏视觉标签，字段语义通过 `label` 保留给辅助技术，并使用 `placeholder` 在控件内展示搜索范围或“全部状态”等当前筛选口径；创建、编辑和审批表单继续显示字段标签。
- 公共壳项目与报告期下拉相邻成组；桌面端项目框宽于报告期，中屏和移动端等分可用宽度，不得因隐藏标签留下空列或内容宽度控件。
- 短暂成功、信息、警告和错误结果复用公共 Toast，从顶部滑入并自动关闭；校验错误、未完成请求及需持续阅读的业务警告使用 `V2Alert`，保持可读并允许手动关闭。页面不得用占据文档流的成功横幅替代 Toast。

### 7.7 自动门禁

- `frontend-admin-v2/tests/unit/design-system.test.ts` 校验 token 引用，禁止裸颜色、字号、间距、控件高度和圆角；页面操作只允许复用公共按钮和菜单，标题层级、标准弹窗、详情分区、分页顺序、表单轮廓和编辑防误关均纳入门禁。
- 公共表格的宽度、字号、行高、间距、边框、对齐、颜色和表头材质只允许在 `styles/components.css` 定义；页面仅保留最小宽度、列宽、换行及具体字段语义等结构差异。
- 表格结构门禁动态扫描全部 `src/pages/**/*.vue`；包含业务编码的单元格只能承载该编码，不得再拼接名称、版本、日期、状态或其他字段，禁止页面白名单。
- `V2Dialog` 公共组件强制附加 `v2-dialog-standard`；页面只追加业务修饰类，不得重建弹窗壳。弹窗门禁动态扫描全部 `src/pages/**/*.vue`，禁止维护页面白名单；每个弹窗必须显式声明背景关闭策略，包含编辑控件时必须禁用背景关闭或提供明确拦截。
- 门禁校验公共壳固定视口与主内容纵向滚动、弹窗正文 `12px` 与长文本换行；动态扫描全部 V2 页面，短暂成功和操作结果只能调用公共 Toast，不得回退为成功横幅或页面私有浮层。
- 全页面门禁禁止原生文本/搜索/下拉控件及“权威、回读、重读、后端状态、后端阶段”等实现术语；原生语义控件只允许日期、数字、文件、复选、单选和隐藏字段。
- 新迁移页面至少通过 `test:unit`、`type-check`、`lint:check`、`build`、`test:e2e:migration-gate` 和 `git diff --check`；确定性迁移浏览器用例自动进入该脚本，依赖真实身份或真实数据的专项用例不得替代确定性门禁。
- 浏览器验收覆盖 `1440 / 1024 / 390`：检查弹窗宽度、内部滚动、下拉不溢出、焦点可见、按钮状态和分页顺序。

### 7.8 所有 V2 路由迁移退出门

本节是每条 Legacy → V2 路由的统一验收清单。阶段计划只能增加业务专项验收，不能删减或放宽本节。任一必检项失败，该路由保持“迁移中”，不得仅凭接口可用、构建通过或主流程可点击判定迁移完成。

#### 公共壳与项目范围

- 页面复用应用壳的项目、报告期、账户和导航上下文；不得在页面正文复制项目选择器、报告期选择器、用户区或主导航。
- 公共壳固定在当前视口，页头和导航保持稳定，`#shell-main-content` 是业务页唯一纵向滚动容器；禁止业务页使用固定高度或 `overflow: hidden` 截断后续区块。
- 项目与报告期是全局上下文，不设置页面能力白名单。所有当前和后续 `AppShell` 页面只要存在对应选项就必须允许选择，并通过 `projectId`、`period=YYYY-MM` 写入 URL、跨导航保留；禁止用路由元数据隐藏或禁用。
- 有日期或统计周期维度的列表、看板和查询必须将报告期转换为服务端查询边界并刷新数据；项目主数据、对象详情等无时间维度页面不据此裁剪完整对象，但必须保留报告期上下文供后续页面使用。
- 支持聚合查询的列表页默认选择“全部项目”，加载服务端授权范围内全部有业务数据的对象；不得把“全部项目”解释为缺少项目上下文或返回空页。
- “全部项目”下隐藏该业务域没有数据的项目，不生成空项目卡片；选择具体项目时只展示该项目数据。两种状态均不得绕过后端对象范围和权限校验。
- 公共壳切换项目后，页面数据、URL 查询参数和可执行动作必须一致更新；详情、编辑或写操作需要确定项目对象时，从当前对象或路由取得，不得移除公共壳的“全部项目”选项。

#### 页面结构与导航

- 每页只有一个语义 `h1`。页面标题区不得重复 eyebrow、大标题、流程说明或实现说明；用户未要求保留的介绍文案直接删除。
- 三级工作区 Tab 使用 `12px / semibold / UI line-height`，文案与第50条主线导航基线一致；当前商务合约代表项固定为“合同台账 / 签证变更 / 投标成本”，页面 `h1` 不得使用倒序或近义别名。
- 标题区无正文时不得渲染分割线、空卡片 body 或额外留白；页面级操作与标题同排并右对齐。
- 完整对象“详情”默认进入独立详情路由，支持深链、刷新恢复、浏览器前进后退和返回列表。审批工作台是受控例外：完整详情与快速预览均使用标准只读 `V2Dialog`，以 `v2-detail-dialog` 毛玻璃悬浮窗覆盖列表；允许同步详情路由保留深链，点击遮罩、按 Escape 或点击关闭按钮后返回进入前审批页签。两类交互都禁止在列表下方以内嵌展开。
- 列表、详情、新建和编辑必须保持现有 URL、route name、权限和对象范围语义；迁移不得顺手改后端状态机、金额口径或业务事实。

#### 信息层级与密度

- 页面标题使用 `21px / bold`，卡片或区块标题使用 `15px / semibold`，详情重点信息使用 `14px / semibold`，表格与事实文本使用 `12px`，表单控件正文使用 `13px`，辅助文字使用 `11px` 或 `12px`；全部通过 V2 token 实现。
- 验收读取浏览器 computed style；仅在源码根容器声明字号、但运行态仍继承旧样式或被公共组件覆盖，视为失败。
- 禁止业务页面新增裸字号、颜色、阴影、圆角或复制公共组件样式；同层级文字不得临时放大、任意加粗或用颜色制造新层级。

#### 弹窗与表单

- 新建、编辑、上下文追溯和确认分别复用 `V2Dialog`、`v2-dialog-standard`、`v2-detail-dialog`、`V2ConfirmDialog`；完整对象详情按 7.8 的导航规则处理，审批工作台完整详情与快速预览均使用公共毛玻璃对话框。不得重建页面私有弹窗壳或调用原生确认框。
- 标题、正文、字段标签、控件、说明和页脚按钮遵守 7.2～7.6；页脚按钮保持次操作在左、主操作在右，同尺寸、同基线且不贴边。
- 表单项目字段需要展示时独占一行并复用受权限约束的数据源；编号由服务端生成时不提供可编辑输入，前端生成默认值时必须有现行业务规则和接口契约依据。
- 编辑态背景点击不得关闭；校验失败、请求失败和附件上传失败必须保留输入并可恢复，loading 时禁止重复提交。

#### 数据、状态与交互

- 至少验收加载、典型数据、全部项目、具体项目、空数据、部分数据、接口失败、无权限和超长内容；不得用模拟数据掩盖服务端空态或错误。
- 所有可见状态使用统一中文映射和语义 badge；颜色不是唯一状态证据，不暴露英文枚举、后端字段名或“回读/权威”等实现术语。
- 主要按钮、筛选、分页、Tab、详情跳转、返回、刷新和弹窗提交各至少完成一次真实交互；结果必须由 URL、DOM、网络结果或服务端回读事实证明。

#### 响应式、无障碍与运行态

- `1440 / 1024 / 390` 均无横向溢出、遮挡、裁切、滚动陷阱或不可达操作；移动端可点目标、焦点、弹窗和单列收敛继续满足第 4 节。
- 页面具有正确 title、主内容地标、标题层级、表单关联、可见焦点和键盘路径；无框架错误遮罩，控制台无相关 warning/error。
- CSS、路由、代理或运行配置变更后必须刷新对应 V2 运行态，再以真实 URL 验收；源码、构建产物和浏览器 computed style 不一致时归类为 `runtime_stale`，不得误判为业务已修复。

#### 必需证据与裁决

- 静态证据：路由台账、权限/项目范围、公共组件复用和本页专项单测。
- 命令证据：`pnpm test:unit`、`pnpm type-check`、`pnpm lint:check`、`pnpm build`、`git diff --check`；涉及契约、边界、路由台账或包体时追加对应现行脚本。
- 浏览器证据：真实 URL、最终路由、桌面与移动截图、computed style、控制台、至少一个详情深链和一个主要交互结果；浏览器插件不可用时使用仓库 Playwright 并记录回退原因。
- 迁移裁决只允许“通过”“不通过”“需要确认”。发现项必须本轮修复、由唯一治理载体正式承接或有依据关闭；存在悬空项时不得通过。
