# 第52条主线 Phase 4.1：全页面演示数据覆盖扩展任务计划书

状态：已完成并验收通过（2026-07-18）。第52条主线既有 B215、bootstrap 和 `complete-project-v1` 保持冻结；本扩展不新建主线编号。

**Goal:** 将现有 `complete-project-v1` 从“核心业务闭环样本”扩展为可供第53条全量 UI Clean-room V2 逐页设计、开发和验收的 `complete-project-v2`，覆盖当前可达表单的业务字段、关联对象和代表性状态；先通过数据覆盖门，再继续 UI 重构。

**Architecture:** 采用“V1 不变 + V2 独立包 + 页面字段覆盖矩阵 + 分域 SQL 阶段 + 正常/边界/业务异常三态 + 独立 `cgc_pms_demo_v2` 数据库”结构。V2 复用 V1 已验收闭环，但使用独立 checkpoint、manifest、验证脚本和数据库；不修改 Flyway、后端 API、权限码、金额口径或 Legacy/V2 UI。异常态只使用合法的拒绝、取消、逾期、部分完成等业务状态，不向数据库写入违反约束的脏数据。

## 1. 执行顺序

1. 冻结 V1 指标、B215、当前路由与表单基线。
2. 生成“页面 → 表单 → 字段 → API → 数据表 → 演示记录 → 状态”覆盖矩阵。
3. 按项目、商务、供应链、资金、现场、技术、系统域扩展 V2 数据。
4. 每个可写表单至少具备正常态；列表/详情同时具备边界态和业务异常态。
5. 验证必填字段、可选展示字段、枚举、外键、金额数量、日期和幂等。
6. 在独立 demo 实例完成真实页面抽检，冻结包版本后交给第53条。

## 2. 覆盖口径

- 权威范围：`frontend-admin/src/router` 当前可达路由及其直接/局部表单组件。
- 字段范围：静态 `a-form-item`、绑定字段、动态行项目、附件、关联选择器和提交载荷字段。
- “有数据”不是所有可空列强制非空；必须为每个字段标明 `VALUE`、`INTENTIONAL_EMPTY`、`COMPUTED`、`REFERENCE` 或 `NOT_PERSISTED`。
- 必填字段必须存在至少一条可回读正常样本。
- 可选字段至少有一条非空展示样本；只有产品语义要求空值时才允许 `INTENTIONAL_EMPTY`。
- 错误态通过合法业务状态、无权限角色、空集合或受控接口失败验证；禁止持久化非法外键、越权租户、失衡金额或负库存。

## 3. 数据状态标准

| 状态 | 用途 | 数据要求 |
| --- | --- | --- |
| 正常态 | 主流程设计与回读 | 字段完整、关联有效、状态可解释、金额数量守恒 |
| 边界态 | 空值、长文本、零值、部分完成 | 仍满足数据库和业务约束，明确边界原因 |
| 业务异常态 | 驳回、取消、逾期、预警、核销异常 | 使用正式状态机，不伪造损坏数据 |

## 4. 实施边界

- 只修改 `scripts/demo/complete-project-v2/**`、本计划、覆盖矩阵和正式验收报告。
- 不修改 `complete-project-v1`、B215、历史迁移、后端业务代码、Legacy UI 或 V2 UI。
- 不连接生产；仅允许本地 `dev/test/demo`、`127.0.0.1` MySQL 和 reset marker。
- 不清理或重置 `cgc_pms`、`cgc_pms_demo`；V2 使用新建的 `cgc_pms_demo_v2`。
- 不创建明文密码或额外仓库密钥；沿用 bootstrap 管理员作为加载身份。

## 5. 阶段与退出门

### P0：基线与矩阵

- 输出当前页面、表单、字段、API 和目标表清单。
- 所有字段必须有唯一处置，不允许口头“以后补”。

退出门：必填字段映射率 100%；所有未自动映射项均有人工作出的正式分类。

### P1：分域数据扩展

- 保留 V1 四阶段。
- 新增财务与发票、生产计量、现金预测与结账、现场与技术、供应商寻源、变更与竣工明细、组织与系统管理阶段。

退出门：目标表均有演示记录；所有记录有稳定业务键、项目/租户归属和状态说明。

### P2：三态与一致性

- 正常、边界和业务异常样本可区分。
- 外键无孤儿，金额/数量守恒，日期有序，枚举合法，重复加载不增量漂移。

退出门：覆盖验证、业务指标、幂等和安全门全部通过。

### P3：隔离运行态验收

- 独立 backend/demo 入口，不切换现有 8080/5173/5174。
- 抽检八域代表页面、表单回读、空态/异常态和控制台。

退出门：真实 URL 可达；无未解释错误；包版本、数据库版本和验证结果冻结。

4. 外键、空值、枚举、金额数量和日期审计摘要。
