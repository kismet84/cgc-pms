# 第51条主线：成本科目 V2 重整任务计划书

状态：2026-07-18 已完成 Phase 0–4 及 V214 旧科目清理的本地实施与自动化验证；历史业务金额和单据事实不变，旧科目引用已迁至 V2 科目并保留删除审计。生产发布仍受目标环境只读扫描、映射签认和财务口径签认门禁约束。

**Goal:** 将 CGC-PMS 的成本科目整理为项目成本归集、目标成本、预算控制、付款结算、成本预测和经营预警共用的唯一口径；投标费用在中标后进入项目目标成本，质量安全成本独立分析，财务费用允许按受控规则分摊至项目。

**Architecture:** 保持已发生业务事实和金额可追溯；通过“科目体系 + 归集规则 + 项目适用范围 + 映射版本”切换新单据。用户后续明确授权删除旧成本科目，V214 因此先将全部已知引用迁至 V2 科目并写入不可变替代审计，再物理删除 10 个旧节点。成本、收入、应收、结算按领域展示，禁止以模糊 `subject_type` 或任意启用科目兜底归集；不复制成本台账、预算、目标成本或会计凭证事实。

## 1. 范围与非目标

### 1.1 本主线范围

- 形成成本科目 V2 口径、层级、映射、迁移和验收基线。
- 改造前核验 `cost_subject` 与成本、预算、付款、费用、结算、会计、预测的全部引用。
- 补齐显式归集规则、项目适用范围、投标成本转入和项目财务费用分摊。
- 将“成本科目中心”组织为科目体系、归集规则、项目适用与目标成本、影响与转入追踪四个工作区。

### 1.2 非目标

- 不重做企业会计科目表；会计科目仅作为映射维度。
- 不删除任何历史成本、预算、付款、结算或凭证事实。
- 不改变收入、应收、结算的业务数据，只调整其在主数据页面的归属和展示。
- 不删除或改写历史成本、预算、付款、结算、凭证及其他业务事实；旧路由继续兼容。旧科目主数据及其外键值按用户后续明确授权由 V214 受控迁移，不属于本非目标。

## 2. 当前事实与问题

### 2.1 当前数据基线

| -------------- | ---: | ---------------------------------------------- |
| 未逻辑删除记录 |  114 | 包含启用与停用记录                             |
| 启用记录       |  111 | 可被当前树和解析逻辑选择                       |
| `COST`         |   87 | 同时存在标准树、旧简化树和重复旧节点           |
| `REVENUE`      |   19 | 包含 2 条已停用历史收入树                      |
| `RECEIVABLE`   |    2 | 投标保证金、履约保证金，当前挂在投标成本分支下 |
| `SETTLEMENT`   |    5 | 合同结算、合同资产、合同负债                   |
| `ROOT`         |    1 | 已停用历史根节点                               |

该快照仅用于设计和迁移预估，不可推断生产库引用量。

### 2.2 当前结构问题

- `cost_subject` 同时承载成本、收入、应收和结算；前端成本科目页当前不传 `category`，一次加载全部树。
- `subject_type` 同时存在 `MATERIAL`、`LABOR` 等枚举和“材料”“人工”等旧中文值。
- `COST_MATERIAL` 与 `5001.03` 同名为“材料成本”；两者及 `5401.03.02` 表达相近但不应直接合并历史事实。
- 标准成本树同时按投标、采购、施工阶段组织；同一经济性质的材料、检验、管理费用因此分散。
- `CostSubjectResolver` 在类型匹配失败时会回退到根科目，再回退到任一启用科目；该策略可能制造错误归集。
- `CostControlService` 要求完工预测覆盖目标成本和已发生成本科目，并拒绝未归类的非零成本。这证明成本科目已是动态利润控制的关键维度。
- 当前删除保护仅显式检查 `cost_item` 与 `cost_target_item`；但 `project_budget_line`、`pay_application`、`expense_application`、`cost_forecast_item`、`accounting_entry_line` 等也引用成本科目。

## 3. 已确认业务决策

| 决策         | V2 规则                                                                                              |
| ------------ | ---------------------------------------------------------------------------------------------------- |
| 投标费用     | 中标后可转入该中标项目的目标成本；投标阶段事实与项目转入事实必须可双向追溯，且同一金额不得重复计入。 |
| 质量安全成本 | 单设“质量安全成本”一级归集组，支持检验检测、安全文明、劳动保护、质量保修等分析。                     |
| 财务费用     | 允许项目分摊，但仅可通过受控、可审批、可追溯、不可重复的分摊规则进入项目成本。                       |

## 4. V2 成本科目逻辑树

下列为逻辑展示和归集层级，不代表本阶段立即修改既有 `subject_code`。

```text
合同履约成本
├─ 投标及前期成本
│  ├─ 投标费用
│  ├─ 标书与代理
│  ├─ 勘察、咨询、差旅、资格预审
│  └─ 其他前期成本
├─ 直接成本
│  ├─ 材料及设备
│  ├─ 人工
│  ├─ 机械
│  ├─ 专业分包
│  └─ 施工水电及其他直接成本
├─ 措施成本
├─ 质量安全成本
│  ├─ 检验检测
│  ├─ 安全文明
│  ├─ 劳动保护
│  └─ 质量保修
├─ 项目间接费
├─ 项目财务费用
│  ├─ 借款利息
│  ├─ 保函及担保费
│  ├─ 融资服务费
│  └─ 其他项目融资费用
└─ 风险、损失与调整
```

规则：只有末级科目可进入目标成本、预算、付款、费用、结算、成本台账和预测；父级仅用于汇总。收入、应收和结算不再出现在成本中心默认树中。

## 6. 归集、转入与分摊规则

### 6.1 归集规则

- 以“业务来源类型 + 业务分类 + 项目适用范围”显式确定候选末级科目。
- 材料收货/领料、采购订单、分包计量、费用申请、付款申请、合同变更、投标成本、质量安全后果和财务费用分摊分别配置规则。
- 映射失败返回“待归类”，阻断提交或过账；不得退回根科目或任意启用科目。
- 用户手工覆盖默认科目时，必须拥有对应权限并填写原因；审批后冻结。

### 6.2 投标成本转入

- 仅中标项目可发起；投标项目、目标成本版本、转入明细、来源成本事实和审批实例必须绑定。
- 同一投标成本事实在同一目标成本版本中只能转入一次；撤销必须生成反向事实，不得静默修改原始金额。
- 转入后由目标成本版本承担责任成本，投标台账仍保留原始成本与转入状态。

### 6.3 质量安全成本

- 检验检测、安全文明、劳动保护和质量保修使用独立的分析父级；实际归集仍进入末级科目。
- 质量安全整改的处罚、返工、保修等事实应可追溯到检查/问题/整改或合同来源，不用页面临时金额替代。

### 6.4 项目财务费用分摊

- 规则必须包含：费用来源、项目、期间、金额、分摊依据、审批实例、操作人和幂等键。
- 同一来源费用的已分摊金额不得超过可分摊金额；冲销使用反向事实。
- 默认不自动分摊。允许的分摊依据在实施前由财务确认，例如合同额、融资受益金额、占用天数或明确项目归属。

### 6.5 财务生产启用确认单

系统已将以下四种依据实现为人工发起、逐批审批的受控枚举；不提供默认自动分摊。生产启用前必须由财务对可用项、会计映射和例外申请口径签认。

| 候选依据     | 适用条件                               | V2 处理要求                                       | 当前裁决                                                     |
| ------------ | -------------------------------------- | ------------------------------------------------- | ------------------------------------------------------------ |
| 明确项目归属 | 融资、保函或担保业务本身已绑定唯一项目 | 100% 直接归集；来源会计事实、项目、期间和审批绑定 | 已实现 `DIRECT_PROJECT`；生产启用待签认                      |
| 融资受益金额 | 单笔融资可获得各项目实际受益金额       | 按受益金额比例分摊；总额不得超过来源金额          | 已实现 `BENEFIT_AMOUNT`；生产启用待签认                      |
| 资金占用天数 | 有可核验的项目资金占用余额与期间       | 按余额 × 天数比例分摊；期间封闭后仅允许冲销       | 已实现 `OCCUPIED_DAYS`；生产启用待签认                       |
| 合同额       | 无法取得更强归因证据且财务批准例外使用 | 仅人工发起、逐批审批；不得设为默认自动规则        | 已实现 `CONTRACT_AMOUNT_EXCEPTION`，强制备注；生产启用待签认 |

无论选择何种依据，必须新增独立的财务费用来源、分摊批次、明细、审批和反向事实；不得改造或复用 `OVERHEAD_ALLOCATION` 的规则、运行记录或来源类型。

## 8. 权限与审计

```text
科目体系维护权限
  ├─ 新增、编辑、停用、映射维护
  └─ 禁止直接删除存在任一业务引用的科目

归集规则维护权限
  ├─ 来源规则、项目适用范围、财务分摊规则
  └─ 修改需版本、审批和生效日期

业务执行权限
  ├─ 选择末级科目
  ├─ 覆盖默认科目并填写原因
  └─ 发起投标转入、财务费用分摊

审计查询权限
  └─ 查询映射版本、来源、审批、转入、冲销和历史科目
```

权限不得使用 `if (role === 'xxx')` 硬编码菜单或科目逻辑；复用既有权限码、项目数据权限和审批边界。

## 9. 数据安全、回滚与验收门禁

- V213 阶段不改写历史引用；用户后续明确授权删除旧科目后，V214 只改写科目外键，不改业务金额、来源、状态、期间或审批事实。
- 删除旧科目前必须覆盖全部引用列、记录旧 ID 与替代科目审计，并以数据库约束保证残留引用时迁移失败。
- 迁移前必须导出“旧科目 ID → V2 逻辑科目 → 生效日期 → 映射版本 → 审批依据”清单。
- 迁移后按项目、期间、成本来源、科目四维对账；总额、来源数量、未归类数量、重复转入和超额分摊均为零差异。
- V214 生产执行前必须备份；旧科目物理删除不能靠 Git 回退，恢复时须根据删除审计重建旧主数据并由数据库备份恢复原引用。已提交转入/分摊仍只以反向事实冲销。

## 10. 生产发布前置结论

本地实现已完成旧科目引用迁移和删除验证，但不构成生产授权。任何生产发布、V214 执行、映射版本激活或业务新口径切换前必须完成：

1. 生产或目标环境的完整引用扫描与数据质量报告。
2. 财务确认项目财务费用可用的分摊依据与会计映射口径。
3. 成本、商务、采购、质量安全共同确认本计划第 5 节映射矩阵。
4. 在脱敏目标副本重放 V214，证明旧科目、全部引用残留和金额差异均为 0，再重放已有自动化用例、审批边界和只读对账脚本。
