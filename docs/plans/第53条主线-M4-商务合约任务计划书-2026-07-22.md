# 第53条主线 M4 商务合约任务计划书

**Goal:** 在 M3 已正式收口基础上，将商务合约域 18 条 Legacy 路由迁移为可验收的 Clean-room V2 闭环，确保合同、变更、投标成本、目标成本、成本台账、动态利润、预算及产值计量的金额、权限、租户/项目范围、状态与历史快照均以服务端事实为准。 **Architecture:** 复用现有 V2 公共壳、设计令牌、请求层、权限守卫与服务端 API；按“契约与金额金丝雀 → 合同 → 变更/投标 → 目标成本 → 成本与利润 → 预算/产值 → 全量收口”串行推进；禁止新增平行金额账本、前端权威派生、权限扩张及成本科目中心提前迁移，坚持最小改造、可回滚、逐切片验收。

## 2. 目标与验收方向

M4 完成后应具备：

1. 18 条目标路由全部进入 `V2_ACCEPTED`，不存在 `V2_SOURCE_AVAILABLE` 中间态。
2. 合同创建、编辑、详情回读、审批提交、变更追溯、目标成本版本、成本汇总历史、预算提交与产值计量形成服务端闭环。
3. 所有权威金额使用十进制定点语义；API 金额字段以字符串传输，前端不得用二进制浮点计算或覆盖服务端口径。
4. 页面级、动作级、对象级权限同时成立；跨租户、跨项目访问失败关闭。
5. V2 公共壳项目选择器统一生效；“全部项目”仅用于服务端允许的聚合查询，不得绕过对象范围。
6. 旧页面保留快速回滚入口，直至对应切片完成同 SHA 验收。

## 6. ISSUE-053-017：契约与金额金丝雀

### 6.1 范围

- 只核对、测试并在必要时最小修正共享类型、格式化器、请求契约和权限映射。
- 启动核验若证明既有后端金额出口不是十进制字符串，或列表/详情未按服务端项目可见范围失败关闭，允许在 ISSUE-053-017 精确文件所有权内修正对应 VO/API 出口、Service 查询和负向测试；不得扩为全仓序列化或通用权限重构。
- 选取合同台账只读查询、详情查询、成本汇总历史各一个请求作为金丝雀。
- 不创建或切换 V2 业务路由，不修改数据库结构，不引入新业务状态。

### 6.2 必须产出

- 字段级金额契约矩阵。
- 页面/动作/对象三级权限矩阵。
- 合同、变更、目标成本、成本、预算、产值状态机动作清单。
- 项目与租户范围失败关闭用例。
- 前端禁止浮点权威计算的静态检查或可重复测试。

### 6.3 硬门禁

- 金额、税额、成本、利润、预算、产值金额不得声明为 `number` 或在提交前经 `Number`、`parseFloat`、隐式数值运算处理。
- 税率、数量、百分比即使允许数值类型，也必须明确其精度及与金额计算的服务端边界。
- 列表汇总、详情、历史快照三个来源必须可对账；不一致即停止。
- 401/403、跨项目访问、对象不存在、超范围查询必须失败关闭，且不得先渲染敏感数据。
- 金丝雀引入任何额外写请求、重复提交或未经授权的预取即不通过。

### 6.4 回滚

- 限定在共享类型、格式化器、测试和请求适配层；逐文件回退。
- 任一既有 Legacy 页面回归失败，恢复旧类型/适配逻辑并停止后续 Issue。

## 9. 金额与对账契约

| 口径 | 权威来源 | 前端职责 | 核心对账 |
| --- | --- | --- | --- |
| 合同原额/当前额 | 合同服务 | 字符串展示、原样提交 | 当前额与已核定变更关系 |
| 税额/不含税额 | 服务端税务口径 | 不自行舍入 | 含税额 = 服务端核定关系 |
| 变更申报/核定额 | 变更服务 | 展示状态与差额 | 核定额进入合同当前额的服务端结果 |
| 投标/责任/目标成本 | 成本目标服务 | 展示、录入字符串 | 版本明细与合计 |
| 承诺/实际/预计剩余 | 成本服务 | 展示与追溯 | 台账、汇总、历史快照一致 |
| 动态利润 | 成本控制服务 | 展示，不本地权威计算 | 合同当前额与动态成本口径 |
| 预算/占用/消耗/可用 | 预算服务 | 展示、动作提交 | 预算可用额接口结果 |
| 申报/核定产值 | 产值服务 | 展示、提交业务输入 | 期间累计、合同与结算追溯 |

统一规则：金额 JSON 字段使用规范十进制字符串；禁止科学计数法；空值与 `0` 区分；负数、超大值、最大精度、小数位超限、边界舍入均需后端校验和前端错误展示。
