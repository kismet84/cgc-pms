- 后端已有 8 个驾驶舱只读端点及成本下钻端点：项目、商务、成本、采购、生产、总工、财务、管理层和项目成本分解。

- 项目级角色不得因未选项目、无可见项目或接口失败回退租户全量数据。

## 3. 实施范围

### 3.1 驾驶舱

- 8 类角色：项目经理、商务经理、成本经理、采购经理、生产经理、总工程师、财务、管理层。
- 项目、报告期、角色切换、刷新、空数据、部分失败、过期请求取消和深链恢复。
- 经营健康提示、最高风险、利润/偏差/资金/进度指标、趋势、风险待办和快捷入口。
- 成本科目最多两级下钻；只调用现有 `/dashboard/project/{id}/cost-breakdown`。
- 快捷入口必须同时满足权限和迁移状态。未达到 `V2_ACCEPTED` 的目标不显示为可用入口，不用壳占位冒充完成。

### 3.2 审批工作台

- 待我处理、我已处理、抄送我的、我发起、审批实例详情。
- 已有合法动作：同意、驳回、撤回、重新提交、转办、加签；按后端 `availableActions` 与权限双重控制。
- 审批 POST 使用现有 CSRF；需幂等键时使用平台 `crypto.randomUUID()`，提交中禁用重复点击。
- 不迁移审批流程模板配置 `/approval/process`。

### 3.3 预警中心与报表目录

- 预警筛选、分页、详情/追踪、已读、确认、状态更新、批量操作、订阅配置和规则效果汇总。
- `alert:view` 只读；`alert:edit` 才显示处置动作；`alert:evaluate` 才显示人工评估入口。
- 报表目录读取 `/reports/catalog`，按后端返回的权限码、来源类型和状态决定可达性。
- M1 通知入口可显示有权用户的预警摘要；不接入 SSE、不伪造未读数、不新增通知后端。

- 不增加 ECharts、Ant Design、图表库、状态库或请求库。趋势图使用原生 SVG/CSS，并提供文本/表格等价信息。

- 不切换正式入口，不删除 Legacy，不连接生产数据库，不执行生产发布。

## 5. 目标架构与复用边界

### 5.1 契约层

在 `packages/frontend-contracts/src` 扩展最小稳定契约：

- `dashboard.ts`：8 角色完整 DTO、成本下钻、角色权限/端点映射、查询参数。
- `workflow.ts`：四类列表、实例详情、动作请求和 `availableActions`。
- `alert.ts`：列表、汇总、追踪、订阅及处置请求。
- `report.ts`：报表目录条目。
- `api.ts`：只复用现有分页/响应信封；不建立重复基础类型。

契约只含类型、常量和无副作用纯函数。禁止引入 Vue、Pinia、浏览器 API 或 Legacy 路径。

### 5.2 V2 服务层

复用 `apiRequest`，按业务域增加薄服务文件：

- `src/services/dashboard.ts`
- `src/services/projects.ts`
- `src/services/workflow.ts`
- `src/services/alerts.ts`
- `src/services/reports.ts`

服务层只负责 URL、query、method、body 和 AbortSignal；不持有页面状态，不重复实现错误通知、401 或 CSRF。

### 5.3 页面与状态

- `src/pages/dashboard/**`：驾驶舱容器、角色视图和无副作用格式化/派生函数。
- `src/pages/workbench/**`：审批、预警和报表页面。
- 继续复用 `useWorkspaceStore` 保存全局项目、报告期和对象上下文；不新增 dashboard Store。
- 页面本地状态保存当前角色、筛选、分页、加载、空态和错误态。
- 每次项目/报告期/角色变化取消上一请求；只有最新请求可以回写。
- `router.ts` 用明确组件映射替换上述 M2 路由的 `ShellPlaceholderPage`，其余路由仍走占位页。

### 5.4 视觉与可访问性

- 使用 M1 V2 tokens、`V2Card`、`V2Alert`、`V2Badge`、`V2Button`、`V2Select`、`V2PageState`、`V2Skeleton`、`V2Dialog` 和布局原语。
- 新版经营指挥视图保留信息层级，不复制 Legacy CSS/DOM。
- 原生 SVG 趋势图必须有标题、描述、键盘不可交互时的表格等价信息；颜色不是唯一状态信号。
- 1440、1024、390 三视口无横向溢出；移动端触控目标至少 44px；减少动效模式关闭非必要动画。

## 7. 权限与数据范围矩阵

| 场景 | 前端行为 | 后端证据 | 失败裁决 |
| --- | --- | --- | --- |
| ADMIN/SUPER_ADMIN | 可见 8 角色；仍遵守项目选择 | 既有角色放行 | 不绕过租户边界 |
| 单角色权限 | 只显示对应角色并只请求对应端点 | 对应 `dashboard:*:view` | 其他角色 403/不请求 |

| 审批动作权限 | 仅显示允许动作 | `availableActions` + `@PreAuthorize` | 403/冲突不改事实 |
| `alert:view` | 只读预警 | Alert 访问范围解析 | 不显示编辑/评估 |
| `alert:edit` | 已读、确认、状态和批量操作 | Alert Controller 权限 | 部分失败逐项展示 |
| `alert:evaluate` | 人工评估入口 | Alert Controller 权限 | 无权不请求 |
| 报表目录 | 登录可读，目标按条目权限过滤 | catalog 返回 permissionCode | 未授权目标不可达 |

## 9. 风险、故障注入与停止条件

失败分类：工具或规则未加载归入 `tool_config`，命令参数或转义错误归入 `tool_invocation`，运行服务/端口/测试数据未就绪归入 `environment_prerequisite`，图谱召回不足归入 `retrieval_gap`，可复现的权限、金额、数据范围、契约、测试或构建失败归入 `quality_or_security`；证据冲突时归入 `unknown` 并停止裁决。

金丝雀：`ISSUE-053-005` 是 M2 唯一本地实施金丝雀。只有其契约、项目范围、权限请求选择、类型、边界和目标测试全部通过，才允许进入 `ISSUE-053-006`；该金丝雀不包含生产流量、正式入口或数据写入。

| 风险 | 预防/验证 | 停止条件 |
| --- | --- | --- |
| 项目数据范围扩大 | 普通项目角色真实 API 正负样本；无项目不请求全量 | 观察到越权项目或租户全量 |
| 金额精度/口径漂移 | 字符串传输、格式化单测、Legacy/后端响应对照 | 任一权威金额不一致 |
| 无权端点被预取 | 活动角色按需请求、网络断言 | 无权请求大于 0 |
| 陈旧响应覆盖 | AbortController + 最新请求断言 | 快速切换后展示旧项目/角色数据 |
