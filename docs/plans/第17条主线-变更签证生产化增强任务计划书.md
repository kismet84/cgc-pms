# 变更签证生产化增强 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将变更签证列表页增强到可生产验收状态，收敛查询参数、金额口径、审批状态、移动端可用性和真实浏览器验收。

**Architecture:** 最小可行方案优先复用现有 `variation/order.vue`、`lg-list-page`、`ColumnSettingsButton`、`useColumnSettings`、`/var-orders` 后端接口和既有审批流。不新增依赖，不改数据库 schema；只有真实验收证明接口契约不足时才触碰后端。

**Tech Stack:** Vue 3、TypeScript、Vite、Ant Design Vue、VxeGrid、Vitest、Playwright、Spring Boot `/var-orders` API。

---

## 规则边界

- 主线程只负责计划、分派、督办和验收，不亲自实现代码。
- 仓库存在 `.codegraph/`，执行者理解代码前必须优先使用 CodeGraph。
- 不新增 UI 组件库、状态库、表格库或金额库。
- 不改已应用的 Flyway migration。
- 不重构整个变更模块；本主线只处理变更签证订单页生产化。
- 不把当前页统计包装成全量经营统计；没有后端聚合接口前只标注为当前结果口径。
- 完成后清理本轮 Node/Vite/Playwright 残留进程，或说明保留原因。

## 当前事实

- 前端页面：`frontend-admin/src/pages/variation/order.vue`
- 前端 API：`frontend-admin/src/api/modules/variation.ts`
- 前端类型：`frontend-admin/src/types/variation.ts`
- 后端控制器：`backend/src/main/java/com/cgcpms/variation/controller/VarOrderController.java`
- 后端服务：`backend/src/main/java/com/cgcpms/variation/service/VarOrderService.java`
- 后端列表参数：`pageNo`、`pageSize`、`projectId`、`contractId`、`partnerId`、`varType`、`direction`、`varCode`
- 前端当前列表请求已使用 `pageNo/pageSize`，需要继续通过测试守住该契约。
- 后端 `submitForApproval()` 将状态更新为 `APPROVING`，前端状态字典当前包含 `PENDING` 但不包含 `APPROVING`，这是需要优先核实的生产化风险。
- 前端金额展示 `fmtWan()` 以元转万元展示；明细金额保存直接使用数量乘单价结果，需要验收创建、编辑、列表展示是否存在单位漂移。
- 当前浏览器入口为 `http://localhost:5173/variation/order`。

## 最终上线裁决标准

通过条件：

- `/api/var-orders` 主列表请求使用 `pageNo/pageSize`。
- 所有前端查询控件都有对应后端参数或明确不发送。
- 审批状态 `APPROVING` 可被前端正确显示为“审批中”。
- 金额明细、订单金额、列表万元展示不发生单位漂移。
- 桌面端查询、重置、列设置、刷新、新建、查看、编辑、提交、删除核心入口可用。
- 移动端不出现不可操作的宽表格。
- `pnpm type-check` 通过。
- `pnpm build` 通过。
- 相关 Vitest 通过，或已证明失败项为无关历史基线。
- 浏览器控制台无本任务引入红错。

不通过条件：

- 审批后页面显示原始状态码或错误状态。
- 金额出现 10000 倍漂移。
- 查询参数与后端契约不一致导致筛选失效。
- 桌面或移动端核心操作不可用。
- 类型检查或构建失败且无法证明无关。
- 浏览器验收出现本任务引入的 API 或控制台错误。
