### 2.1 M4-1 按钮材质补充规则

根据页面 02 的用户标注，M4-1 使用以下可检查规则：

- 页面标题区、筛选区、表格行、分页和页面正文操作使用普通 `V2Button`，不得使用玻璃材质。
- `V2Dialog`、`v2-detail-dialog` 内的上下文操作和表单页脚动作允许使用 `V2GlassButton`。
- `V2ConfirmDialog` 的普通、危险确认继续保持语义按钮，不为视觉统一改成玻璃材质。
- 页面不得自行实现玻璃效果；需要玻璃材质时只能复用 `V2GlassButton`。
- M4-1 批次收口前同步校准 `m1-design-system-baseline.md` 的 S14/7.5 和对应自动门禁，消除旧文字与本规则的冲突。

## 4. 单页执行合同

每页严格执行以下闭环：

1. **事实冻结**
   - 记录入口 URL、权限、项目/报告期上下文、主要状态和当前工作区 diff。
   - 打开真实页面，确认不是旧缓存、错误代理、占位页或错误身份。
2. **静态审查**
   - 按 S01～S24 检查标题、壳层、导航、token、层级、结构、表格、分页、筛选、表单、按钮、弹窗、反馈、权限、响应式和交付门禁。
   - 检查业务页是否复制 `components.css` 已有材质或使用裸颜色、字号、阴影、圆角。
3. **交互审查**
   - 覆盖查询、筛选、分页、主要按钮、预览/详情、创建/编辑、取消、错误、loading、空态和无权限态中本页实际存在的路径。
   - 校验失败和请求失败必须保留输入；无权限必须 fail-close。
4. **最小修复**
   - 优先删除页面私有覆盖、复用现有组件和原生语义能力。
   - 仅修改根因文件和最小回归测试，不建立单页抽象或新依赖。
5. **分层验证**
   - L1：本页单测、设计系统门禁、`git diff --check`。
   - L2：`type-check`、`lint:check`、`build` 和本页 E2E。
   - L3：1440、1024、390 实览，检查 computed style、焦点、溢出、关键动作和控制台。
6. **记录与裁决**
   - 只允许“通过”“不通过”“需要确认”。
   - 所有发现归入“本页修复并复验、正式承接、关闭”之一；有悬空项不得进入下一页。

## 8. 风险、回滚与失败分类

| 风险 | 控制 | 最小回滚 |
| --- | --- | --- |
| 页面修复改变业务语义 | 只调整展示和交互；状态、金额、权限与接口保持服务端权威 | 回退本页模板/样式和对应测试 |
| 公共组件修复影响已审页面 | 修改前核对调用方；修改后回归页面 01、02 和设计系统门禁 | 回退公共组件 diff，恢复页面原复用方式 |
| 运行态显示旧代码 | 按运行态规范刷新 V2 前端；以真实 URL、DOM 和 computed style 复验 | 不改业务代码，先恢复正确运行态 |
| 用户标注与旧标准冲突 | 用户明确标注优先；在进入后续页前同步基线和自动门禁 | 回退未完成的基线改动，保持当前两页已确认行为 |
| 脏工作区误覆盖 | 每次修改前核对分支和状态，只改当页必要文件 | 按任务 diff 精确回退，不使用破坏性 Git 命令 |

失败先按唯一契约分类为 `tool_config`、`tool_invocation`、`environment_prerequisite`、`ready_issue_config`、`retrieval_gap`、`quality_or_security` 或 `unknown`；未分类前不得把运行态、代理或工具问题判为页面缺陷。
