# 成本管理模块生产化增强 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将成本管理模块的成本台账、动态成本汇总、目标成本和成本科目入口增强到可生产验收状态，收敛分页契约、筛选控件、金额展示、移动端可用性和真实浏览器验收。

**Architecture:** 最小可行方案优先复用现有 `lg-list-page`、`lg-search-bar`、`ColumnSettingsButton`、`useColumnSettings`、Ant Design Vue 和 VxeGrid。不新增依赖，不改数据库 schema；后端接口已存在时优先修前端契约和控件覆盖，只有验收证明后端缺口时才启用后端补丁。

**Tech Stack:** Vue 3、TypeScript、Vite、Ant Design Vue、VxeGrid、Vitest、Playwright、Spring Boot `/cost-ledger`、`/cost-summary`、`/cost-targets`、`/cost-subjects` API。

### 已发现的明确缺口

- `CostLedgerController.getPage` 后端接收 `pageNo/pageSize`，`CostLedgerQueryParams` 类型也定义 `pageNo`，但 `frontend-admin/src/pages/cost/ledger.vue` 当前发送 `pageNum`。
- `cost/summary.vue` 拉项目列表使用 `getProjectList({ pageNum: 1, pageSize: 50 })`，与项目接口生产口径 `pageNo` 不一致。
- `cost/ledger.vue` 请求对象包含 `contractId`、`partnerId`、`costType`、`startDate/endDate`，但查询栏当前只暴露 `keyword`、`projectId`、`costSubjectId`、`sourceType`、`costStatus`。
- `cost-target/index.vue` 请求对象包含 `approvalStatus`、`isActive`，但查询栏当前只暴露 `versionNo`、`projectId`。
- `cost/ledger.vue`、`cost-target/index.vue`、`cost/summary.vue` 主要依赖桌面 `vxe-grid`，移动端缺少同类主线16/17已验证过的最小卡片 fallback。
