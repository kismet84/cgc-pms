# 11 安全规范

## 认证安全

1. JWT + HttpOnly Cookie，前端不存 Token
2. 生产 Cookie 必须 Secure + HTTPS
3. JWT secret 来自环境变量，不用默认值
4. Refresh Token 失败清理用户状态并跳转登录

## 授权安全

1. 所有保护接口必须鉴权
2. 关键接口 `@PreAuthorize` 或等价校验
3. Service 层校验租户和业务数据归属
4. 管理员权限不被普通角色通过菜单配置绕过

## 多租户与项目隔离

1. 主业务表必须有 `tenant_id`
2. 查询按当前租户过滤
3. 项目级业务按项目权限过滤
4. 导出、统计、驾驶舱不绕过数据权限

```java
Long tenantId = UserContext.getCurrentTenantId();
queryWrapper.eq("tenant_id", tenantId);
```

## 输入校验

1. 后端强制校验必填、长度、枚举、金额范围
2. 前端校验仅为体验优化
3. 文件名、备注等自由文本不进入 SQL 拼接
4. SQL 使用 MyBatis 参数绑定
5. 导入/上传限制文件类型和大小

## 文件安全

1. 通过后端上传，前端不持有永久 MinIO 密钥
2. 下载校验当前用户对业务对象的权限
3. 预签名 URL 设置有效期
4. 合同、发票、结算附件不公开桶访问
5. 文件记录保留上传人、时间、业务类型、业务 ID

## 敏感配置

不提交到仓库：生产数据库密码、JWT secret、MinIO secret key、Redis 密码、HTTPS 私钥、第三方系统密钥。

`.env.example` 只提供示例值。`.env` 按环境隔离管理。

## 日志与审计

必须记录：登录/退出、增删改、提交审批、审批处理、文件上传下载、权限不足和认证失败。

不记录：明文密码、Token、Cookie 完整值、密钥、大文件内容。

## 上线安全检查

1. 生产 `JWT_SECRET` 已替换
2. HTTPS 开启，Cookie Secure
3. Swagger 生产禁用或受保护
4. CORS 只允许正式域名
5. 数据库和 MinIO 不暴露公网
6. 默认账号密码已修改
7. 备份任务已配置
8. 日志不含敏感信息
