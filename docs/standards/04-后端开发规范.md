# 04 后端开发规范

> 适用版本：v1.5。历史结论和旧路径仅作回溯，当前实现以本分支代码、配置和实时验证为准。

## 角色入口

主负责人：后端设计师 / 后端架构助手。补充阅读：[02 系统架构](02-系统架构.md)、[06 API 契约规范](06-API契约规范.md)、[09 测试规范](09-测试规范.md)。

## 技术栈

Java 21、Spring Boot 3.3.5、Spring Security、MyBatis-Plus 3.5.9、Flyway、Redis、MinIO、Jasypt、JUnit 5、MockMvc。

新增框架需说明收益、风险、替代方案。

## 分层规范

| 层 | 职责 | 约束 |
|----|------|------|
| Controller | 参数绑定、权限注解、响应包装 | 不写业务流程 |
| Service | 业务规则、事务、状态机、跨表写 | 方法表达业务动作 |
| Mapper | SQL、MyBatis-Plus 查询、批量操作 | 不处理权限和状态 |
| Entity | 数据库表映射 | 不承载业务逻辑 |
| DTO / VO | 请求输入、响应输出 | 不与 Entity 混用 |

## 统一响应

成功：
```json
{ "code": "0", "message": "success", "traceId": "xxx", "data": {} }
```

失败：
```json
{ "code": "COST_TARGET_NOT_FOUND", "message": "目标成本不存在", "traceId": "xxx", "data": null }
```

规则：
1. `code="0"` 表示成功
2. 业务失败通过 `BusinessException` 抛出，全局异常处理器转换
3. 错误码稳定可依赖，不随中文文案变化
4. 系统异常不暴露堆栈、SQL、密钥
5. 日志必须带 `traceId`

## 权限与用户上下文

```java
// Controller 层
@PreAuthorize("hasAuthority('cost:target:add')")

// Service 层
Long tenantId = UserContext.getCurrentTenantId();
if (!Objects.equals(entity.getTenantId(), tenantId)) {
    throw new BusinessException("RESOURCE_NOT_FOUND");
}
```

规则：
1. 查询必须限制当前租户
2. 项目级数据需校验项目访问权限
3. 即使未实现项目权限，也不得绕过租户校验

## ID 与金额规范

1. 数据库主键：`Long`（雪花算法）
2. JSON 输出 ID 必须序列化为字符串
3. 创建接口返回 ID 优先返回 `String`
4. 金额使用 `BigDecimal`，JSON 输出为字符串
5. 禁止 `double`/`float` 做金额计算

```java
@JsonSerialize(using = ToStringSerializer.class)
private Long id;

@JsonSerialize(using = ToStringSerializer.class)
private BigDecimal contractAmount;
```

## 事务规范

必须使用事务的场景：
- 主表 + 子表创建/更新
- 审批提交/通过/驳回及业务回写
- 成本生成、付款状态更新、库存交易
- 逻辑删除主表并校验子表引用

事务边界位于 Service 层。审批引擎回调同应用内模块用领域服务直接调用，不走 HTTP。

## 逻辑删除与唯一约束

系统使用 MyBatis-Plus 逻辑删除（`deleted_flag`）。唯一索引需包含该字段：

```sql
UNIQUE KEY uk_tenant_code (tenant_id, code, deleted_flag)
```

若历史索引不含 `deleted_flag`：
- 删除记录仍占用唯一键
- 唯一性检查需包含已删除记录
- 不能用默认查询（过滤 `deleted_flag=0`）判重

## 日志规范

1. 关键动作写 info：创建、提交、审批、删除
2. 异常写 warn/error，不吞堆栈
3. 不输出密码、Token、密钥、完整 Cookie
4. 大对象、附件内容不默认写入日志

## Swagger 规范

1. 接口路径不带 `/api` 前缀（由 context-path 统一处理）
2. 查询接口使用 `pageNo`、`pageSize`
3. 删除接口需明确是逻辑删除、物理删除还是作废
4. 操作接口用动词子路径：`/{id}/submit`、`/{id}/activate`

## 测试要求

至少覆盖：
- 状态流转（Controller + Service）
- 权限拦截
- ID 字符串序列化
- 金额计算与成本生成
- 逻辑删除唯一键
- 审批并发、重复提交、驳回重提

cd backend
.\mvnw.cmd test "-Djasypt.encryptor.password=dev-jasypt-key"
