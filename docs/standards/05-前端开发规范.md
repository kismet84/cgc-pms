# 05 前端开发规范

## 角色入口

主负责人：前端设计师 / 前端架构助手。补充阅读：[00 UI 设计基线](00-UI-Design-Baselines-and-Code-Specifications.md)、[prompt/frontend-docker-ui-test-rules.md](prompt/frontend-docker-ui-test-rules.md)、[09 测试规范](09-测试规范.md)。

## 技术栈

Vue 3.5、TypeScript 5.6、Vite 6、Ant Design Vue 4、Pinia、Vue Router、Axios、VxeTable、ECharts。

不引入第二套 UI 组件库或状态管理方案，除非有明确迁移计划。

## 目录职责

| 目录 | 职责 |
|------|------|
| `src/pages/<module>/` | 页面与页面私有组件 |
| `src/api/modules/<module>.ts` | 模块 API 封装 |
| `src/types/<module>.ts` | 接口类型、枚举、状态标签 |
| `src/router/index.ts` | 路由定义、守卫、页面标题 |
| `src/router/navigation.ts` | 导航菜单业务域、顺序 |
| `src/stores/` | 跨页面共享状态 |
| `src/components/` | 共享组件 |

页面私有逻辑留在页面目录，不用过早提升。

## TypeScript 规范

1. API 返回值必须声明泛型类型
2. 后端 ID 统一为 `string`（不用 `number`）
3. 金额统一为 `string`，展示时格式化
4. 枚举集中定义，不在页面硬编码
5. 禁止 `any` 绕过接口契约

```ts
interface CostTargetVO {
  id: string
  projectId: string
  versionNo: string
  totalTargetAmount: string
  approvalStatus: 'DRAFT' | 'APPROVING' | 'APPROVED' | 'REJECTED'
}
```

## API 调用规范

1. 页面不裸用 `axios`，通过 `src/api/modules/` 调用
2. URL 不写 `/api` 前缀（Axios baseURL 处理）
3. 删除、提交、审批等操作必须有确认交互和加载态
4. 异常不吞错，至少 `console.error(e)` 并恢复加载态

```ts
import { invoiceApi } from '@/api/modules/invoice'

const handleDelete = async (id: string) => {
  try {
    await invoiceApi.delete(id)
    message.success('删除成功')
    fetchList()
  } catch (e) {
    console.error(e)
  }
}
```

## 页面状态模式

| 状态 | 用途 |
|------|------|
| `loading` | 列表首次加载 |
| `saving` | 保存中 |
| `submitting` | 提交审批中 |
| `deleting` | 删除中 |
| `query` | 查询条件 |
| `tableData` | 表格数据 |
| `pagination` | 分页 `{pageNo, pageSize, total}` |

规则：
- 加载态在 `finally` 中恢复
- 删除成功后刷新；当前页为空可回退一页
- 按钮由后端权限和业务状态控制，不以静态角色判断

## UI 与样式

### 样式命名体系

项目 CSS 类名经历三代演进，当前以 `lg-*` 为目标标准：

| 前缀 | 来源 | 状态 | 说明 |
|------|------|------|------|
| `lg-*` | `src/assets/styles/global.css` (L387-847) | ✅ **目标标准** | 从 ContractLedgerPage 提炼的共享布局类，覆盖页面容器、搜索栏、双栏布局、KPI 卡片、表格容器、工具栏、分页、分析面板、类型分布、预警列表、响应式 |
| `app-*` | `global.css` (L78-118) | ⚠️ 遗留 | `app-page`、`app-page-title`、`app-panel`、`app-kpi-value`、`app-muted` |
| `pt-*` | `global.css` (L124-360) | ⚠️ 遗留 | `pt-page-head`、`pt-filter-surface`、`pt-kpi-strip`、`pt-ledger-layout`、`pt-panel`、`pt-form-grid` 等 |

**核心原则：新代码一律使用 `lg-*`，不再引入 `app-*` 或 `pt-*`。**

### `lg-*` 核心类一览

| 类 | 用途 |
|----|------|
| `lg-page` | 页面容器（min-height, padding） |
| `lg-page-head` | 页面头部（flex, space-between） |
| `lg-search-bar` | 搜索栏（flex row + border-bottom） |
| `lg-grid` | 双栏布局（left + analysis rail） |
| `lg-kpi-strip` / `lg-kpi-card` | KPI 横条卡片（5 列 grid） |
| `lg-table-wrap` | 表格容器（白底，阴影，表头 #ebede9） |
| `lg-toolbar` / `lg-toolbar-left` / `lg-toolbar-right` | 工具栏（左右分组） |
| `lg-panel` | 分析面板（--surface-subtle 背景） |
| `lg-type-list` / `lg-type-row` | 分布色条列表（5 列 grid） |
| `lg-card-list` / `lg-card-item` | 移动端卡片列表 |
| `lg-pagination` | 分页容器 |
| `lg-link` / `lg-money` / `lg-ops` / `lg-del` | 原子样式 |

### 增量迁移清单

将现有页面从 `pt-*` / `app-*` 迁移至 `lg-*` 时，按以下 checklist 逐项对照：

- [ ] 页面容器：`pt-*` 页面的外层容器 → `lg-page`
- [ ] 页面头部：`pt-page-head` → `lg-page-head`
- [ ] 搜索/筛选区：`pt-filter-surface` + `pt-filter-row` → `lg-search-bar`
- [ ] KPI 横条：`pt-kpi-strip` + `pt-kpi` → `lg-kpi-strip` + `lg-kpi-card`
- [ ] 表格容器：`pt-panel` / `pt-table-panel` → `lg-table-wrap`
- [ ] 工具栏：内联 flex 工具栏 → `lg-toolbar` + `lg-toolbar-left` / `lg-toolbar-right`
- [ ] 分页区域：`pt-pagination` → `lg-pagination`
- [ ] 分析面板：`pt-analysis-rail` → `lg-analysis-rail`（配合 `lg-panel`）
- [ ] 双栏布局：`pt-ledger-layout` → `lg-grid`
- [ ] 原子样式：`pt-link`、`pt-danger` → `lg-link`、`lg-del`
- [ ] 金额展示：`cl-money`、`pt-kpi-value` → `lg-money`、`lg-kpi-card-value`
- [ ] 迁移完成后，确认页面中不再引用 `app-*` / `pt-*` 类（可 grep 验证）

> 迁移过程中发现功能缺失的 `lg-*` 类，应在 `global.css` 中补充，保持与现有 `lg-*` 命名一致，不回退使用 `pt-*`。

### 通用 UI 规范

1. 表格页必须包含：筛选区、操作区、分页、空状态、错误恢复
2. 按钮不溢出文字
3. 图标优先用 Ant Design Icons

## 表单规范

1. 必填项前端后端双重校验
2. 日期使用后端约定格式
3. 金额保留两位精度，提交为字符串
4. 下拉字典来自字典接口/store，不硬编码中文标签
5. 编辑页加载失败需给出提示并可重试

## 权限规范

1. 菜单权限来自后端用户信息 + 路由配置
2. 按钮权限以后端 `availableActions` 或接口返回为准
3. 前端隐藏按钮只是体验优化，后端强制拦截

## 测试

```bash
pnpm build              # 类型检查 + 构建
pnpm test:unit          # 单元测试（Vitest）
pnpm test:coverage      # 覆盖率
pnpm exec playwright test   # E2E（Playwright）
```

高风险页面需覆盖：登录与权限、审批流转、删除/保存/刷新、表单校验、ID/金额字符串契约。
