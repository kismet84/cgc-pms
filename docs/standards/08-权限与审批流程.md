# 08 权限与审批流程

> 适用版本：v1.5。历史结论和旧路径仅作回溯，当前实现以本分支代码、配置和实时验证为准。

## 角色入口

主负责人：产品/需求负责人负责流程口径，后端设计师负责安全边界，安全审计员负责越权风险复核。补充阅读：[11 安全规范](11-安全规范.md)。

## 权限模型

RBAC + 数据权限：

```
用户 → 角色 → 菜单权限 / 按钮权限 / API 权限 → 项目数据范围
```

权限校验分层：

| 层 | 职责 | 强度 |
|----|------|------|
| 前端路由 | 控制菜单和页面入口 | 体验优化 |
| 前端按钮 | 控制操作可见性 | 体验优化 |
| 后端 `@PreAuthorize` | 强制 API 权限 | 安全边界 |
| Service | 租户、项目、业务状态校验 | 安全边界 |

> 前端隐藏按钮不等于授权。后端必须强制拦截。

## 权限编码

格式：`module:resource:action`

| 权限 | 说明 |
|------|------|
| `cost:target:query` | 查询目标成本 |
| `cost:target:add` | 新增 |
| `cost:target:edit` | 编辑 |
| `cost:target:delete` | 删除 |
| `cost:target:submit` | 提交审批 |
| `workflow:task:approve` | 审批处理 |

菜单权限和按钮权限需可区分。权限编码修复通过迁移脚本完成。

## 项目数据权限

1. 管理员可按角色拥有全局权限
2. 项目成员默认只访问所属项目数据
3. 合同、成本、付款、库存等查询必须限制租户，逐步补齐项目范围
4. 导出、统计、驾驶舱遵守同样的数据权限

## 审批引擎对象

| 对象 | 说明 |
|------|------|
| `wf_template` | 审批模板 |
| `wf_template_node` | 模板节点 |
| `wf_instance` | 审批实例 |
| `wf_node_instance` | 节点实例 |
| `wf_task` | 审批任务 |
| `wf_record` | 审批记录 |
| `wf_cc` | 抄送 |

## 业务类型与状态

常见业务类型：`CONTRACT`、`PURCHASE_ORDER`、`MATERIAL_RECEIPT`、`SUB_MEASURE`、`PAY_REQUEST`、`VAR_ORDER`、`SETTLEMENT`、`COST_TARGET`

实例状态：`RUNNING` | `APPROVED` | `REJECTED` | `WITHDRAWN` | `VOIDED`

任务状态：`PENDING` | `APPROVED` | `REJECTED` | `TRANSFERRED` | `CANCELED`

## 提交审批规则

1. 业务单据存在且属于当前租户
2. 业务状态允许提交
3. 必填字段完整
4. 提交后业务状态更新为 `APPROVING`
5. 审批引擎创建实例、节点、任务、记录
6. 重复提交必须拦截

## 审批处理规则

1. 只有当前处理人可操作
2. 任务必须是 `PENDING`，实例必须是 `RUNNING`
3. 并发处理通过任务版本或状态二次校验拦截
4. 同意、驳回、撤回、转办、加签写入记录
5. 完成后通过业务回调更新业务状态

## 前端按钮规则

前端不硬编码审批按钮。推荐后端返回可用动作：

```json
{
  "availableActions": [
    { "actionType": "APPROVE", "actionName": "同意", "enabled": true, "requireComment": false },
    { "actionType": "REJECT", "actionName": "驳回", "enabled": true, "requireComment": true }
  ]
}
```

审批中心固定入口：`/approval/todo`、`/approval/done`、`/approval/cc`。

## 审批审计

1. 审批记录不可物理删除
2. 驳回重提、撤回重提保留历史轮次
3. 审批意见、附件、处理人、时间必须可追溯
4. 业务详情可查看审批轨迹
