# 02 系统架构

> 适用版本：v1.5。历史结论和旧路径仅作回溯，当前实现以本分支代码、配置和实时验证为准。

## 架构形态

CGC-PMS 是前后端分离系统：

```text
Vue 管理后台 -> /api -> Spring Boot 后端 -> MySQL/H2
                                     -> Redis
                                     -> MinIO
                                     -> SSE/站内通知
```

后端统一上下文路径为 `/api`。前端 Axios 默认 baseURL 为 `/api`，生产环境由 Nginx 代理。

## 后端架构

入口：`backend/src/main/java/com/cgcpms/CgcPmsApplication.java`

典型分层：

| 层 | 职责 |
|----|------|
| Controller | 参数绑定、权限注解、统一响应 |
| Service | 业务规则、事务、状态流转 |
| Mapper | MyBatis-Plus 查询与 XML SQL |
| Entity | 表结构映射 |
| DTO / VO | 请求和响应模型 |

当前后端模块：

| 模块 | 职责 |
|------|------|
| `auth`、`system` | 登录、JWT、Token 刷新、用户、角色、菜单、字典 |
| `project`、`partner`、`org`、`material` | 项目、合作方、组织和物料主数据 |
| `contract`、`variation`、`bid` | 合同、合同清单、付款条件、变更、签证、投标成本 |
| `cost`、`revenue`、`overhead`、`accounting` | 成本、收入、间接费、核算相关能力 |
| `purchase`、`receipt`、`inventory`、`requisition` | 采购、收货、库存、领料 |
| `subcontract`、`settlement` | 分包任务、计量、结算 |
| `payment`、`invoice` | 付款申请、财务回写、发票登记与识别 |
| `workflow`、`notification`、`alert`、`audit` | 审批、通知、预警、审计 |
| `dashboard`、`file`、`common`、`config` | 驾驶舱、文件、公共能力、全局配置 |

## 数据架构

- MySQL 迁移：`backend/src/main/resources/db/migration/`
- H2 迁移：`backend/src/main/resources/db/migration-h2/`
- MyBatis XML：`backend/src/main/resources/mapper/**`
- 数据库结构变化必须新增 Flyway migration，不修改已应用脚本。

## 认证与会话

- JWT + HttpOnly Cookie。
- 后端 `@PreAuthorize` 是安全边界。
- Service 层必须校验租户、项目和业务状态。
- Redis 用于 Token 黑名单、缓存和运行时能力。

## 部署架构

- 开发：Docker Compose 可启动 MySQL、Redis、MinIO、Backend、Frontend。
- 生产：Nginx HTTPS 入口，反向代理前端静态资源与后端 `/api`。
- 监控：Spring Actuator + Prometheus 指标。
