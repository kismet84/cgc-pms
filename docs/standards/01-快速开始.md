# 01 快速开始

> 适用版本：v1.5。历史结论和旧路径仅作回溯，当前实现以本分支代码、配置和实时验证为准。

## 环境要求

| 项 | 版本 |
|----|------|
| JDK | 21 |
| Node.js | 20+ |
| pnpm | 11.x，仓库声明为 `pnpm@11.0.9` |
| Docker | 支持 Compose v2 |
| 数据库 | MySQL 8；本地/测试可用 H2 MySQL 模式 |

## 一键启动

Windows 本地优先使用：

```bash
scripts\start-dev.bat
```

全新数据库只执行 B215 最终基线，默认业务数据为空。首次平台管理员必须通过外部 Secret 显式 bootstrap；配置与演示项目加载见[数据库最终基线 B215](../database/database-baseline-v215.md)。

启动后访问：

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:5173 |
| 后端 API | http://localhost:8080/api |
| Swagger | http://localhost:8080/api/swagger-ui.html |
| MinIO 控制台 | http://localhost:9001 |

## Docker 开发环境

```bash
cd deploy
copy .env.example .env
docker compose -f docker-compose.dev.yml up -d
docker compose -f docker-compose.dev.yml logs -f
```

如果前端改动后浏览器仍显示旧页面，重启 `cgc-pms-frontend-dev`，等待 `180秒` 并确认 Vite ready 后再验收。

## 运行态刷新

```bash
python scripts/rebuild.py
python scripts/rebuild.py backend
python scripts/rebuild.py frontend
python scripts/rebuild.py --test
```

前后端刷新后按 `scripts/codex-autopilot/codex-autopilot.config.json` 的 `runtimeRefresh.waitSeconds` 等待再做验收；本地前端默认以 `http://localhost:5173` 为验证入口。

## 手动启动

基础设施：

```bash
cd deploy
docker compose -f docker-compose.dev.yml up -d mysql redis minio
```

后端：

```bash
cd backend
.\mvnw.cmd spring-boot:run "-Dspring-boot.run.profiles=dev"
```

H2 本地模式：

```bash
cd backend
.\mvnw.cmd spring-boot:run "-Dspring-boot.run.profiles=local"
```

前端：

```bash
cd frontend-admin
pnpm install
pnpm dev
```

## 常用验证

后端：

```bash
cd backend
.\mvnw.cmd test "-Djasypt.encryptor.password=dev-jasypt-key"
.\mvnw.cmd verify "-Djasypt.encryptor.password=dev-jasypt-key"
```

前端：

```bash
cd frontend-admin
pnpm build
pnpm test:unit
pnpm test:coverage
pnpm check:bundle-size
```

E2E：

```bash
cd frontend-admin
pnpm exec playwright test
```

## 常见故障

| 现象 | 处理 |
|------|------|
| PowerShell 报 Maven `Unknown lifecycle phase` | 带点号的 `-D` 参数加引号 |
| 前端端口 5173 混乱 | 优先访问 `127.0.0.1:5173` 或重启 Docker 前端 |
| backend 重启后浏览器回到 `/login`，前端日志报 `/api/*` 命中旧容器 IP（如 `ECONNREFUSED 172.19.x.x:8080`） | 优先判定为前端 dev server 代理缓存旧 backend IP，先执行 `python scripts/rebuild.py frontend`，再复验 `http://localhost:5173`；不要先按路由守卫或后端业务回退排查 |
| Flyway checksum 不匹配 | 不修改已执行迁移，新增版本化 migration |
| 页面可达但功能报错 | 先看浏览器 console、Network 和后端日志，不按端口故障处理 |
