# 业务编号生成规范

## 1. 统一格式

内部业务编号统一为：`{PREFIX}-{yyyyMMdd}-{SEQ}`。

- `PREFIX`：固定大写业务前缀。
- `yyyyMMdd`：创建业务记录时的系统日期。
- `SEQ`：同一租户、同一业务类型、同一日期内从 `001` 开始的三位序号。
- 唯一范围：`tenant_id + business_code`。
- 编号创建后不可编辑，不因业务状态、项目或责任人变化而改变。

## 2. 前缀表

| 业务对象 | 前缀 | 示例 |
| --- | --- | --- |
| 项目 | `XM` | `XM-20260720-001` |
| 合作方 | `PTN` | `PTN-20260720-001` |
| 合同 / 合同变更 | `CT` / `CC` | `CT-20260720-001` |
| 变更签证 | `VO` | `VO-20260720-001` |
| 采购申请 / 采购订单 | `PR` / `PO` | `PO-20260720-001` |
| 收料 / 领料 / 退料 / 退供 | `MR` / `REQ` / `MRT` / `SRT` | `MR-20260720-001` |
| 分包任务 / 计量 | `SUB` / `SM` | `SM-20260720-001` |
| 付款申请 / 付款记录 | `PAY` / `PMT` | `PMT-20260720-001` |
| 结算 / 费用 / 收入 | `STL` / `EXP` / `RV` | `STL-20260720-001` |
| 资金日记账 | `CJ` | `CJ-20260720-001` |
| 文档生成记录 | `DOC` | `DOC-20260720-001` |

## 3. 实现边界

- 新编号复用 `CodeGenerationService`，禁止各模块新增另一套日期与序号算法。
- 序号计算包含软删除记录，避免编号复用。
- 数据库保留租户内唯一约束，作为并发冲突最终防线。
- 银行/支付渠道流水号、会计凭证号、供应商发票号等外部编号保持原系统语义，不按本规范重写。
- 历史内部编号通过版本化数据库迁移一次性规范化；迁移按租户、创建日期、主键稳定排序，保证重复执行结果确定。
