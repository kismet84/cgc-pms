# v1.0 历史问题四态分类

## 定位与边界

本文件是 v1.0 正式历史问题的当前分类索引，不是 v1.0 backlog 的恢复，也不直接产生 Ready。事实范围仅为 Git 跟踪的 `docs/archive/v1.0/**`；`archive/v1.0/private/**` 未读取、未扫描、未入图。

分类定义：

- **已解决**：v1.0 记录已明确 Done/关闭，或 v1.5 有新鲜证据证明对应问题已解除。
- **仍适用**：问题方向在 v1.5 当前项目地图或 Current Focus 中仍成立，但尚未通过决策门成为 Ready。
- **已过期**：只针对旧分支、旧 SHA、旧工作区、旧工具能力或已被当前事实取代。
- **需要复验**：历史证据不足以证明 v1.5 当前行为，且风险仍可能影响权限、数据、运行态或上线裁决。

## 跟踪规则

1. “仍适用”先进入产品情报或 Ad-hoc Candidate，不直接拆 Ready。
2. “需要复验”只有直接阻塞已选产品目标、安全边界或正式验收时，才按缺口修复/回归证明进入 Ready。
3. “已解决”在 v1.5 被重新引用时必须补当前证据。
4. “已过期”保留历史索引，不进入 Current Focus。
5. 默认知识图谱搜索只返回 current；历史查询必须显式使用 `scope=historical` 或 `scope=all`。
6. 10 个“仍适用”和 8 个“需要复验”域的全量处置顺序、验收门槛和 Ready 边界统一见 [第40条主线修复计划](../plans/第40条主线-v1.0历史问题分类治理任务计划书.md)。
