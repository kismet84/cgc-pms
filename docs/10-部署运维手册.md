# 10 部署运维手册

## 本地开发

### 全容器化（推荐）

```bash
scripts/start-dev.bat
```

启动 5 个容器：MySQL、Redis、MinIO、Backend、Frontend。宿主机无需 JDK/Node。

### 手动启动

```bash
# 基础设施
cd deploy && docker compose up -d

# 后端
cd backend
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev

# 前端
cd frontend-admin
pnpm install && pnpm dev
```

## 生产部署

```bash
cd deploy
cp .env.example .env
# 编辑 .env：修改密码、JWT secret、CORS 域名、MinIO 密钥
mkdir -p ssl
docker compose -f docker-compose.prod.yml up -d
```

生产 `.env` 必须修改：数据库密码、Redis 密码、MinIO 密钥、JWT secret、CORS 域名、HTTPS 证书路径。

## Docker 后端 JAR 更新

容器内后端运行 JAR，代码变更后需重打 JAR：

```bash
docker exec cgc-pms-backend-dev sh -lc "MAVEN_CONFIG= ./mvnw -q -DskipTests clean package"
docker restart cgc-pms-backend-dev
```

## `/api` 上下文路径

后端配置了 `server.servlet.context-path: /api`。直接访问接口必须带 `/api`：

```
http://localhost:8080/api/workflow/templates  ← 正确
http://localhost:8080/workflow/templates      ← 404
```

## 备份

| 对象 | 方式 | 频率 |
|------|------|------|
| MySQL | `mysqldump` 全量 | 每日 |
| MinIO | 对象存储生命周期 / `mc mirror` | 每日增量 |
| `.env` | 安全存储，不入仓库 | 每次修改后 |

## 回滚

- 代码回滚：回滚镜像/JAR → 重启后端 → 验证健康检查
- 数据库：Flyway 只前进不回滚。高风险迁移上线前准备反向修复脚本和备份

## 监控

至少监控：
- 后端健康检查端点和 error 日志
- MySQL 连接数、慢查询、磁盘
- Redis 可用性
- Nginx 4xx/5xx
- 审批待办积压

## 常见故障

| 症状 | 常见原因 | 处理 |
|------|----------|------|
| 接口 404 `No static resource` | 漏了 `/api` | 检查路径 |
| 页面 401 | Token 过期 | 重新登录 |
| 页面 403 | 权限不足 | 检查角色/权限码 |
| Duplicate entry | 逻辑删除唯一索引冲突 | 检查索引含 `deleted_flag` |
| 目标成本不存在 | Long ID 精度丢失 | ID JSON 必须字符串 |
| Docker 重启后接口失败 | 服务未完全启动 | 等待 ~2 分钟后重试 |
| Flyway checksum mismatch | 迁移脚本被修改 | 不修改已执行脚本 |
