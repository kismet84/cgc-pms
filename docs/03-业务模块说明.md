# 03 业务模块说明

## 总体业务闭环

| 闭环 | 链路 |
|------|------|
| 合同闭环 | 起草 → 审批 → 签订 → 履约 → 变更 → 结算 |
| 采购库存闭环 | 需求 → 订单 → 收货 → 入库 → 出库 → 成本归集 |
| 分包闭环 | 任务 → 履约 → 计量确认 → 成本归集 → 付款 → 结算 |
| 成本闭环 | 目标成本 → 合同锁定 → 业务生成 → 动态成本 → 偏差预警 |
| 资金闭环 | 付款条件 → 申请 → 审批 → 支付记录 → 发票 → 分析 |
| 结算闭环 | 合同 + 变更 + 验收 + 付款 + 发票 + 成本 → 结算确认 |

核心原则：

| 原则 | 说明 |
|------|------|
| 项目为根 | 所有业务主表必须关联 `project_id` |
| 合同为纲 | 涉及履约、付款、结算的业务必须关联 `contract_id` |
| 执行为据 | 成本依据来自验收、计量、签证等业务事实 |
| 成本为果 | 成本台账由业务模块生成，不作为独立录入口 |
| 付款为流 | 付款是资金流，不等同于成本发生 |

## 项目管理

项目是所有业务数据的根节点。

规则：
1. 业务主表必须持有 `projectId` 或可通过上游追溯
2. 用户访问数据前需通过角色 + 项目权限双重校验
3. 项目禁用/归档后仅允许查询，不允许新增业务单据

## 合同管理

核心对象：

| 对象 | 说明 |
|------|------|
| 合同主表 | 编号、名称、类型、金额、状态、合作方 |
| 合同清单 | 明细、工程量、单价、金额 |
| 付款条件 | 节点、比例、金额、质保金 |
| 合同变更 | 金额、范围或条款调整 |

规则：
- 合同编号租户内唯一
- 金额使用 `BigDecimal`/字符串传输
- 审批通过后才能作为正式付款和结算依据
- 变更必须走审批

## 成本与目标成本

成本口径：

| 口径 | 说明 |
|------|------|
| 目标成本 | 项目级成本控制目标 |
| 合同锁定成本 | 已签合同的成本承诺 |
| 已发生成本 | 已验收、已计量、已确认的业务事实 |
| 已付款金额 | 财务实际支付，不等同于成本 |
| 动态成本 | 已发生 + 预计待发生 |
| 成本偏差 | 动态成本 vs 目标成本 |

规则：
- 成本生成必须有 `sourceType` + `sourceId` 可追溯
- 同一来源不得重复生成成本
- 目标成本审批中不可编辑/删除

## 审批与状态

通用业务状态：

| 状态 | 说明 |
|------|------|
| `DRAFT` | 草稿，可编辑 |
| `APPROVING` | 审批中，关键字段锁定 |
| `APPROVED` | 已通过，可进入后续业务 |
| `REJECTED` | 已驳回，可按规则修改后重提 |
| `WITHDRAWN` | 已撤回 |
| `VOIDED` / `CANCELLED` | 已作废 |

规则：
- 前端不自行判断审批按钮，以后端返回为准
- 审批通过后由后端回调更新业务状态
- 审批记录不可物理删除

## 预警

预警触发类型：

| 类型 | 示例 |
|------|------|
| 合同 | 到期、超金额、履约逾期 |
| 成本 | 动态成本超目标、来源缺失 |
| 付款 | 超比例付款、未验收付款 |
| 库存 | 低库存、异常出入库 |
| 结算 | 资料缺失、数据不一致 |
