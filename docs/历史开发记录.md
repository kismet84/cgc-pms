## 稳定架构决策

- 前后端分离，后端统一 `/api` 上下文路径。
- 后端按业务域分包，保持 Controller → Service → Mapper → Entity/DTO/VO 分层。
- 数据库变化使用新的 Flyway migration，禁止修改已应用脚本。
- 前端复用 `lg-*` 设计系统和共享组件。
- 验收结论必须来自当前命令、代码、运行态或 UI 证据。
