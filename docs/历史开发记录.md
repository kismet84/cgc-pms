## 架构决策

- 前后端分离，后端统一 `/api` 上下文路径。
- 后端按业务域分包，保持 Controller -> Service -> Mapper -> Entity/DTO/VO 分层。
- 数据库结构变化使用 Flyway 新 migration，禁止修改已应用脚本。
- 前端使用 `lg-*` 设计系统和共享组件，列表页优先对齐合同台账等已确认基线。
- 验收结论必须来自当前命令、代码、日志或 UI 证据，历史报告不能替代重新验证。
