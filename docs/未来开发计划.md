## 二、业务闭环完善

### 2.1 P1 — 分包任务与计量显式关联

**现状**：`SubMeasure` 与 `SubTask` 通过 `projectId + contractId + partnerId` 间接关联。
**方案**：
1. `sub_measure` 表新增 `sub_task_id` 列（可空，渐进式迁移）
2. 后端 `SubMeasure` 实体/VO/Service 同步更新
3. 前端计量编辑页新增「关联分包任务」下拉选择器
4. Flyway 迁移 V91+

### 2.2 P2 — 出库业务单据化

**现状**：出库仅支持手工输入（`POST /inventory/stock/out`），无审批流。
**方案**：新增「领料申请」模块（`MaterialRequisition`），审批通过后自动调用 `MatStockService.stockOut(sourceType="MAT_REQUISITION")`。

### 2.3 P2 — 项目自动归档通知

**现状**：项目归档需手动调用 `PUT /projects/{id}/archive`。
**方案**：结算审批通过后异步检查项目是否满足归档条件（所有合同已结算 + 所有付款已付清 + 所有结算已定案），满足时通知项目经理。

### 2.4 P3 — 发票强制关联付款记录

**现状**：`POST /invoices` 可选关联 `payRecordId`。
**方案**：统一入口，发票创建强制要求关联付款记录。
