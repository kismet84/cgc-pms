## 中期（2026 Q3 - Q4）：二期业务扩展

优先级：🟡 P1

### 财务接口集成（P0-1）

打通 CGC-PMS 与外部财务系统的双向数据流。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 付款推送 | 付款申请审批通过后自动推送财务系统，含重试机制和失败告警 | P0 |
| 付款回写 | 财务系统实际付款后回调 CGC-PMS，含幂等防重和金额差异检测 | P0 |
| 发票同步 | 财务系统开具发票后自动同步到 CGC-PMS，含自动关联付款申请 | P0 |
| 供应商校验 | 新建合作方时校验财务系统供应商主数据（是否停用） | P1 |
| 自动对账 | 付款申请 vs 实际付款 vs 发票金额 三方差异引擎，含预警联动 | P1 |

### 设备租赁管理（P0-2）

填补一期业务主线图中标注但未实现的设备租赁模块。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 设备台账 | 设备主数据 CRUD（编号、类型、规格、归属、状态） | P0 |
| 进退场记录 | 设备进场/退场时间线，自动计算租赁天数 | P0 |
| 台班记录 | 工时×单价自动计算租赁费用，3 级审批闭环 | P0 |
| 成本归集 | `EquipmentLeaseCostStrategy`，sourceType=EQ_LEASE | P0 |
| 维保管理 | 设备维修保养记录，支持计入项目成本 | P1 |
| 租赁付款与结算 | 台班记录作为付款依据，租赁结算汇总 | P1 |

### 劳务管理（P1-3）

填补人工成本归集缺口。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 班组管理 | 劳务班组 CRUD，工人花名册 | P1 |
| 出勤记录 | 出勤天数×日工资自动计算人工费，2 级审批 | P1 |
| 成本归集 | `LaborRecordCostStrategy`，sourceType=LAB_PAYROLL | P1 |
| 工资发放 | 工资发放记录管理 + 付款依据 | P1 |

### BI 分析（P1-4）

在设备/劳务数据完备后补充分析维度。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 合同分析 | 执行率、变更率、履约率、金额分布 | P1 |
| 成本分析 | 实际 vs 目标对比、成本构成饼图、月度趋势、科目下钻 | P1 |
| 付款分析 | 付款进度、逾期付款、资金占用、付款周期 | P1 |
| 供应商分析 | 合作频次、履约评价、风险评级、金额排名 | P2 |
| 结算分析 | 结算周期、结算偏差、完成率 | P2 |
| 成本历史快照 | 每月自动生成项目成本快照，支撑趋势分析 | P1 |

## 远期（2026 Q4 - 2027 Q1）：移动端与外部集成

优先级：🟢 P2

### 审计归档

| 功能 | 说明 |
|------|------|
| 归档目录结构 | 按项目→合同→阶段层级自动目录 |
| 自动归档索引 | 文件上传自动归入对应目录 |
| 归档清单模板 | 按合同类型定义必须归档的资料类型 |
| 资料缺失预警 | 结算提交前自动检查，定期巡检，竣工前强化检查 |
| 审计检索视图 | 合同全链路数据追溯，一站式审计 |

### 移动端

- uni-app 构建，覆盖 iOS / Android / 小程序
- 最小闭环：登录、待办、审批操作、通知中心、材料验收拍照上传
- 扩展：现场签证、出入库扫码、进度填报、质量安全记录

### 外部集成

- 企业微信 / 微信小程序入口
- 与真实财务系统、BI 系统的生产级双向集成
- OA 系统审批流桥接

## 远景（2027+）：平台化

- 多项目大规模数据压测后的容量扩展
- 微服务拆分（按业务域：合同域、成本域、资金域、审批域）
- 低代码审批流配置
- AI 辅助：智能预警、成本预测、合同风险评估

---

## 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 财务系统接口边界不明确 | 付款闭环无法生产化 | 先定义接口契约和回写口径，再开发 |
| 移动端范围蔓延 | 排期失控 | 一期验收前明确移动端不纳入本期，二期最小闭环 |
| 生产运维缺口 | 功能验收后仍无法上线 | 备份/监控/发布/回滚设为上线门禁 |
| 复杂业务链路改动引发回归 | 数据不一致 | 新增需求绑定回归测试和迁移校验 |
| 团队规模限制 | 并行能力有限 | 按优先级串行推进，财务→设备→劳务→BI→归档 |
