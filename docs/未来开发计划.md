# v1.5 未来开发计划

本文件只保留未完成方向和选题边界；可执行任务必须进入 [Ready 队列](backlog/ready-issues.md)。v1.0 已完成记录见 [归档](archive/v1.0/README.md)。

## 当前原则

- 上线结论必须重新验证，不沿用 v1.0 测试数量或旧审计结论。
- 数据库、认证授权、权限、金额和数据一致性改动必须有客观证据。
- AutoPilot 继续使用 `docs/plans/`、`docs/quality/`、`docs/iterations/` 作为 v1.5 活动目录。
- 候选项只有拆成合格 Ready Issue 后才能实施。

## 当前候选方向

| 优先级 | 方向 | 验收边界 |
| --- | --- | --- |
| P0 | CI/CD 与上线门禁复验 | 以当前远端 checks、分支保护和可复现命令为准 |
| P0 | 权限、租户、项目数据边界 | 覆盖真实角色、接口和前端入口，不用超级管理员替代 |
| P1 | 采购补货、现场日报、WBS 延期预警 | 先确认真实业务数据和最小闭环 |
| P1 | 供应商履约档案 | 复用现有采购、验收和评分数据，不扩成通用供应链平台 |
| P2 | 报表、通知与监控深化 | 有明确口径、权限、性能和运维证据后再实施 |
