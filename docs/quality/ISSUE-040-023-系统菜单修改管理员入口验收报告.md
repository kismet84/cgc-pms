## 实现结果

1. 前端 `updateMenu` 精确发送 `PUT /system/menus/{id}`，载荷只包含 parentId、menuName、menuType、path、component、perms、icon、orderNum、status、visible。
2. 页面入口仅 ADMIN/SUPER_ADMIN 可见；普通用户无论无权限或仅持 `system:menu:edit` 均不可见。目标来自完整菜单树，选择后加载当前详情；切换目标时清空旧详情，拒绝时保留目标、表单和错误。
3. 提交成功后并行刷新菜单树、平铺列表和目标详情；任一刷新失败不伪造完整成功，并保留当前交互供用户重试。
4. 后端在同一事务内按菜单 ID 顺序对当前租户整棵菜单树执行 `FOR UPDATE`，再以锁定快照完成目标、父链、子节点和环校验；并发互设父节点最多允许一个事务提交，不会形成双节点环。
5. 更新只复制10个业务字段；客户端不能覆盖 ID、tenantId、审计、逻辑删除、备注或 children。更新仅允许 DIR、MENU、BUTTON；根节点归一为0，非根父节点必须属于当前租户且不能是 BUTTON，拒绝自身或任一后代为父节点以及带子节点目标改为 BUTTON，角色菜单关系无写操作。
