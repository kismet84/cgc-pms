## 6. 问题分类

### 设计一致性问题

1. 列表页在工具栏、表格实现、空态和主从布局上仍需继续对齐历史 UI 基线。
2. 登录页品牌当前可见，但还缺少 token 级复核，暂不宣告完全通过。

### 真实功能风险

1. `440px` 下项目列表移动卡片存在长项目名、日期行局部右溢风险。
2. `440px` 下系统字典表格头和筛选控件存在明显局部右溢风险。

### 环境前置问题

无。后端 health、前端入口、dev-login 均可用；页面访问未回退 `/login`。

### 观察项

1. 可访问性巡检为轻量 DOM 检查，不等同完整 WCAG / axe / 读屏测试。
2. 页面级 `scrollWidth` 未超出视口，但局部元素右溢仍可能影响真实触控和阅读体验。

## 7. 后续建议

1. 拆一条小范围整改 Issue：只处理项目列表与系统字典 `440px` 局部右溢，不扩大为移动端专项重构。
2. 拆一条可访问性小修 Issue：优先补项目列表、系统字典、合同台账、结算列表的 icon-only 按钮 `aria-label/title` 与输入控件标签。
3. 登录页品牌若要收口，应另做源码 token 复核；本轮只保留为观察项。
