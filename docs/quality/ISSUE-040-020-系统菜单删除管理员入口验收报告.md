## 实现结果

1. 页面仅向 ADMIN、SUPER_ADMIN 展示“删除菜单”；普通用户即使单独持有 `system:menu:delete` 也不会被前端单元测试伪装为页面可达，后端权限码授权仍独立有效。
2. 删除目标来自完整 `MenuTreeVO` 树，不依赖只展示有权限码节点的表格行；目录、菜单、按钮及无权限码节点均可选择，并展示目标名称、不可逆提示与子节点/角色引用拒绝说明。
3. `deleteMenu` 固定发送无请求体的 `DELETE /system/menus/{id}`。删除成功后同时刷新菜单树与角色列表，并确认新树不再包含目标，满足后才提示成功并关闭。
4. DELETE 拒绝时保留目标、不刷新成功态并显示后端错误；DELETE 已成功但刷新失败或刷新树仍含目标时，明确提示部分失败，保留交互且不伪造完整成功。
5. 后端生产 Controller、Service、Entity、Mapper 和 schema 保持不变；既有租户、子节点和角色引用 fail-close 语义未放宽。
