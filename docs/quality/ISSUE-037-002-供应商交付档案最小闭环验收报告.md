# ISSUE-037-002 供应商交付档案最小闭环验收报告

## 结论

- 结论：通过。
- 阻塞：无。
- 上线裁决：本报告只证明本地开发环境、目标回归与真实采购经理角色页面通过，不授权生产发布。

## 交付范围

- 采购经理驾驶舱按供应商汇总已到计划交期的已审批、非取消订单，并以已审批验收明细累计实际数量还原订单首次完成日。
- 交付状态明确拆为按期完成、迟交完成、逾期未完成；月份筛选按计划交期归属，未来未完成不进入分母。
- 页面分别展示“迟交完成数”和“逾期未完成数”，保留 `overdueOrderCount` 作为两类合计的兼容字段；交期达成率同时扣减两类未按期订单。
- 采购驾驶舱的指定项目和全项目入口按当前租户与项目 dataScope 过滤；共享项目、财务驾驶舱既有聚合缺口不扩入本 Issue，已进入 Current Focus 作为下一优先治理任务。
- 验收明细保存前拒绝跨订单 `orderItemId`、非正实际数量、负合格数量和合格数量大于实际数量。
- 未新增数据库表、字段、独立供应商门户、综合评级、质量/价格/退货评分或供应商自动选择。

## 风险与回滚

- 当前仅代表采购订单交付表现，不代表综合供应商评级；质量、价格、退货和跨项目综合选择仍不在首版范围。
- `overdueOrderCount` 暂保留为两类未按期订单合计，供旧消费者兼容；新页面只使用两个明确分类字段。
- 共享项目/财务驾驶舱仍存在既有租户全量聚合与项目 dataScope 语义不一致，已进入下一优先治理任务，不影响本 Issue 的采购路径通过结论。
- 回滚可撤回供应商交期计算、三分类 VO/页面字段和验收明细新增校验；不涉及数据库结构或存量数据迁移。
