# 第48条主线 M4 模板治理闭环验收报告

**通过（本地实施与验收）。** M4 已完成受限模板治理、字段契约校验、多模板默认绑定和保存版本预览闭环。未发现未承接的阻塞或非阻塞问题。生产发布未授权，本报告不构成生产上线结论。

## 验收范围

- 付款与结算模板的字段目录、schema、字段清单、占位符和单层循环上下文校验。
- 模板列表、详情、草稿、复制、导入/导出、发布、停用和默认绑定。

- 已保存草稿/发布版本的 PDF 预览，以及默认绑定 CAS 冲突控制。

## 证据

- 服务端字段目录以付款、结算 Provider 的稳定输出模型为唯一字段来源；创建、更新、导入、复制和发布均复用同一校验，未知字段、字段清单不匹配、错误循环上下文和不支持语法明确失败。
- 模板版本发布后不可原地编辑；停用版本不能作为新正式生成默认版本。每个租户和业务类型只有一个默认绑定，更新以锁版本 CAS 比较并写入，陈旧请求返回明确冲突。
- 模板管理 API 将查询、编辑和发布权限拆分；保存版本预览额外要求生成权限与业务对象访问，使用同一 Provider 和渲染器但不创建生成记录或文件归档。
- 后端定向回归通过：`DocumentTemplateServiceIntegrationTest`、`DocumentGenerationServiceTest`、`DocumentControllerAuthorizationTest`、`PaymentSystemTemplateServiceIntegrationTest`、`SettlementSystemTemplateServiceIntegrationTest`。其中覆盖默认绑定陈旧锁冲突和有效默认版本保持。
- 前端模板管理页与路由回归 48 项通过，`vue-tsc --noEmit` 通过。管理员浏览器验收完成字段插入、无效字段校验拒绝、付款/结算模板切换和已保存结算版本 PDF 预览；预览请求返回 PDF 成功，未产生正式生成或归档事实。
