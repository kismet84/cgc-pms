## 实施边界

- 日报 CREATE 的操作审计表达式改为 `#log.id`，复用插入后回填的实体 ID；UPDATE/SUBMIT 保持 `#id`。
- 日报详情在项目访问校验后，按 tenant、`SITE_DAILY_LOG`、日报 ID 查询统一审计记录。
- 前端只读展示动作、用户 ID、成功/失败和时间。
- 未返回 IP、请求路径、错误码、耗时，未修改统一审计表或异步写入机制。

## 风险与回滚

- 非阻塞：审计为异步持久化，写操作完成后的极短时间内详情可能暂未出现最新记录；本轮不改同步机制。
- 非阻塞：历史仅展示动作级信息，不提供字段级前后值或用户名称快照。
- 回滚：移除 CREATE 表达式、详情 auditTrail 与前端展示；无数据迁移。
