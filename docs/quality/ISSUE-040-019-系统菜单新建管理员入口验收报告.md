## 实现结果

1. 权限清单页与 `/system` 父路由的 admin-only 边界保持一致，只向 ADMIN、SUPER_ADMIN 展示“新建菜单”，并复用当前菜单树选择根节点或非 BUTTON 父节点；普通用户即使单独持有 `system:menu:add` 也不被单元测试伪装为页面可达。
2. `createMenu` 固定调用 `POST /system/menus`，载荷类型不包含 `id`、`tenantId`、审计字段或 `children`；成功关闭表单并重新获取菜单树，失败保留输入且不刷新成功态。
3. `SysMenuService.create` 将空父节点归一为 `parentId=0`，菜单类型仅接受 DIR/MENU/BUTTON；非根父节点必须在当前租户可见且不能是 BUTTON。
4. 客户端提供的租户 ID 继续被当前 `UserContext` 覆盖；控制器授权注解未修改，前端隐藏不替代后端门禁。
