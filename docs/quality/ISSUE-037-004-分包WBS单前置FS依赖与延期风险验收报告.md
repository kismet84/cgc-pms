## 失败分类与修正

1. 首轮后端 500 为 H2 未加载新字段，归类为测试 migration 镜像缺失；补充 `migration-h2/V140` 后恢复。
2. 列表首轮 NPE 为 `Map.of().get(null)`，归类为真实代码缺陷；空前置 ID 改为直接跳过后通过复验。
3. 原租户边界用例允许租户 A 引用租户 B 项目，与新 fail-close 规则冲突；修正为显式拒绝跨租户项目，并保留客户端 tenantId 注入无效断言。
4. 运行态写请求首轮 403 为缺少 CSRF 双提交头；补齐后进入业务校验。固定项目 `10001` 不属于当前身份，改为只读查询当前可见项目后闭环通过。

## 剩余风险

- 前置候选当前按单项目最多读取 1000 条任务；大项目的候选分页/搜索后续再单独设计，不在本轮扩展。
- 本轮没有浏览器 DOM 控制运行时，页面交互由 9/9 组件单测、类型检查、真实 API 和 health gate 共同裁决；不声称完成真实浏览器点击验收。
- 多前置、多依赖类型、依赖连线、自动改期、关键路径和资源平衡仍是明确非目标。
