## 实施范围

- 新增共享预警、通知、报表目录契约和 V2 薄服务，全部复用既有端点。
- 具备 `alert:view` 的驾驶舱直接读取权威预警列表；点击预警行打开毛玻璃悬浮窗，首次阅读、接单和单项状态更新要求 `alert:edit`，规则评估要求 `alert:evaluate`。
- 独立预警中心页面与工作区标签已取消；历史 `/alert` 深链重定向到 `/dashboard#risk-list`。无预警权限角色保留既有派生风险只读列表，不发起权威预警请求。
- 既有批量服务与契约保留供后续复用，但不再暴露一套与驾驶舱重复的批量处置页面。
- 公共壳通知摘要只在同时具备 `alert:view` 与 `notification:view` 时读取有限通知列表和真实未读数；标记已读继续要求 `notification:edit`。未接入 SSE，未伪造角标或通知事实。
- 通知项标题固定 12px，正文与时间固定 11px。
- 公共壳普通内容区统一使用四边 10px 内边距，不启用视口铺满；预警和报表页面移除重复外边距，审批工作台继续由其共享根容器保持 10px。
- 报表目录以服务端返回的当前用户可见集合为权威数据；仅 `page`、`available` 且命中已注册内部路由的目标提供页面入口，`api_only`、未知和不可用目标明确关闭导航。
- 驾驶舱预警、通知和报表页面保持公共壳项目/报告期上下文，不复制 Legacy 页面、状态或请求层。
- 后端角色域解析覆盖项目、商务、成本、采购、生产、总工、财务和管理员；纯 `QUALITY_SAFETY` 域不再因空旧规则映射生成异常查询。七个业务角色补齐 `alert:edit`，`alert:evaluate` 仍仅限财务与管理员。
