## 2. 事实与最小方案

- CodeGraph 首查确认5个 Controller 方法、Service 租户过滤、`DRAFT → POSTED → REVERSED` 状态门禁及既有测试影响面。
- `codebase-memory-mcp` 的补充查询目的为跨后端路由、权限、前端页面和测试的关系核验；首次项目名 `cgc-pms` 未命中，按工具提示改用已索引项目 `D-projects-test-cgc-pms` 后命中5条路由、Controller/Service和现有测试，结论与 CodeGraph 交叉一致。首次失败归类为 `tool_config`，未伪造证据。
- 明确检索生产代码后，没有任何类实现 `EntryGenerationStrategy`。因此不猜测来源单据、借贷科目和会计规则，也不以“成功返回 null”冒充生成成功。
- 前端新增一个会计凭证页面，复用既有 API 响应、分页、用户权限和 Ant Design 组件，不新增表或会计平台抽象。

## 3. 权限、租户与状态边界

- V148（MySQL/H2 同号）新增菜单960 `accounting:query` 和按钮961 `accounting:edit`，仅对实际存在的 SUPER_ADMIN、ADMIN、FINANCE 角色授权；当前本地 MySQL 角色集中不存在 ADMIN，实测绑定为 SUPER_ADMIN、FINANCE。
- `accounting:add` 不进入菜单和角色授权；页面明确提示生成入口待会计规则确认。
- FINANCE 持查询权限可访问列表，无权限返回403；编辑按钮按权限与记录状态双重收敛。
- 后端详情、过账和冲销继续通过 `requireExisting` 校验当前租户，跨租户或不存在记录统一按 `ENTRY_NOT_FOUND` fail-close。
- 不支持的生成来源返回400 `ENTRY_SOURCE_UNSUPPORTED`，不再返回成功空值。
