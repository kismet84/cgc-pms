## 4. 缺口清单

### GAP-001 `/approval/cc` 抄送详情直出可执行动作

- 页面 / 入口：审批中心 `/approval/cc`
- 现象：运行中合同样本详情仍直出并启用 `同意 / 驳回 / 撤回`，不符合抄送页只读、不误导动作能力口径。
- 证据：小包 1 测试验收员确认该问题为阻塞；小包 2 前端确认 `frontend-admin/src/pages/approval/todo.vue` 内嵌详情动作区只判断 `availableActions.length > 0 && isDetailRunning`，未按 `activeTab === 'cc'` 屏蔽；小包 3 安全确认这不是已证实的后端动作执行越权，而是前端误导展示层问题。
- 建议归因：前端显示口径 / 页面入口语义。
- 级别：阻塞。
- 处置建议：立即修。只在前端内嵌详情层按页签限定动作展示：`todo / mine` 可按冻结口径展示当前入口允许动作；`cc / done` 保持只读，不直出同意、驳回、撤回等动作。
- 收益：消除抄送入口对用户和验收的动作能力误导。
- 成本：低，局部改 `approval/todo.vue` 的内嵌详情动作渲染判断。
- 风险：需复验 `todo / mine` 入口不要被误伤；不需要拆后端权限修复包。

### GAP-002 `/approval/todo` 业务类型裸露 `CONTRACT`

- 页面 / 入口：审批中心 `/approval/todo`
- 现象：`项目经理待办审批演示` 的业务类型仍显示原始码值 `CONTRACT`。
- 证据：小包 1 测试验收员确认该问题为阻塞；小包 2 产品确认该问题属于状态码 / 业务类型中文化扫尾范围且阻塞；小包 2 前端确认 `frontend-admin/src/pages/approval/todo.vue` 的 `businessTypeMap` 缺少 `CONTRACT`，且 `businessTypeMap[row.businessType] || row.businessType` 会裸露未知码值。
- 建议归因：前端显示映射缺口。
- 级别：阻塞。
- 处置建议：立即修。局部补 `CONTRACT: '合同审批'`，并将未知业务类型兜底改为不裸露内部码值的中文文案，例如 `未知业务类型` 或产品确认后的通用业务文案。
- 收益：消除用户可见业务类型码，满足小包 2 中文化扫尾目标。
- 成本：低，局部映射与兜底调整。
- 风险：未知类型兜底文案需要产品确认；不应扩展为后端枚举重构或全站国际化。

### GAP-003 采购经理 `待验收入库` 未来日期显示 `今日`

- 页面 / 入口：驾驶舱 `/dashboard` 采购经理页。
- 现象：`待验收入库` 中未来日期 `2026-07-03`、`2026-07-05` 仍显示 `今日`。
- 证据：小包 1 测试验收员确认该问题为阻塞；小包 2 产品确认该问题属于摘要 / 展示口径阻塞；小包 2 前端确认 `DashboardPurchaseView.vue` 的 `pendingInfo(record)` 只消费 `record.pendingDays`，后端 `DashboardService.pendingDays(LocalDate)` 对未来日期也返回 0，导致未来待验显示 `今日`；小包 3 后端确认 `pendingDays` 当前真实语义是“已等待 / 待验天数”，不是未来“剩余天数”。
- 建议归因：前端展示口径与后端字段语义混用。
- 级别：阻塞。
- 处置建议：立即修。短期在采购经理前端按 `record.date` 兜底计算并展示 `剩余X天 / 今日 / 逾期X天 / -`，不要把 `pendingDays=0` 直接解释为今日到期。长期将 `pendingDays` 语义单独清晰化，必要时另建 `remainingDays` 或明确到期日期字段。
- 收益：恢复驾驶舱日期语义可信度，避免未来事项被误判为今日。
- 成本：低到中，前端局部计算与回归验证。
- 风险：长期字段契约仍需后续治理；当前不建议为此立即拆后端契约改造包。

### GAP-004 生产经理 `近期验收` 未来日期显示 `0天`

- 页面 / 入口：驾驶舱 `/dashboard` 生产经理页。
- 现象：未来日期显示 `0天`。
- 证据：小包 1 标记为需要确认；小包 2 产品明确当前不写死为 fail。
- 建议归因：需要确认，疑似前端时效文案与后端天数字段语义未对齐。
- 级别：非阻塞，需要确认。
- 处置建议：后续增强。待 GAP-003 修复并复验后，再判断生产经理是否复用相同日期展示规则；不要在当前最小修复包中扩大到生产经理，除非主线程明确裁决。
- 收益：避免把未确认问题写成正式阻塞。
- 成本：低，后续可通过一次运行态复验确认。
- 风险：若后续证明确为同类问题，应补入下一轮展示口径收敛。

### GAP-005 审批详情弹窗未见显式业务详情入口

- 页面 / 入口：合同审批、采购申请审批、分包计量审批详情弹窗。
- 现象：抽样详情接口 200，但弹窗未见显式业务详情入口。
- 证据：小包 1 标记为需要确认；小包 2 产品明确当前不写死为 fail。
- 建议归因：需要确认，可能是产品入口口径或页面布局问题。
- 级别：非阻塞，需要确认。
- 处置建议：后续增强 / 暂不处理。当前只记录为验收观察项，不纳入立即修复包；若后续产品裁决“业务详情入口必须显式可达”，再拆独立小包处理。
- 收益：保留证据线索，不扩大当前阻塞修复范围。
- 成本：待确认。
- 风险：若业务详情入口是用户主流程必需项，后续可能升级为阻塞。

## 5. 下一阶段最小修复包范围

建议下一阶段只拆一个最小前端修复包，范围限定为：

1. `frontend-admin/src/pages/approval/todo.vue`
   - 补 `CONTRACT: '合同审批'`。
   - 改未知业务类型兜底，避免裸露内部码值。
   - 内嵌详情动作按页签限定：`cc / done` 只读，`todo / mine` 按冻结口径展示。
2. `frontend-admin/src/pages/dashboard/components/DashboardPurchaseView.vue`
   - `待验收入库` 时效文案按 `record.date` 兜底计算 `剩余X天 / 今日 / 逾期X天 / -`。
   - 不改变后端 `pendingDays` 当前“已等待 / 待验天数”语义。
3. 验收建议
   - 复验 `/approval/todo` 不再显示 `CONTRACT`。
   - 复验 `/approval/cc` 不再直出同意、驳回、撤回。
   - 复验采购经理未来待验日期不再显示 `今日`。

明确禁止在该最小修复包中扩大到：

- 后端权限修复包。
- 后端枚举重构。
- 审批引擎、权限链路或业务状态机改造。
- UI 大改。
- 付款申请。
- 新业务模块。
- 移动端。
- 搜索 / 高级筛选。
- 全站日期、金额、摘要、tooltip 统一治理。

## 6. 后端与安全裁决

当前不建议拆后端权限修复包。依据是小包 3 安全与后端结论均未确认存在明显后端越权：查询边界、详情边界、动作执行边界未确认存在明显漏洞；`availableActions` 是用户对实例的真实可执行动作集合，不感知当前页面入口；动作执行接口仍有租户、状态、审批人 / 发起人等校验。

`pendingDays` 长期契约建议后续单独清晰化，但不阻塞当前最小前端修复。当前后端口径是“已等待 / 待验天数”，若页面要表达未来剩余天数，应在前端使用日期兜底或后续单独设计 `remainingDays` / 到期日期字段，不应在本轮强行改动后端语义。

## 7. 剩余风险

1. `todo / done / mine` 轻量回归通过不等于重新执行全量闭环矩阵。
2. GAP-004 与 GAP-005 仍为需要确认项，当前不能写成已确认阻塞，也不能直接放弃跟踪。
3. 后续新增业务类型、状态码、动作码时，前端映射与兜底仍需同步维护。
4. 当前报告基于各小包已回传结论归档，未重新跑测试、未重启服务、未读取运行态页面。
