# ISSUE-040-046 库存阈值与出库真实并发防覆盖回归验收报告

**Goal:** 以可重复的 local H2 真实事务并发，证明安全库存阈值更新与出库争用同一库存版本时，乐观锁和出库重试不会覆盖任一成功字段，且库存余额与流水一致。

**Architecture:** 仅新增独立 Spring 集成测试，复用 `MatStockService`、MyBatis、`@Version` 和既有出库重试；Mockito spy 只建立确定性首次更新屏障，真实 SQL 由同一事务内的 `SqlSessionTemplate` Mapper 执行。禁止修改生产逻辑、数据模型、权限、前端及控制面，不扩展为压力测试或分布式锁平台。
