# 全量审计：安全

## 结论

**代码安全边界基本通过；生产安全门禁未通过。评分 70/100。** 未发现当前源码中可确认的 P0/P1 远程利用漏洞；生产凭据与存量文件门禁仍是硬阻塞。

## 已确认控制

- `SecurityConfig` 默认 CSRF 开启，Cookie CSRF 仓库；仅登录类路径和健康检查白名单，其他请求要求认证。
- JWT 过滤器校验 token 类型、黑名单，并在请求结束清理用户/租户上下文；黑名单能力启用但不可用时失败关闭。
- CORS 拒绝空列表与通配来源，同时允许凭据；生产默认仅配置明确 HTTPS 域名。
- 生产禁用 dev login、Swagger/API docs；Cookie Secure；JWT 15 分钟默认过期。
- 76 个 Controller 中 74 个有方法级 `@PreAuthorize`；其余为 profile 限制的开发登录与由过滤链认证的 Prometheus 端点。
- 日志配置对 token、authorization、cookie、电话、证件、银行卡等字段脱敏。
- 文件生产配置默认启用 ClamAV；文档生成 Provider 对付款状态、租户、项目和附件授权做服务端裁决。

## 阻塞

- `SEC-001`（P0）：目标环境新凭据生效、旧凭据失效、双人脱敏复核未执行。
- `FILE-001`（P0）：生产存量对象逐对象真实病毒复扫未执行。

## 判定边界

本报告不把“缺目标环境证据”伪装成源码漏洞；但在发布裁决中，两项均按正式 `RELEASE_GATE` 阻断生产上线。
