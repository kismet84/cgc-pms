# ISSUE-040-017 知识图谱当前问题结构化查询验收报告

## 1. 裁决

结论：**通过本地验收。当前问题查询由文档全文检索改为结构化 Issue 查询，原3分29秒查询路线的系统根因已关闭。**

阻塞：本地无阻塞。新工具独立stdio MCP烟测通过；Codex已启动的旧MCP进程需要运行态重载后才会在既有任务工具清单中出现。

是否可上线：该组件仅为本地工程治理工具，不构成CGC-PMS生产发布。

## 2. 根因

- 原图谱只有 Artifact、ArtifactVersion、Section 等文档节点，没有 Issue 节点。
- “查询所有既有问题”使用3个宽泛全文检索，返回约91303 tokens，并召回同一文档的多个历史版本。
- 图查询本身约0.2秒，主要耗时来自大结果解析、历史去重和重复读取正式台账。
- 当前工作区与Git HEAD可能不同，仅靠历史全文索引不能表达唯一当前状态。

## 3. 最小实现

- 新增 `docs/backlog/current-issues.json`，作为机器可读当前问题唯一快照；每条包含状态、分类、优先级、阻塞性、父项、验收标准和来源。
- 采集器使用Zod严格校验重复ID、未知父项和字段边界，确定性生成 `Issue`、`PART_OF`、`DEFINED_IN`、`SUPPORTED_BY`、`SEEN_IN`；不使用LLM抽取。
- 新增 `kg_list_issues`：默认 `view=summary`，支持状态、分类、优先级、父项、阻塞性和关键字过滤；明细最多200条。
- schema增加Issue唯一约束及当前状态、分类、优先级索引；采集运行记录 `issuesIndexed/issuesDeactivated`。
- 原有私有目录禁入、历史隔离、来源证据、敏感值扫描和只读Cypher边界保持不变。
