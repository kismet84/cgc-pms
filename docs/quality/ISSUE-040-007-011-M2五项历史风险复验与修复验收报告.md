## 3. A-08 文件病毒扫描

- 新增 ClamAV INSTREAM 扫描客户端、连接/读取超时和响应长度上限；dev/prod Compose 增加内部 ClamAV 服务，不暴露宿主机端口。
- V147 为文件记录增加 `virus_scan_status/detail/scanned_at`，H2 与 MySQL migration 同步。
- 上传链路在写 MinIO 前扫描：`CLEAN` 才保存；`INFECTED` 返回 `FILE_VIRUS_DETECTED`；扫描器不可用返回 `FILE_VIRUS_SCAN_UNAVAILABLE`。
- 下载和临时 URL 仅允许 `CLEAN`；历史 `NOT_SCANNED/NOT_CONFIGURED/FAILED` 均 fail-close。
- 真实运行态：干净 PDF 返回 `CLEAN` 并可取下载 URL；标准 EICAR 样本被 `Eicar-Test-Signature` 命中且未落库；停止 ClamAV 后上传按预期失败。
- 当前本地库活动文件为 0，无本地存量回填；生产发布前必须对存量对象执行受控复扫，不能直接把旧记录标记为 `CLEAN`。

## 4. V-05 外部依赖与可观测性

- MinIO 停机：上传返回 `FILE_STORAGE_UNAVAILABLE`，恢复后健康。
- ClamAV 停机：上传返回 `FILE_VIRUS_SCAN_UNAVAILABLE`，恢复后健康。
- Redis 复验发现真实缺陷：`TokenBlacklistService` 使用启动顺序敏感的 `@ConditionalOnBean`，当前 dev 容器即使 Redis 正常也未注册服务，令牌黑名单长期失效。
- 修复后服务始终注册；dev/prod 默认启用，local/test 按 profile 与显式配置禁用。启用时 Redis bean 缺失、连接异常均 fail-close，禁用状态才允许本地无 Redis 运行。
- 新 jar 运行态复验：Redis 正常时 `/api/auth/userinfo` 为 200；停机后连续两次为 401，Actuator health 同期为 503；恢复后连续观察 180 秒保持 200/healthy。
- 三套 Compose 为 Redis 容器显式注入健康检查所需密码，并改用 `REDISCLI_AUTH` 验证 `PONG`；修复了旧 healthcheck 变量未注入导致的假健康风险。
- Prometheus 鉴权端点返回 200，HTTP 访问日志含脱敏 `traceId/requestId/userId/tenantId/status/duration`；运行态原始日志不入库。
