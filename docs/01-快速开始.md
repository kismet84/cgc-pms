# 01 快速开始

## 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 21 | 后端编译运行 |
| Node.js | 20+ | 前端构建 |
| pnpm | 9+ | 前端包管理 |
| Docker | 24+ | 基础设施容器 |

## 一键启动（推荐）

在项目根目录执行：

```bash
scripts/start-dev.bat
```

自动检查后端 JAR（首次自动构建），通过 Docker Compose 启动全部 5 个容器：MySQL、Redis、MinIO、Backend、Frontend。

> 宿主机无需安装 Java、Node 或 pnpm。

## 手动启动

### 1. 基础设施

```bash
cd deploy
cp .env.example .env
docker compose up -d
```

启动 MySQL（端口 3306）、Redis（端口 6379）、MinIO（API 9000，控制台 9001）。

### 2. 后端

```bash
cd backend
# MySQL 模式（默认）
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev

# H2 内存库模式（无需 MySQL）
./mvnw spring-boot:run -Dspring-boot.run.profiles=local
```

### 3. 前端

```bash
cd frontend-admin
pnpm install
pnpm dev
```

## 访问入口

| 服务 | 地址 |
|------|------|
| 前端 | `http://localhost:5173` |
| 后端 API | `http://localhost:8080/api` |
| Swagger UI | `http://localhost:8080/api/swagger-ui.html` |
| MinIO 控制台 | `http://localhost:9001` |

默认账号：`admin` / `admin123`

## Docker 开发环境

使用 `deploy/docker-compose.dev.yml` 全容器化运行（后端和前端也在容器内）：

```bash
scripts/start-dev.bat
```

容器管理：

```bash
cd deploy
docker compose -f docker-compose.dev.yml logs -f      # 查看日志
docker compose -f docker-compose.dev.yml restart       # 重启全部
docker compose -f docker-compose.dev.yml restart frontend  # 仅重启前端
docker compose -f docker-compose.dev.yml down          # 停止
```

## 常用命令

### 后端

```bash
cd backend
./mvnw clean package -DskipTests                           # 构建 JAR
./mvnw test -Djasypt.encryptor.password=dev-jasypt-key    # 运行全部测试
./mvnw -Dtest=ClassName test                               # 单个测试类
```

### 前端

```bash
cd frontend-admin
pnpm build              # 类型检查 + 构建
pnpm type-check         # 仅类型检查
pnpm lint               # ESLint 修复
pnpm format             # Prettier 格式化
pnpm test:unit          # 单元测试
pnpm exec playwright test   # E2E 测试
```

## 故障处理

### Docker 容器启动失败

```bash
scripts/restart-docker.bat    # Windows
scripts/restart-docker.sh     # Git Bash
```

### 后端 404（`No static resource`）

访问路径漏了 `/api` 上下文。正确示例：`http://localhost:8080/api/workflow/templates`

### Flyway checksum 不匹配

不要修改已执行的迁移脚本。若需修复，参考 `README.md` 中的 Flyway 修复步骤。

### 接口正常但页面报错

- 刷新页面
- 重新登录
- 检查浏览器 Network 确认请求到了正确的端口
