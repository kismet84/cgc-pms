# v1.0 Workspace Archive Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Safely archive v1.0 project history and private workspace data, refresh current v1.5 documentation, and leave a clean, working `develop/1.5` workspace.

**Architecture:** Use Git-native moves for tracked documents and native PowerShell for private local copies, inventories, and SHA-256 verification. Keep current v1.5 contracts at `docs/plans/`, `docs/quality/`, `docs/iterations/`, and `docs/backlog/`; move only historical artifacts beneath `docs/archive/v1.0/`. Repair references from an explicit old-to-new path map and validate with repository scans instead of adding a link-check dependency.

**Tech Stack:** Git, PowerShell 7, Markdown, JSON

## Global Constraints

- Work only in `D:\projects-test\cgc-pms` on `develop/1.5`.
- Do not modify business source, database migrations, production configuration, or Git history.
- Do not push; commit locally after each independently recoverable tracked-document phase.
- Do not stage `archive/v1.0/private/**`; preserve private data until file-count, byte-count, and SHA-256 verification succeeds.
- Keep generic v1.5 contracts `docs/plans/`, `docs/quality/`, and `docs/iterations/` unchanged.
- Refresh standards and manuals only for paths, v1.5 applicability, and statements proven stale by current repository evidence.
- Preview every cleanup; do not delete repository-external files.
