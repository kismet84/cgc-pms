# v1.0 工作区封存与 v1.5 目录重整设计

**Goal:** 将 v1.0 及更早的计划、质量报告、迭代记录、backlog 快照、日志和已授权私有目录完整封存，使 `develop/1.5` 工作区只保留当前源码、有效规范、当前任务事实和必要运行入口。

**Architecture:** 正式历史文档移动到 Git 跟踪的 `docs/archive/v1.0/`；当前规范统一移动到 `docs/standards/`；八个已授权私有目录及历史运行日志移动到 Git 忽略的 `archive/v1.0/private/`。可重建构建产物不封存，确认后删除。移动过程保持目录结构、生成清单与哈希、修复全仓链接，并以单独提交完成。

## 1. 已确认决策

- 当前有效规范目录：`docs/standards/`。
- 正式历史目录：`docs/archive/v1.0/`，纳入 Git。
- 私有归档目录：`archive/v1.0/private/`，不得进入 Git。
- 用户已对以下目录解除本次封存禁止：`.omc/`、`.omo/`、`.opencode/`、`.claude/`、`.mimocode/`、`graphify-out/`、`.sisyphus/`、`.archive/`。
- `graphify-out/` 当前不存在，只记录在清单中，不创建空归档内容。
- 不修改业务源码、数据库迁移、生产配置或 Git 历史。

## 3. 分类规则

### 保留在当前工作区

- `AGENTS.md`、`AGENTS.override.md`、`README.md`、`CHANGELOG.md`。
- 源码、部署配置、工作流、当前脚本和插件。
- 当前有效的 00～13 规范，移动到 `docs/standards/`。
- 用户手册、培训、runbook、prompt 及其当前索引。
- v1.5 的 current-focus、Ready、Blocked、epics 和长期计划入口。

### Git 跟踪封存

- 第 3～36 条主线及其他 v1.0 历史计划书。
- v1.0 及更早的质量、审计、验收、收口报告。
- 2026-07-11 及更早的迭代计划和报告。
- v1.0 封版时的 backlog 完整快照；当前 backlog 文件另行初始化为 v1.5 事实源。
- 已失效或被当前规范替代的正式规范。

### 本地私有封存

- 八个已授权私有目录。
- `.codex-autopilot/` 中旧 runs、events、state、closeout ledger 和执行日志；保留当前控制 flag 与测试数据 reset marker。
- `.agent-runtime/` 及其他历史会话日志。
- 私有文件仅移动，不读取其业务内容；清单只记录相对路径、字节数和 SHA-256，不提交 Git。

### 删除而不封存

- `backend/target/`。
- `frontend-admin/dist/`、`frontend-admin/coverage/`。
- `playwright-report/`、`test-results/`、临时缓存、`*.tmp`。
- 删除前必须预览并确认路径位于仓库内。

## 4. 安全与一致性

1. 开始前核对 branch/status、`v1.0.0` tag 和远端分支。
2. 先复制/移动到归档临时目录，生成文件数、总字节数和 SHA-256 清单；校验一致后再移除原目录。
3. `archive/v1.0/private/` 在任何 `git add` 前写入 `.gitignore`，并用 `git check-ignore` 验证。
4. 显式白名单暂存正式文档和链接修复，不使用无审查的 `git add -A`。
5. 不展开或提交私有文件内容；Git staged 清单不得出现 `archive/v1.0/private/**`。
6. 移动后执行全仓 Markdown 链接检查、关键路径检查、`git diff --check` 和文档索引检查。

## 5. 回滚

- 正式文档移动在一个独立提交中完成，可用普通 revert 恢复路径。
- 私有目录在清单校验前不删除源；若中途失败，保留源目录并删除未完成的目标副本。
- v1.0 原始状态始终可由 annotated tag `v1.0.0` 恢复。

## 6. 验收标准

- 根目录不再存在八个私有目录；其文件数、字节数和哈希与私有归档清单一致。
- `docs/` 根只保留索引和分类目录；00～13 规范全部位于 `docs/standards/`。
- `docs/plans`、`docs/quality`、`docs/iterations` 只保留 v1.5 当前入口或 README。
- v1.0 历史文件全部可从 `docs/archive/v1.0/` 或私有归档定位。
- 全仓无指向旧规范路径的已知死链。
- `git status` 只显示预期的正式归档与索引变更，私有归档不进入 Git。
- 不影响 backend/frontend 构建入口、AutoPilot 控制入口及 v1.5 分支版本号。
