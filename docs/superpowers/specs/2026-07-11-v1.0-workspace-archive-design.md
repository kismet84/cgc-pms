# v1.0 工作区封存与 v1.5 目录重整设计

**Goal:** 将 v1.0 及更早的计划、质量报告、迭代记录、backlog 快照、日志和已授权私有目录完整封存；同步修复移动产生的引用，并受控更新现行规范与手册，使 `develop/1.5` 工作区只保留当前源码、有效文档、当前任务事实和必要运行入口。

**Architecture:** 正式历史文档移动到 Git 跟踪的 `docs/archive/v1.0/`；当前规范统一移动到 `docs/standards/`；八个已授权私有目录及历史运行日志移动到 Git 忽略的 `archive/v1.0/private/`。可重建构建产物不封存，确认后删除。移动前建立旧路径到新路径的映射，移动后修复精确历史引用但保留 `docs/plans/`、`docs/quality/`、`docs/iterations/` 的 v1.5 活动目录契约；现行规范和手册只更新路径、版本口径及经当前代码证实的过时说明，不借机改写业务规则。

## 1. 已确认决策

- 当前有效规范目录：`docs/standards/`。
- 正式历史目录：`docs/archive/v1.0/`，纳入 Git。
- 私有归档目录：`archive/v1.0/private/`，不得进入 Git。
- 现行规范与手册采用“受控更新”：修复导航和引用，统一 v1.5 口径，只纠正有代码或配置证据的过时内容。
- 历史归档正文保持原貌；仅允许为保证可定位性更新链接或增加归档说明。
- 用户已对以下目录解除本次封存禁止：`.omc/`、`.omo/`、`.opencode/`、`.claude/`、`.mimocode/`、`graphify-out/`、`.sisyphus/`、`.archive/`。
- `graphify-out/` 当前不存在，只记录在清单中，不创建空归档内容。
- 不修改业务源码、数据库迁移、生产配置或 Git 历史。

## 3. 分类规则

### 保留在当前工作区

- `AGENTS.md`、`AGENTS.override.md`、`README.md`、`CHANGELOG.md`。
- 源码、部署配置、工作流、当前脚本和插件。
- 当前有效的 00～13 规范，移动到 `docs/standards/`。
- 用户手册、培训、runbook、prompt 及其当前索引；其中规范、手册和导航按第 4 节更新。
- v1.5 的 current-focus、Ready、Blocked、epics 和长期计划入口。

### Git 跟踪封存

- 第 3～36 条主线及其他 v1.0 历史计划书。
- v1.0 及更早的质量、审计、验收、收口报告。
- 2026-07-11 及更早的迭代计划和报告。
- v1.0 封版时的 backlog 完整快照；当前 backlog 文件另行初始化为 v1.5 事实源。
- 已失效或被当前规范替代的正式规范。

### 本地私有封存

- 八个已授权私有目录。
- `.codex-autopilot/` 中旧 runs、events、state、closeout ledger 和执行日志；保留当前控制 flag 与测试数据 reset marker。
- `.agent-runtime/` 及其他历史会话日志。
- 私有文件仅移动，不读取其业务内容；清单只记录相对路径、字节数和 SHA-256，不提交 Git。

### 删除而不封存

- `backend/target/`。
- `frontend-admin/dist/`、`frontend-admin/coverage/`。
- `playwright-report/`、`test-results/`、临时缓存、`*.tmp`。
- 删除前必须预览并确认路径位于仓库内。

## 4. 引用迁移与现行文档更新

### 4.1 引用迁移规则

- 先生成旧路径到新路径的清单，再移动文件；不做无差别全局替换。
- `docs/00-*.md`～`docs/13-*.md` 的活动引用统一改为 `docs/standards/...`，并移除仓库内文档中的本机绝对路径。
- 指向某份 v1.0 计划、质量报告、迭代报告或 backlog 文件的精确引用改到 `docs/archive/v1.0/...`。
- 泛指当前产物目录的 `docs/plans/`、`docs/quality/`、`docs/iterations/` 保持不变；AutoPilot runner、测试脚本和项目规则继续使用这些活动目录。
- 活动文档不直接链接 Git 忽略的 `archive/v1.0/private/`；需要说明私有历史时，只链接 `docs/archive/v1.0/README.md` 中的归档清单。
- 历史报告中的事实陈述不因目录移动而改写，例如“当时未读取 `.archive/`”应原样保留。

### 4.2 必须同步更新的活动入口

- `AGENTS.md`、`AGENTS.override.md`、根 `README.md`、`docs/README.md`。
- `docs/manuals/README.md`、`docs/manuals/admin-manual.md`、`docs/manuals/end-user-manual.md`。
- 00～13 规范之间的相互引用。
- `scripts/backup-scheduler.sh`、`scripts/check-flyway-immutability.bat`、`scripts/check-flyway-immutability.sh` 中的规范路径。
- 当前 backlog 中指向 v1.0 计划、质量报告和迭代报告的精确链接。
- `.gitignore`、`scripts/codex-autopilot/autopilot-exec-issue.ps1` 和 AutoPilot owner boundary：新增私有归档禁止路径，同时保留原私有目录保护规则。

### 4.3 规范与手册的受控更新范围

- 为现行规范、用户手册和管理员手册补充或校正 v1.5 适用口径、当前入口和现行/历史边界。
- 对照当前代码、配置和运行入口，只修明确过时的功能、角色权限、审批、部署、备份、恢复和操作说明。
- 不新增未经实现的能力，不因文档整理改变权限、接口、数据库或业务流程，不做纯风格重写。
- 培训材料和 runbook 只修受规范或手册路径变化影响的入口；没有证据表明内容过时时不扩写。

### 4.4 私有目录的活动依赖

- `skills-lock.json` 当前引用 `.claude/skills/release-skills/SKILL.md`；移动 `.claude/` 前必须确认当前技能来源并刷新锁文件，1.5 不得依赖 Git 忽略的私有归档路径。
- 若任何公开脚本、配置或文档仍把待封存私有目录作为运行依赖，必须先移除或迁移该依赖；仅作为禁止清单或历史事实的文本不视为失效引用。

## 5. 安全与一致性

1. 开始前核对 branch/status、`v1.0.0` tag 和远端分支。
2. 先复制/移动到归档临时目录，生成文件数、总字节数和 SHA-256 清单；校验一致后再移除原目录。
3. `archive/v1.0/private/` 在任何 `git add` 前写入 `.gitignore`，并用 `git check-ignore` 验证。
4. 显式白名单暂存正式文档和链接修复，不使用无审查的 `git add -A`。
5. 不展开或提交私有文件内容；Git staged 清单不得出现 `archive/v1.0/private/**`。
6. 移动后执行旧路径残留扫描、Markdown 相对链接检查、关键路径检查、`git diff --check` 和文档索引检查。
7. `skills-lock.json` 不得引用已封存的私有目录；AutoPilot 活动目录和禁止路径必须通过静态检查。

## 6. 回滚

- 正式文档移动在一个独立提交中完成，可用普通 revert 恢复路径。
- 私有目录在清单校验前不删除源；若中途失败，保留源目录并删除未完成的目标副本。
- v1.0 原始状态始终可由 annotated tag `v1.0.0` 恢复。

## 7. 验收标准

- 根目录不再存在八个私有目录；其文件数、字节数和哈希与私有归档清单一致。
- `docs/` 根只保留索引和分类目录；00～13 规范全部位于 `docs/standards/`。
- `docs/plans`、`docs/quality`、`docs/iterations` 只保留 v1.5 当前入口或 README。
- v1.0 历史文件全部可从 `docs/archive/v1.0/` 或私有归档定位。
- 活动区无指向旧规范路径或已移动历史文件的已知死链，Markdown 相对链接目标存在。
- `AGENTS.md`、README、规范、手册和相关脚本均使用新的规范路径；仓库文档不含指向规范文件的本机绝对路径。
- 规范和手册已标明 v1.5 现行口径，所有内容修订都有当前代码、配置或运行入口证据，未写入未实现能力。
- `skills-lock.json` 不引用已封存的 `.claude/`；AutoPilot 继续使用活动的 plans、quality、iterations 目录，并拒绝触碰私有归档。
- `git status` 只显示预期的正式归档与索引变更，私有归档不进入 Git。
- 不影响 backend/frontend 构建入口、AutoPilot 控制入口及 v1.5 分支版本号。
