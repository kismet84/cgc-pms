# 项目台账页面生产化增强 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将项目台账页从可用列表页增强到可生产验收的台账页面，修正分页、金额、筛选、移动端和真实浏览器验收问题。

**Architecture:** 最小可行方案优先复用现有 `lg-list-page`、`lg-search-bar`、`lg-list-table-panel`、`ColumnSettingsButton` 和 `useColumnSettings`。不新增依赖，不扩后端 schema；除非验收证明接口能力不足，否则只在项目台账前端和 API 参数封装内收敛。

**Tech Stack:** Vue 3、TypeScript、Vite、Ant Design Vue、VxeGrid、Vitest、Playwright、Spring Boot `/projects` API。

---

## 规则边界

- 主线程只负责计划、分派、督办和验收，不亲自实现代码。
- 代码实现必须由子智能体执行；每个子智能体派工必须显式写明 `model`、`thinking` 和理由。
- 代码检索时仓库存在 `.codegraph/`，执行者需要优先使用 CodeGraph。
- 不修改已经应用过的 Flyway migration。
- 不新增 UI 组件库或表格库。
- 合同台账是同类列表页生产化基准；本任务复用其成熟交互，不照搬无关业务逻辑。
- 任务完成后清理本轮 Node/Vite/Playwright 残留进程或说明保留原因。

## 当前事实

- 项目台账页：`frontend-admin/src/pages/project/index.vue`
- 项目 API 封装：`frontend-admin/src/api/modules/project.ts`
- 项目类型定义：`frontend-admin/src/types/project.ts`
- 后端列表接口：`backend/src/main/java/com/cgcpms/project/controller/PmProjectController.java`
- 后端列表参数：`pageNo`、`pageSize`、`keyword`、`projectCode`、`projectName`、`projectType`、`status`
- 前端当前列表请求使用 `pageNum`，与后端 `pageNo` 不一致。
- 前端创建项目金额直接传 `contractAmount`；编辑项目金额传 `contractAmount * 10000`；列表展示时按 `/ 10000` 显示为万元，存在单位漂移风险。
- 页面已经有 `isMobile`，但当前主要展示仍是桌面 `vxe-grid`。
- 页面 KPI 当前主要基于当前页 `tableData`，不能包装成全量经营指标。

  ```ts
  function amountWanToYuan(value: number | undefined): string | undefined {
    return value != null ? String(Math.round(value * 10000)) : undefined
  }

  function amountYuanToWan(value: string | undefined): number | undefined {
    const n = Number(value)
    return Number.isFinite(n) ? n / 10000 : undefined
  }
  ```

  创建和编辑提交统一使用：

  ```ts
  contractAmount: amountWanToYuan(createForm.contractAmount)
  ```

- `pnpm build` 通过。
- 相关 Vitest 通过，或已证明失败项为无关历史基线。
- 浏览器控制台无本任务引入的红错。
- `/api/projects` 无本任务引入的失败请求。

不通过条件：

- 金额存在 10000 倍漂移。
- 分页仍发送 `pageNum` 导致后端分页不可控。
- 桌面或移动端核心操作不可用。
- 类型检查或构建失败且无法证明无关。
- 浏览器验收出现本任务引入的 API 或控制台错误。

## 暂不做

- 不做全量项目经营统计后端聚合接口。
- 不做历史金额数据清洗。
- 不重构项目页为多个组件。
- 不新增表格、日期、图表或状态管理依赖。
- 不扩大到项目成员、项目总览或合同台账页面。
