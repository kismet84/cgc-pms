# 第21条主线-非阻塞遗留问题治理任务计划书

**创建日期**: 2026-07-05  
**适用阶段**: 第15至第20条主线已形成当前归档结论后的非阻塞遗留治理阶段  
**版本**: v1.0  
**计划属性**: 主线计划，不是返工总计划，不是全量重构方案

**Goal:** 在不推翻第15至第20条既有通过/非阻塞/可上线结论的前提下，对当前已筛查出的非阻塞遗留问题形成串行治理计划，优先完成权限治理剩余裁决、上线前总控状态校准，以及项目级隔离/角色菜单绑定/入口注册等最小闭环任务，并为后续正式收口提供明确执行顺序、分工和验收口径。  
**Architecture:** 本计划继续复用现有权限体系、菜单绑定机制、入口注册模式、质量归档流程和主线收口口径，遵循“最小可行治理、最少必要变更、先裁决后扩展、先收口后增强”原则；禁止将第21条扩写为全系统 RBAC 重构、消息平台重构、全量历史主线返工或一次性全站架构翻修。

---

## 一、结论

第21条主线应立为“**非阻塞遗留问题治理主线**”，可以启动，但必须严格收口为以下边界：

1. 当前主线**不是**返工第15至第20条。
2. 当前主线**不是**一次性全量重构。
3. 当前主线**不是**把所有历史计划重新拉回阻塞状态。
4. 当前主线只处理本轮已明确筛查出的遗留事项，并按 `P0 -> P1 -> P2` 串行推进。

本主线建议结论如下：

1. `P0` 必须优先处理：
   - 第10/10A 权限治理剩余逐码裁决；
   - 第13条上线前总控状态校准。
2. `P1` 作为本主线的实现型治理包：
   - 第10B 项目级强隔离最小实现；
   - 第10C 角色菜单绑定编辑增强；
   - 第10D 全站入口 registry 首批实现。
3. `P2` 仅保留为后续增强型复验：
   - 第15后续增强仅限外部渠道正式打通与更多角色/域/异常路径复验；
   - 不作为当前阻塞，不回退第15条既有通过/非阻塞归档。

最终判断口径：

1. 第21条主线是“非阻塞遗留治理”，不是“当前版本不可上线”的阻塞总包。
2. 只有第10/10A 剩余高危权限缺口与第13条总控校准结果，才有资格影响最终上线裁决。
3. 第16、第17、第18、第19、第20，以及第15的已通过阶段，不得被本计划重新定义为“当前待实现阻塞”。

## 五、优先级与执行清单

### 5.1 P0 清单

#### P0-1：第10/10A 权限治理剩余逐码裁决

目标：

1. 对剩余高风险权限码逐条给出当前有效裁决。
2. 明确哪些属于：
   - 缺权限码
   - 缺菜单绑定
   - 缺 `submit` 绑定
   - 缺 `authority` 控制
   - 高危 `DB-only` 残留
3. 输出“通过 / 不通过 / 需最小修复 / 需专项治理”分类结果。

重点对象：

1. `invoice:list`
2. `inventory:transaction:list`
3. `dashboard:view`
4. `project:list`
5. 未绑定 `submit` 权限
6. `authority` 缺口
7. 高危 `DB-only` 权限

#### P0-2：第13条上线前总控状态校准

目标：

1. 重建当前有效的上线前总控结论。
2. 不沿用旧“已完成”表述冒充当前状态。
3. 对质量门禁、历史阻断项、核心链路、文档索引、上线决策包形成最新口径。

产出要求：

1. 当前有效门禁结果。
2. 当前有效阻断项复核。
3. 当前有效核心链路回归结论。
4. 当前文档索引口径。
5. 上线决策包。

### 5.2 P1 清单

#### P1-1：第10B 项目级强隔离最小实现

目标：

1. 在不做全局权限重构的前提下，建立项目级强隔离最小闭环。
2. 优先保证后端真实边界，不把前端默认筛选当作隔离。

#### P1-2：第10C 角色菜单绑定编辑增强

目标：

1. 补齐角色菜单绑定的编辑能力与治理闭环。
2. 让权限平台对菜单绑定关系具备可解释、可调整、可验收的最小能力。

#### P1-3：第10D 全站入口 registry 首批实现

目标：

1. 建立首批入口注册清单与最小治理机制。
2. 优先解决“入口存在但未注册”“注册缺失导致口径漂移”的问题。

### 5.3 P2 清单

#### P2-1：第15后续增强复验储备

目标：

1. 仅补外部渠道正式打通计划。
2. 仅补更多角色、更多业务域、更多异常路径复验计划。
3. 不回退第15当前“通过 / 非阻塞”归档。

同时满足以下条件，才可判定第21条主线通过：

1. 第10/10A 剩余权限码与高危缺口已形成当前有效裁决。
2. 第13条总控校准已形成当前有效门禁、阻断项、核心链路和决策包结论。
3. 第10B 已完成最小项目级强隔离闭环，且后端边界真实生效。
4. 第10C 已完成角色菜单绑定编辑增强并可验收。
5. 第10D 已完成首批入口 registry 实现并形成明确范围。
6. 第15后续增强边界已明确：哪些已正式打通，哪些仍属占位或后续复验。
7. 最终归档能明确区分：
   - 通过 / 不通过
   - 阻塞 / 非阻塞
   - 当前关闭项
   - 后续储备项

### 10.2 不通过标准

出现以下任一情况，应判定第21条主线当前不通过：

1. 仍无法解释高危 `DB-only` 权限或 `authority` 缺口影响范围。
2. 总控校准仍依赖历史旧数字、旧报告、旧截图冒充当前证据。
3. 第10B 强隔离仍停留在前端默认筛选。
4. 第10C 或第10D 被扩写为超范围平台重构，导致无法收口。
5. 第15后续增强被误写成第15当前未通过。

### 10.3 阻塞/非阻塞判断口径

1. 能直接影响上线裁决的权限高危缺口或总控校准缺失，按阻塞处理。
2. 第10B/10C/10D 未完成但不影响当前主线放行结论时，可按阶段性非阻塞处理，但不得伪装为已完成。
3. 第15外部渠道正式打通与更多覆盖面复验，默认按非阻塞增强处理，除非用户另行提升标准。

## 十一、风险与回滚

### 11.1 主要风险

1. 权限误判风险：
   - 把历史“修过”误判为当前“通过”。
2. 越权风险：
   - 强隔离只做在前端，后端仍可越权访问。
3. 扩包风险：
   - 第10B/10C/10D 被扩成平台级重构。
4. 口径漂移风险：
   - 第15已通过项被错误拉回阻塞。
5. 文档失真风险：
   - 第13总控校准继续引用失效结论。

### 11.2 回滚原则

1. 若第10B/10C/10D 实现中发现必须升级为大范围架构改造，应立即停止当前包并回报主线程重新立专项。
2. 若第13总控校准发现新的真实阻塞，应优先更新上线决策口径，而不是继续推进低优先级增强。
3. 若第15后续增强需要突破现有通知边界，应拆为后续专项，不回滚第15当前归档。

## 十二、最终收口口径

第21条主线收口时，只允许输出以下类型结论：

1. **通过，非阻塞遗留治理已完成当前阶段收口**
   - 适用条件：`P0/P1` 已闭环，`P2` 边界清晰，未误伤既有通过主线
2. **不通过，存在阻塞项**
   - 适用条件：高危权限缺口或总控校准仍未形成当前有效结论
3. **需要确认，证据不足**
   - 适用条件：当前缺少足够证据支持明确裁决

必须明确强调：

1. 第21条是非阻塞遗留治理主线。
2. 第21条不是第15至第20条返工主线。
3. 第21条不是一次性全量重构主线。
4. 第21条完成后，仍可继续拆出后续专项，但不得把当前最小闭环治理无限扩写。
