# 硬编码常量关联字典服务实施方案

## 一、背景与目标

### 1.1 当前问题

1. **字典服务未被业务代码使用**：后端大量使用硬编码常量（如 `ContractStatusConstants`、`SettlementStatusConstants`、`WorkflowConstants`），未通过字典服务获取
2. **同一状态值重复定义**：`DRAFT`、`APPROVED`、`REJECTED` 等在多个 Constants 类中重复定义
3. **字典服务无缓存**：每次请求都查询数据库，性能瓶颈
4. **前端不一致**：部分页面使用字典服务，部分硬编码

### 1.2 目标

1. 为字典服务添加缓存机制，提升性能
2. 将硬编码常量关联到字典服务，实现配置化管理
3. 新增必要的字典类型，覆盖所有业务场景
4. 前端统一从字典服务获取状态选项

## 二、实施范围

### 2.1 已有字典类型（V5 种子数据）

#### 3.1.1 添加 Guava Cache 缓存

**修改文件**：`backend/src/main/java/com/cgcpms/system/dict/service/SysDictDataService.java`

**实现方案**：
```java
@Service
@Slf4j
@RequiredArgsConstructor
public class SysDictDataService {

    // 缓存：dictCode -> List<SysDictDataVO>
    private final LoadingCache<String, List<SysDictDataVO>> dictCache = CacheBuilder.newBuilder()
            .maximumSize(1000)
            .expireAfterWrite(10, TimeUnit.MINUTES)
            .build(new CacheLoader<>() {
                @Override
                public List<SysDictDataVO> load(String dictCode) {
                    return fetchByDictCode(dictCode);
                }
            });

    // 新增：根据字典编码查询（带缓存）
    public List<SysDictDataVO> getByDictCodeCached(String dictCode) {
        try {
            return dictCache.get(dictCode);
        } catch (ExecutionException e) {
            log.error("Failed to load dict data for code: {}", dictCode, e);
            return List.of();
        }
    }

    // 清除缓存（字典数据变更时调用）
    public void evictCache(String dictCode) {
        dictCache.invalidate(dictCode);
    }

    // 实际查询逻辑（原 getByDictCode 方法）
    private List<SysDictDataVO> fetchByDictCode(String dictCode) {
        // 原有实现...
    }
}
```

**关键点**：
- 缓存 TTL 设置为 10 分钟（字典数据变动频率低）
- 最大缓存条目 1000（足够覆盖所有字典类型）
- 字典数据增删改时调用 `evictCache()` 清除缓存

#### 3.1.2 修改字典数据 Service 的增删改方法

在 `create`、`update`、`delete` 方法中调用 `evictCache()`：

### 阶段三：创建字典查询工具类（优先级高）

#### 3.3.1 创建 DictUtils 工具类

**文件**：`backend/src/main/java/com/cgcpms/system/dict/util/DictUtils.java`

```java
package com.cgcpms.system.dict.util;

import com.cgcpms.system.dict.service.SysDictDataService;
import com.cgcpms.system.dict.vo.SysDictDataVO;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

/**
 * 字典工具类 - 提供静态方法访问字典数据
 */
@Component
public class DictUtils {

    private static SysDictDataService dictDataService;

    @Autowired
    public void setDictDataService(SysDictDataService service) {
        DictUtils.dictDataService = service;
    }

    /**
     * 根据字典编码获取字典数据列表
     */
    public static List<SysDictDataVO> getDictDataByCode(String dictCode) {
        return dictDataService.getByDictCodeCached(dictCode);
    }

    /**
     * 根据字典编码和键值获取标签
     */
    public static String getLabelByValue(String dictCode, String dictValue) {
        return getDictDataByCode(dictCode).stream()
                .filter(item -> item.getDictValue().equals(dictValue))
                .map(SysDictDataVO::getDictLabel)
                .findFirst()
                .orElse(dictValue);
    }

    /**
     * 根据字典编码获取键值-标签映射
     */
    public static Map<String, String> getValueLabelMap(String dictCode) {
        return getDictDataByCode(dictCode).stream()
                .collect(Collectors.toMap(
                        SysDictDataVO::getDictValue,
                        SysDictDataVO::getDictLabel,
                        (v1, v2) -> v1
                ));
    }

    /**
     * 根据字典编码和键值判断是否有效
     */
    public static boolean isValidValue(String dictCode, String dictValue) {
        return getDictDataByCode(dictCode).stream()
                .anyMatch(item -> item.getDictValue().equals(dictValue));
    }
}
```

## 六、风险与注意事项

### 6.1 风险点

1. **缓存一致性**：字典数据变更后，缓存需及时清除
2. **向后兼容**：保留原有常量类，避免破坏现有代码
3. **性能影响**：缓存初始化时可能有短暂延迟

### 6.2 注意事项

1. **多租户隔离**：缓存 key 需包含 tenantId，避免跨租户数据混淆

### 8.2 缓存 Key 的多租户隔离问题

**问题**：当前方案中缓存 key 仅为 `dictCode`，未包含 `tenantId`，可能导致跨租户数据混淆。

**修正方案**：
```java
// 缓存 key 需包含 tenantId
private String buildCacheKey(String dictCode) {
    return UserContext.getCurrentTenantId() + ":" + dictCode;
}
```

**注意**：由于 `UserContext` 是线程级变量，Guava LoadingCache 的 `load()` 方法中调用 `UserContext.getCurrentTenantId()` 可能返回 null。建议：
1. 在 `getByDictCodeCached()` 方法中先获取 tenantId，再构建缓存 key
2. 或使用 `CacheBuilder.newBuilder()` 的 `build(CacheLoader.from(key -> ...))` 模式

### 8.3 DictUtils 实现优化

**当前方案问题**：使用 `@Component` 注解和静态方法注入，虽然可行但不是最佳实践。

**优化方案**：使用 Spring 的 `ApplicationContextHolder` 模式：
```java
@Component
public class ApplicationContextHolder implements ApplicationContextAware {
    private static ApplicationContext context;
    
    @Override
    public void setApplicationContext(ApplicationContext ctx) {
        context = ctx;
    }
    
    public static <T> T getBean(Class<T> clazz) {
        return context.getBean(clazz);
    }
}

// DictUtils 中使用
public static String getLabelByValue(String dictCode, String dictValue) {
    SysDictDataService service = ApplicationContextHolder.getBean(SysDictDataService.class);
    return service.getByDictCodeCached(dictCode).stream()
            .filter(item -> item.getDictValue().equals(dictValue))
            .map(SysDictDataVO::getDictLabel)
            .findFirst()
            .orElse(dictValue);
}
```
