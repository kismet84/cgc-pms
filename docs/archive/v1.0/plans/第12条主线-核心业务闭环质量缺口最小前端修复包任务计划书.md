## 6. 串行小包

### 小包 1：审批中心前端显示口径修复

执行角色：

- 主执行：前端实现助手
- 协同：测试验收员
- 复核：主负责人裁决

目标：

1. 处理 GAP-001，确保 `/approval/cc` 详情为只读，不再直出同意、驳回、撤回等可执行动作。
2. 处理 GAP-002，确保 `/approval/todo` 不再裸露 `CONTRACT`，并给出中文化标签。

涉及文件建议：

- `frontend-admin/src/pages/approval/todo.vue`

验收标准：

- Given 当前页签是 `/approval/cc`，When 打开审批详情，Then 页面不展示同意、驳回、撤回等执行动作，只保留只读信息。
- Given 当前页签是 `/approval/todo`，When 业务类型为 `CONTRACT`，Then 页面显示中文业务类型，不展示原始码值 `CONTRACT`。
- Given 当前页签是 `/approval/todo`、`/approval/mine`，When 打开详情，Then 既有允许的动作展示逻辑不被误伤。
- Given 业务类型未知，When 页面渲染，Then 兜底显示中文占位文案，不回退到裸码值。

风险点：

- 只读门禁过严可能误伤 `/approval/todo` 或 `/approval/mine` 的合法动作展示。
- 业务类型兜底文案若写得过于生硬，后续可能需要产品再校准一次，但不能把内部码值直接露给用户。

### 小包 2：采购经理待验日期文案修复

执行角色：

- 主执行：前端实现助手
- 协同：测试验收员
- 复核：主负责人裁决

目标：

1. 处理 GAP-003，修复采购经理 `待验收入库` 未来日期显示 `今日` 的问题。
2. 保持后端 `pendingDays` 当前语义不变，只修前端展示口径。

涉及文件建议：

- `frontend-admin/src/pages/dashboard/components/DashboardPurchaseView.vue`

验收标准：

- Given `待验收入库` 项目的日期在未来，When 打开采购经理 dashboard，Then 文案显示 `剩余X天`，不显示 `今日`。
- Given `待验收入库` 项目的日期就是今天，When 打开 dashboard，Then 文案显示 `今日`。
- Given `待验收入库` 项目的日期已过去，When 打开 dashboard，Then 文案显示 `逾期X天`。
- Given 日期缺失或非法，When 打开 dashboard，Then 文案显示 `-`，不抛出页面异常。

风险点：

- 如果继续沿用后端 `pendingDays` 直接作展示文案来源，未来日期仍可能出现语义偏差。
- 仅改展示口径时要避免影响现有排序和其他统计块。

## 8. 数据流变化

### 8.1 审批中心

当前数据流大致是：

`/approval/todo` 拉取审批列表与详情数据 -> 前端根据页签和详情状态渲染动作区 -> 展示业务类型、时间、状态和入口。

本主线后的变化是：

1. 后端返回的 `availableActions` 仍然保留原始语义，不做契约改造。
2. 前端在 `/approval/cc` 只渲染只读详情，不把 `availableActions` 误展示成可执行动作。
3. 前端在 `/approval/todo` 将 `CONTRACT` 映射为中文业务类型，不再裸露码值。
4. 未知业务类型统一走中文兜底，不回显内部编码。

### 8.2 采购经理 dashboard

当前数据流大致是：

`DashboardService` 输出 `pendingDays` 和日期字段 -> `DashboardPurchaseView.vue` 渲染 `待验收入库` 文案。

本主线后的变化是：

1. 后端 `pendingDays` 仍然保持原有语义，不改字段契约。
2. 前端不再把 `pendingDays = 0` 直接解释为 `今日`。
3. `待验收入库` 文案改为基于 `record.date` 的日期差值展示。
4. 数据缺失时继续显示 `-`，不新增错误状态。
