## 3. GAP-004：生产经理 `近期验收` 未来日期显示 `0天`

### 3.1 现象

驾驶舱生产经理页 `近期验收` 中，未来日期 `2026-07-05`、`2026-07-03` 当前显示为 `0天`。

### 3.2 证据

- 测试验收员已在真实页面复验确认该现象。
- 证据路径：`D:/projects-test/cgc-pms/output/playwright/production-manager-gap004-recheck-20260701/`
- 上一轮归档报告中 GAP-004 原状态为“非阻塞，需要确认”，需要确认点是生产经理是否必须复用采购经理 `剩余X天 / 今日 / 逾期X天 / -` 三态规则。

### 3.3 产品裁决

产品 / 需求负责人已确认：现有冻结文档没有写明生产经理必须复用采购经理三态规则，当前阶段不必须复用采购经理 `剩余X天 / 今日 / 逾期X天 / -` 口径。

### 3.4 测试事实

测试事实支持“现象存在”，但不支持将其直接升级为当前阶段阻塞缺口。原因是该显示没有被当前冻结基线明确约束为必须三态表达。

### 3.5 技术 / 安全补充

该项本质是生产经理页时效文案口径问题，不是已确认的数据权限、安全边界或后端动作执行问题。若后续要统一驾驶舱日期表达，应先补充产品口径，再拆局部前端展示增强，不应在本轮直接套用采购经理规则。

### 3.6 最终分类

- 分类：后续增强
- 级别：非阻塞
- 当前处置：暂不立即修

### 3.7 后续建议

如下一阶段需要提升驾驶舱一致性，建议先补一条明确口径：生产经理 `近期验收` 是否表达“距离验收剩余 / 今日 / 已逾期”还是表达“已等待 / 已完成后的天数”。口径确定后再拆一个最小前端展示小包，避免扩展为驾驶舱全量重构。

## 4. GAP-005：审批详情弹窗未见显式业务详情入口

### 4.1 现象

合同审批、采购申请审批、分包计量审批详情弹窗中，当前未见显式“查看业务详情 / 查看业务单据”入口。

### 4.2 证据

- 测试验收员已在真实页面抽查 `/approval/done` 三类样本详情。
- 合同审批、采购申请审批、分包计量审批均未见显式业务入口。
- 弹窗内可见审批标题、模板名称、发起人、发起时间、当前轮次、审批流程、审批记录等核心审批信息。
- 证据路径：`D:/projects-test/cgc-pms/output/playwright/approval-detail-gap005-recheck-20260701/`
- 上一轮归档报告中 GAP-005 原状态为“非阻塞，需要确认”，需要确认点是“审批详情必须显式提供业务详情入口”是否为当前阶段强制要求。

### 4.3 产品裁决

产品 / 需求负责人已确认：当前阶段合同审批、采购申请审批、分包计量审批详情弹窗不必须显式提供业务详情入口。该入口属于可用性增强项，不是当前冻结基线下的强制要求。

### 4.4 测试事实

测试事实支持“显式业务入口缺失”这一现象，但同时确认审批弹窗仍展示核心审批信息。因此，该现象当前不足以判定为审批中心主流程不可用或冻结基线不满足。

### 4.5 技术 / 安全补充

前端设计师确认：`approval/todo.vue` 内嵌详情与 `approval/detail.vue` 独立详情当前都没有业务详情入口；`businessType / businessId` 字段已具备。合同有较直接详情路由，采购申请与分包计量目前更偏列表 / 弹窗能力，不是完整深链详情。

安全审计员确认：当前无显式业务详情入口不构成安全问题，未发现越权、数据泄露或权限绕过。若后续新增入口，必须进入业务模块自身鉴权链路，不能让 workflow 详情绕过业务接口权限。

### 4.6 最终分类

- 分类：后续增强
- 级别：非阻塞
- 当前处置：暂不立即修

### 4.7 后续建议

若产品后续确认要增强审批详情可达性，建议新增“查看业务单据”入口。合同可优先深链到合同详情；采购申请、分包计量若暂时没有稳定深链详情页，可先跳转到对应列表并记录后续定向打开缺口。该增强必须由目标业务模块接口和页面完成自身鉴权，审批详情页只负责导航，不承担权限兜底。
