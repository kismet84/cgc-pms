# Ad-hoc 临时计划

本文件是 AutoPilot 在 Ready 队列少于 3 条时优先于长期计划读取的临时候选池。任务必须先拆入 `docs/backlog/ready-issues.md` 并成为合格 Ready Issue，才能实施；不得直接执行本文件中的条目。

状态使用 `ReadyToSplit` 或 `Candidate`；优先级使用 `高`。

| 临时待办 | 优先级 | 状态 |
| --- | --- | --- |
| 后端接口无前端入口治理 | 高 | Candidate |
| 采购补货建议 | 高 | Candidate |
| 现场日报 / 施工日志 | 高 | Candidate |
| WBS 任务依赖与延期预警 | 高 | Candidate |
| 供应商履约档案 | 高 | Candidate |
| 子智能体超时、悬挂线程退役与有限重派治理 | 高 | Candidate |

## 子智能体超时、悬挂线程退役与有限重派治理

本节仅定义候选规格，尚未实现自动化。该 Candidate 必须先拆成合格 Ready Issue，才能修改 runner/state。

- 5 分钟无新证据：仅做只读状态核验，不中断线程。
- 10 分钟无新证据：中断并永久退役该线程，禁止复用；超时退役线程不得承担后续 A-F 角色。
- 首次超时后可使用新线程重派最多一次；重派时必须缩小任务范围并补充缺失上下文。
- 同一任务第二次超时：停止重派，并转入 `docs/backlog/blocked-issues.md`。
- 状态记录至少包含：任务、线程、开始时间、最后进展、重试次数、超时原因。
- 预计超过 10 分钟的命令必须在派工时写明命令和预计时长，避免把正常长命令误判为悬挂线程。
