## 3. 安全性（最高权重）

### 3.1 高危漏洞修复（已修复 ✅）

| ID | 问题 | 描述 | 严重度 |
|----|------|------|--------|
| SEC-1 | **SQL注入** — WorkflowQueryService | `inSql()` 中字符串拼接 `tenantId` 和 `instanceStatus`，存在SQL注入风险 | **严重** ✅ 已修复 |
| SEC-2 | **开放重定向** — DevAuthController | `normalizeRedirect()` 仅检查 `//` 前缀，可被用于钓鱼攻击 | **高危** ✅ 已修复 |
| SEC-3 | **SpEL注入** — OperationAuditAspect | `StandardEvaluationContext` 允许调用任意Java方法甚至执行系统命令 | **高危** ✅ 已修复 |
| SEC-4 | **敏感信息泄露** — MinioHealthIndicator | 异常时返回 `e.getMessage()` 可能暴露MinIO连接URL/认证信息 | **中危** ✅ 已修复 |
| SEC-5 | **参数校验不足** — AlertController/SysRoleController | 使用 `Map<String, Object>` 接收请求体，无类型校验 | **中危** ✅ 已修复（引入DTO+@Valid）|

### 3.2 修复详情

#### SEC-1: WorkflowQueryService SQL注入修复 ✅

**修复前**:
```java
.inSql(WfTask::getInstanceId,
    "SELECT id FROM wf_instance WHERE deleted_flag = 0"
    + " AND tenant_id = " + tenantId
    + " AND instance_status = '" + WorkflowConstants.INSTANCE_RUNNING + "'");
```

**修复后**: 替换为 `LambdaQueryWrapper` + 预先查询的 `Set<Long>` 方式，完全消除SQL注入风险。

#### SEC-2: DevAuthController 开放重定向修复 ✅

**修复前**: `normalizeRedirect()` 仅检查 `//` 前缀

**修复后**: 增加白名单验证（30个允许的前端路由前缀）+ 路径遍历防护（`..` 检查），不匹配的回退到 `/`

#### SEC-3: OperationAuditAspect SpEL注入修复 ✅

**修复前**: `StandardEvaluationContext` 允许任意方法调用

**修复后**: `SimpleEvaluationContext.forReadOnlyDataBinding()` 限制为只读数据绑定

#### SEC-4: MinioHealthIndicator 敏感信息泄露 ✅

**修复前**: `e.getClass().getSimpleName() + ": " + e.getMessage()`

**修复后**: 返回通用信息 `"MinIO connection failed"`，健康详情也不返回 bucket 名称

### 3.3 残留风险
