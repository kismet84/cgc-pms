# 子智能体派工单模板

本模板仅在实际派工时使用；主线程直接执行时不填写本模板，也不输出 `model`、`thinking`、`reason`。单派不需要模型分配表；同时派出两个及以上子智能体时，先给出模型分配表，再逐个按本模板下发任务。

用于主线程向实施型、验收型、运维型、审计/归档型子智能体下发任务。先补齐只读前置检查，再按下列字段完整派工。

```text
你是被主线程明确派工的子智能体，不是主线程；在本派工范围内可以执行授权动作。

任务名称=
角色边界=你是被主线程明确派工的子智能体，不是主线程；在本派工范围内可以执行授权动作
目标=
范围=
背景=
前置检查=
- git branch --show-current:
- git status --short:
- git worktree list: （仅在怀疑分支/工作区冲突时补充）
禁止事项=
model=
thinking=
reason=
执行要求=
验收输出=
```

## 填写规则

- 只有派工净收益明确高于派工与回收成本时才使用本模板。
- 同时派出两个及以上子智能体时，模型分配表至少包含 `任务/角色`、`model`、`thinking`、`reason`；单派不得为形式完整机械补表。
- `前置检查` 至少填写当前分支和工作区状态；有并行隔离疑虑时再补 `git worktree list`。
- `范围` 只写本次允许触达的目录、文件、环境或验证动作。
- `禁止事项` 明确不能做的事，例如：`不要提交 git`、`不要启动服务`、`不要修改业务代码`。
- `reason` 只解释为什么这个任务需要当前 `model/thinking` 组合。
- `验收输出` 直接约束回报格式，避免子智能体只回过程不回结论。
