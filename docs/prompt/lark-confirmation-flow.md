# 飞书通知与确认交互流程

当执行过程中遇到必须由用户决策的情况，使用本流程通过飞书发起确认。此流程仅用于需要暂停执行等待选择的场景；可自行判断并继续的问题不要触发确认。

## 固定配置

### 发送消息给用户

```bash
lark-cli im +messages-send --user-id ou_20cc1ff2064c5eb97eaf3c6e78309292 --text "<消息内容>" --as bot
```

### 拉取用户发送的最新 1 条消息

```bash
lark-cli im +chat-messages-list --chat-id oc_d7f0f381ce06dc89f2a3f483be26c825 --page-size 1 --sort desc --as user
```

## 确认消息模板

```text
【执行确认】Phase X - <简短标题>
问题：<描述>
选项：
A) <选项A>（默认）
B) <选项B>
C) <选项C>
请回复 A/B/C；如 1 分钟内未回复，默认选择 A。
```

## 执行步骤

1. 发送确认请求：概述问题，列出明确的 A/B/C 选项，并标注默认选项。
2. 等待 60 秒，暂停执行，不进入自愈循环。
3. 拉取最新消息并解析 A/B/C（大小写不敏感）。
4. 如果匹配到选项，按对应选项继续执行。
5. 如果未匹配或无回复，再等待 30 秒并二次拉取。
6. 如果仍无有效回复，采用预先指定的默认选项，并在执行记录中注明“用户未回复，默认选择 A”。

## 关键规则

- 总等待时间最多 90 秒（60 秒 + 30 秒），不得无限等待。
- 每次确认都必须提供明确的单字母选项，便于自动解析。
- 默认选项必须在消息中提前说明。
- 发送确认请求后暂停当前决策点，不要一边等待一边执行依赖该决策的工作。
