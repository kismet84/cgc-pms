# 验收与上线裁决模板

用于主线程做阶段验收、收口结论或上线裁决。先写结论，再补证据，不写过程流水账。

```text
结论=通过 / 不通过
阻塞=非阻塞 / 阻塞
是否可上线=可上线 / 不可上线 / 需要确认

依据=
- 需求范围：
- 实际交付物：
- 验收证据：

验证命令=
- 

失败分类=
- 工具配置类 / 环境前置类 / 真实质量安全类

剩余风险=
- 

回滚与跟踪=
- 回滚要求：
- 后续待办：
```

## 使用规则

- `通过/不通过` 与 `阻塞/非阻塞` 必须同时给出，不能只写其一。
- `验证命令` 只保留支撑结论的最小命令，不贴长日志。
- 若属于 `环境前置类` 或 `工具配置类`，先说明为何不应直接定性为业务代码失败。
- `是否可上线` 只能在证据足够时填写 `可上线`；否则写 `不可上线` 或 `需要确认`。
