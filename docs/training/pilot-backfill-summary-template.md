# 现场回填会后汇报模板

## 1. 文档定位

本文件是“现场回填会后汇报模板”，用于会后整理与汇报，不是已完成汇报，也不是已确认最终结论。

使用边界：
- 只定义会后怎么报、引用哪些现有文件、哪些事项能进入回写准备。
- 不填写真实账号、真实密码、真实审批模板名、真实会议结论。
- 未签收通过事项不得写成正式已完成结论。

## 2. 可直接复制的汇报模板

```text
各位好：

以下为本次“现场回填会”会后汇报模板，请按实际会议结果补齐后发送；本模板本身不是已完成汇报。

一、本次确认范围
- 会议时间：`待填写`
- 组织方：`待填写`
- 记录人：`待填写`
- 参与角色：`待填写`
- 本次覆盖事项：试点租户范围 / 账号开通范围 / 角色归属 / 初始密码发放口径 / 密码重置口径 / 审批模板命名规范 / 审批人映射方式 / 生产经理能力边界（按实际勾选）
- 现场记录文件：`pilot-confirmation-record-seeded.md`
- 现场签收文件：`pending-items-signoff-seeded.md`

二、已确认项
- `待填写`
- 填写要求：仅列出在 `pending-items-signoff-seeded.md` 中已明确为“通过”的事项。

三、不通过项
- `待填写`
- 填写要求：写明对应事项、主要原因、阻塞点，不扩写未确认承诺。

四、保留待确认项
- `待填写`
- 填写要求：写明缺失资料、待补证据或待统一口径。

五、待回写项
- `待填写`
- 填写要求：仅列出已签收“通过”的事项，并注明拟按 `confirmation-writeback-sop.md` 回写到哪个正式文件。

六、未完成原因
- `待填写`
- 填写要求：聚焦资料缺失、演示不完整、口径冲突、责任人未到场等客观原因。

七、下一步责任人
- `待填写`
- 填写要求：逐项写明谁补资料、谁复核 seeded 文件、谁按 `confirmation-writeback-sop.md` 组织回写。

八、引用文件
- `pilot-confirmation-record-seeded.md`
- `pending-items-signoff-seeded.md`
- `confirmation-writeback-sop.md`
- 如涉及生产经理能力边界：`../manuals/production-manager-capability-boundary.md`

九、说明
- 本次汇报只反映本次现场回填会的记录和签收状态，不代表正式手册已完成回写。
- 只有按 `confirmation-writeback-sop.md` 完成回写与复核后，相关事项才可视为进入正式文档口径。
```

## 3. 使用提醒

- 汇报前先核对 seeded 文件中的现场记录与签收结论是否一致。
- 如存在“保留待确认”或“不通过”事项，汇报中必须显式保留，不得省略。
