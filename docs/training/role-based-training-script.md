# 角色化培训脚本（最小版）

## 1. 使用说明

- 本脚本只服务试点培训最小交付。
- 当前上线收口已通过，但培训材料仍是试点交付阻塞项。
- 总工程师当前冻结，不纳入本轮正式培训主清单。

## 2. 培训总顺序

1. 开场说明当前边界：上线收口已通过，但培训材料本身是试点落地阻塞项。
2. 先讲通用登录、菜单、权限边界。
3. 再按角色分段演示。
4. 现场问题统一记录到 [FAQ 与问题回流](faq-feedback-loop.md)。

## 3. 角色分段

### 3.1 项目经理

- 目标：理解项目执行协同、待办推进、预警与合同履约主线。
- 关键演示步骤：
  1. 登录并进入工作台或驾驶舱。
  2. 查看待办、预警、临期事项。
  3. 进入项目相关台账或审批中心处理任务。
- 常见疑问：
  - 为什么某些经营类图表不显示：因为项目经理与商务经理口径不同。
  - 为什么页面无数据：先检查项目选择和数据权限。
- 建议时长：15 分钟。

### 3.2 商务经理

- 目标：理解合同、变更、成本控制、结算和付款相关台账主线。
- 关键演示步骤：
  1. 进入商务经理驾驶舱查看经营结果与成本偏差。
  2. 进入合同、签证、成本、付款相关模块查看台账。
  3. 演示审批状态跟踪。
- 常见疑问：
  - 为什么当前角色不展示采购执行主图：因为采购经理是独立角色。
  - 付款与发票入口关系是什么：以付款与发票相关模块联动说明为准。
- 建议时长：15 分钟。

### 3.3 采购经理

- 目标：理解采购执行、验收、库存联动和供应商相关主线。
- 关键演示步骤：
  1. 进入采购、验收、库存相关菜单。
  2. 查看采购订单状态、未入库情况、库存预警。
  3. 说明验收与库存的联动关系。
- 常见疑问：
  - 为什么不展示商务利润分析：因为当前角色边界不包含经营驾驶舱。
  - 默认演示环境为什么现在有数据：因为当前最小 demo 数据已补齐。
- 建议时长：12 分钟。

### 3.4 生产经理

- 目标：理解现场执行近似主线，包括验收、领料、库存和分包计量。
- 关键演示步骤：
  1. 进入验收、领料、库存、分包计量相关菜单。
  2. 演示材料验收状态、领料出库、库存变化。
  3. 强调当前可演示能力与“完整施工管理能力”的边界。
- 常见疑问：
  - 是否已经覆盖完整进度、劳务、机械、产值：当前需要确认，不能按已具备能力承诺。
  - 页面空白是否代表功能缺失：先分清数据为空还是能力未纳入当前边界。
- 建议时长：12 分钟。

### 3.5 系统管理员

- 目标：理解账号角色、菜单权限、审批入口和交付前检查。
- 关键演示步骤：
  1. 说明角色放行边界与冻结角色。
  2. 演示角色与菜单权限的核对方式。
  3. 演示审批中心入口和交付前检查清单。
- 常见疑问：
  - 菜单显示是否代表权限完整：不是，还需核对按钮和 API 权限。
  - 交付前是否需要重复部署步骤：部署动作跳转运维手册，不在本脚本展开。
- 建议时长：15 分钟。

## 4. 收尾话术

- 本轮培训只覆盖当前有效范围，不承诺冻结角色和未确认能力。
- 现场问题不口头遗留，统一记入 FAQ 并按分类回流。
