# 试点现场确认记录单模板

## 1. 文档定位

本文件是试点现场使用的“记录单模板”，不是已完成确认结果。

使用边界：
- 只用于记录账号、密码发放口径、审批模板、审批人映射、生产经理能力边界等事项的现场确认过程。
- 不得在本模板中填写真实账号、真实密码、真实模板名、真实签字结论之外的虚构信息。
- 未形成现场证据的内容，只能标记为“待确认”或“不通过”，不得倒填为“已确认”。

## 2. 状态口径

| 状态 | 说明 |
|------|------|
| 已确认 | 现场已完成核对，且有可回查证据 |
| 待确认 | 当前仍缺证据、缺责任人或缺统一口径 |
| 不通过 | 现场核对后确认当前口径不可用、不可回写或存在冲突 |
| 不适用 | 本次试点场景不涉及该项 |

## 3. 基本信息

| 字段 | 填写说明 |
|------|------|
| 确认日期 | 填写现场确认日期 |
| 确认场景 | 如：试点培训前核对、试点演示现场、交付前复核 |
| 项目/试点范围 | 填写本次确认覆盖的试点范围或租户范围描述，不写真实敏感信息 |
| 组织方 | 填写组织确认的责任团队或角色 |
| 记录人 | 填写本记录单整理人 |
| 参与角色 | 至少写明实施负责人、系统管理员、流程管理员、培训讲师、业务代表等角色 |

## 5. 扩展记录区

如现场出现清单外事项，按同一口径补充记录。

| 确认日期 | 确认场景 | 确认项 | 当前状态 | 参与角色 | 证据 | 现场结论 | 后续动作 | 回写位置 |
|------|------|------|------|------|------|------|------|------|
| 待填写 | 待填写 | 待填写 | 待确认 | 待填写 | 待填写 | 待填写 | 待填写 | 待填写 |

## 6. 签收口径

- “已确认”必须同时满足：有责任角色、有现场结论、有证据、有明确回写位置。
- “待确认”不得回写正式手册，只能继续保留在确认清单或本记录单中跟踪。
- “不通过”必须写明冲突点、缺失点或阻塞原因，并转入签收单决定是否允许后续回写。
- “不适用”必须能说明本次试点为何不涉及，避免被误解为已默认通过。

## 7. 回写规则

- 先更新本记录单，再由责任人决定是否进入 [待确认事项签收单模板](pending-items-signoff.md)。
- 只有在签收结论为“通过”时，才允许把已确认口径回写到正式手册或培训材料。
- 回写时只写最终确认口径，不把现场草稿、口头分歧或敏感信息直接转写进正式文档。
