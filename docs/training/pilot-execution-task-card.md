# 试点现场执行任务卡

## 1. 文档定位

本文件用于下一次真实现场执行前，明确谁按什么顺序做什么。

使用边界：
- 本文件是执行任务准备文档，不是已执行结果，也不是已签收结果。
- 本文件不填写真实账号、真实密码、真实审批模板名、真实执行结论。
- 本文件只规定执行目标、责任角色、输入资料、执行顺序、完成标志和未完成留痕方式。

## 2. 执行目标

在一次现场动作内，把当前待确认的账号、角色、密码、审批配置和生产经理能力边界事项，按既有模板完成“资料输入 -> 现场记录 -> 签收判断 -> 回写移交准备”的最小闭环。

## 3. 责任角色

| 角色 | 主要职责 |
|------|------|
| 实施负责人 | 确认本次范围、召集参与人、处理阻塞升级 |
| 系统管理员 | 提供租户、账号、角色、密码相关资料与现场证据 |
| 流程管理员 | 提供审批模板与审批人映射资料、配合演示 |
| 培训讲师 | 控制培训口径，只按已确认范围演示 |
| 业务代表 | 对试点使用口径和生产经理能力演示给出确认意见 |
| 记录人 | 填写 seeded 记录单、签收单，并保留未完成项留痕 |

## 4. 输入资料

执行前至少准备：
- `external-input-checklist.md`
- `pilot-account-and-approval-checklist.md`
- `pilot-confirmation-record-seeded.md`
- `pending-items-signoff-seeded.md`
- `pilot-confirmation-runbook.md`
- `confirmation-writeback-sop.md`
- 如涉及生产经理：`../manuals/production-manager-capability-boundary.md`

## 5. 执行顺序

| 顺序 | 谁做 | 做什么 | 完成标志 | 未完成时如何留痕 |
|------|------|------|------|------|
| 1 | 实施负责人 + 记录人 | 按 `external-input-checklist.md` 核对外部资料是否到位，标出缺失项 | 已形成“本次可确认项 / 仍缺资料项”列表 | 在 `pilot-confirmation-record-seeded.md` 对应项写“待确认”，并注明缺失资料 |
| 2 | 实施负责人 | 依据 `pilot-account-and-approval-checklist.md` 锁定本次现场范围与参与角色 | 本次确认范围明确，责任角色明确 | 在 `pilot-confirmation-record-seeded.md` 基本信息或后续动作栏标“需要确认” |
| 3 | 系统管理员 + 流程管理员 | 逐项提供账号、角色、密码、审批相关资料或演示 | 每项至少有一种可回查证据 | 无证据项保留“待确认”，不得口头改为已确认 |
| 4 | 培训讲师 + 业务代表 | 对生产经理能力边界做最小路径演示与现场确认 | 演示范围、证据和确认意见都已记录 | 演示不完整时，相关项保留“待确认”或记“不通过” |
| 5 | 记录人 | 将现场情况写入 `pilot-confirmation-record-seeded.md` | 记录单已补齐状态、证据、结论、后续动作 | 未形成证据或结论的项继续保留默认值 |
| 6 | 责任角色 + 记录人 | 按记录单填写 `pending-items-signoff-seeded.md` | 每项已有“通过 / 不通过 / 保留待确认”之一 | 缺依据项不得写“通过”，保持“保留待确认” |
| 7 | 文档回写责任人（如已具备条件） | 按 `confirmation-writeback-sop.md` 仅整理可回写项，不直接扩写未确认内容 | 已形成待回写事项列表 | 未通过或待确认事项继续留在 seeded 文件中，不进入正式回写 |

## 6. 与现有文件的关系

| 文件 | 关系 |
|------|------|
| `pilot-confirmation-record-seeded.md` | 本任务卡对应的现场记录主文件，所有现场核对先写这里 |
| `pending-items-signoff-seeded.md` | 本任务卡对应的签收文件，只有在记录单已有依据后才进入这里 |
| `pilot-confirmation-runbook.md` | 本任务卡是现场执行版摘要，执行顺序应与 runbook 保持一致 |
| `confirmation-writeback-sop.md` | 本任务卡只负责把事项推进到可回写准备，不代替正式回写 SOP |

## 7. 完成定义

本任务卡对应的一次现场执行，只有同时满足以下条件才算完成：
- seeded 记录单已逐项填写现场状态；
- seeded 签收单已区分“通过 / 不通过 / 保留待确认”；
- 未完成项已经明确缺什么、卡在哪、由谁补；
- 只有允许回写的事项才进入回写准备。

## 8. 使用提醒

- 本任务卡是现场执行前的准备卡，不代表现场已经完成。
- 若外部资料仍未补齐，可继续使用本任务卡组织现场，但缺失项必须留痕，不得补写成既成事实。
