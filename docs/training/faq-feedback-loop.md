# FAQ 与问题回流（最小版）

## 1. 使用目的

本文件用于承接试点培训现场和试点早期使用中的高频问题，避免问题只留在聊天记录里。

当前定位：
- 对第 13 条主线当前上线收口非阻塞。
- 对试点交付与培训落地是阻塞支撑项。

## 2. FAQ 记录字段

每条 FAQ 至少包含以下字段：

| 字段 | 说明 |
|------|------|
| 编号 | 便于追踪 |
| 问题 | 用户原始问题 |
| 适用角色 | 项目经理 / 商务经理 / 采购经理 / 生产经理 / 系统管理员 / 其他 |
| 场景 | 培训现场 / 试点试用 / 交付准备 |
| 临时答案 | 当前可给出的最小答复 |
| 状态 | 已确认 / 需要确认 / 待补充 |
| 是否需要回流 | 是 / 否 |
| 回流分类 | 文档补充 / 测试补充 / 产品或流程确认 |
| 责任人 | 后续跟进人 |
| 下次复核时间 | 便于追踪关闭 |

## 4. 问题回流分类规则

### 4.1 文档补充

满足以下任一条件时归入文档补充：
- 当前系统行为已明确，但手册没写清。
- 现场反复问同一入口、路径、角色边界。
- 需要补充“非本轮范围”或“需要确认”提示语。

### 4.2 测试补充

满足以下任一条件时归入测试补充：
- 培训现场发现操作路径与预期不一致。
- 同一问题在多个账号或多个页面重复出现。
- 需要验证权限、审批、数据展示是否稳定。

### 4.3 产品或流程确认

满足以下任一条件时归入产品或流程确认：
- 当前文档没有有效依据支撑。
- 业务方要求超出当前角色边界或主线范围。
- 历史手册与当前系统口径冲突。

## 5. 处理规则

1. 能从当前有效文档直接回答的问题，优先现场给出最小答案。
2. 没有当前证据支撑的问题，必须标记“需要确认”。
3. 不允许把 `.archive` 旧手册直接当成当前结论。
4. 回流问题至少要落入一个分类，不能悬空。
5. 同类问题连续出现两次以上，应优先补文档，再决定是否补测试或升级确认。
