# 待确认事项签收单模板

## 1. 文档定位

本文件是待确认事项完成现场核对后的“签收单模板”，不是已完成签收结果。

使用边界：
- 只用于对待确认事项做“通过 / 不通过 / 保留待确认”的最终签收判断。
- 不记录真实账号、真实密码、真实模板名或未获授权公开的敏感信息。
- 未经现场记录支撑的事项，不得直接签收为“通过”。

## 2. 签收结论口径

| 结论 | 说明 |
|------|------|
| 通过 | 证据充分，允许按签收口径回写正式手册或培训材料 |
| 不通过 | 当前证据不足、口径冲突或能力未成立，不允许回写正式手册 |
| 保留待确认 | 当前暂不下最终结论，继续保留在确认清单和记录单中跟踪 |

## 3. 基本信息

| 字段 | 填写说明 |
|------|------|
| 签收日期 | 填写本次签收日期 |
| 签收场景 | 如：试点培训后、交付前终审、现场复盘 |
| 组织角色 | 填写组织签收的责任角色 |
| 记录人 | 填写本单整理人 |
| 审核参与角色 | 填写参与签收判断的角色 |

## 5. 签收判断规则

- 结论为“通过”时，签收依据至少应包含现场记录编号或记录位置、责任角色、证据类型、允许回写位置。
- 结论为“不通过”时，必须写明阻塞原因，且“是否允许回写正式手册”固定为“否”。
- 结论为“保留待确认”时，表示当前不构成通过，也不形成失败结论，继续留在确认清单中跟踪。

## 6. 回写规则

- 只有“通过”事项，才允许回写到正式手册、培训脚本或培训入口说明。
- “不通过”与“保留待确认”事项，不得被培训现场表述为既成事实。
- 回写前应先核对来源文档、现场记录和签收单三者口径一致。
