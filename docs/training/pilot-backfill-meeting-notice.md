# 现场回填会会议通知正文模板

## 1. 文档定位

本文件是“现场回填会会议通知正文模板”，用于执行人复制后发送，不是已发送通知，也不是已完成会议记录。

使用边界：
- 只定义通知怎么发、会前准备什么、会后产出什么。
- 不填写真实账号、真实密码、真实审批模板名、真实会议结论。
- 生产经理能力边界必须并入同一次会，不单独拆会。

## 2. 可直接复制的通知正文

```text
各位好：

为完成试点现场确认结果回填，拟组织一次“现场回填会”。本通知为正文模板，请按实际时间、地点、参与人补齐后发送；本模板本身不是已发送通知。

一、会议目的
1. 依据 `pilot-backfill-session-plan.md` 完成本次回填范围确认。
2. 现场核对账号、角色、密码、审批配置相关待确认项。
3. 将生产经理能力边界并入同一次会议完成演示、记录和签收准备。
4. 现场更新 seeded 文件，为会后回写和汇报提供依据。

二、时间与地点
- 会议时间：`待填写`
- 会议地点/会议链接：`待填写`
- 组织方：`待填写`
- 记录人：`待填写`

三、参会角色
- 实施负责人
- 系统管理员
- 流程管理员
- 培训讲师
- 业务代表
- 记录人

四、会前需携带资料
- `external-input-checklist.md`
- `pilot-execution-task-card.md`
- `pilot-backfill-session-plan.md`
- `pilot-confirmation-record-seeded.md`
- `pending-items-signoff-seeded.md`
- `confirmation-writeback-sop.md`
- `../manuals/production-manager-capability-boundary.md`（如本次涉及生产经理能力边界核对）

五、现场需要直接更新的 seeded 文件
- `pilot-confirmation-record-seeded.md`
- `pending-items-signoff-seeded.md`

六、会后产物
- 已补充现场记录的 `pilot-confirmation-record-seeded.md`
- 已补充签收结论的 `pending-items-signoff-seeded.md`
- 待回写事项列表
- 未完成项清单（注明缺什么、谁补、何时补）
- 按 `pilot-backfill-summary-template.md` 整理的会后汇报

七、执行提醒
- 本次会议只处理真实现场确认与回填，不把未确认事项写成既成事实。
- 只有签收为“通过”的事项，后续才允许按 `confirmation-writeback-sop.md` 进入正式回写。
- 如资料缺失或口径冲突，现场应保留“待确认”或“不通过”，不要口头视为默认通过。
```

## 3. 使用提醒

- 发送前必须补齐时间、地点、组织方、记录人等占位信息。
- 如本次仅开会未形成签收结论，会后汇报中必须明确哪些事项仍待确认。
