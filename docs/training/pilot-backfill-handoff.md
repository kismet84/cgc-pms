# 试点真实回填外部执行交接单

## 1. 文档定位

本文件是“把当前剩余事项移交给外部执行人”的交接单。

明确说明：
- 本文件不是已完成真实回填结果。
- 本文件不是已完成签收结论。
- 本文件只说明当前仓库内已准备到什么程度、剩余哪些动作必须由外部真实执行继续完成。

- 当前仍缺真实现场执行，因此正式 actual 文件仍保持“待回填”安全占位。
- 在没有真实数据、真实记录、真实签收前，不允许更新 [pilot-confirmation-record-actual.md](D:/projects-test/cgc-pms/docs/training/pilot-confirmation-record-actual.md) 和 [pending-items-signoff-actual.md](D:/projects-test/cgc-pms/docs/training/pending-items-signoff-actual.md) 中的待回填占位。

## 4. 仍待外部执行事项

以下事项必须由外部真实执行人继续完成，仓库内当前不代填、不代签：

1. 真实现场记录
   依据现场真实参与人、真实演示过程、真实证据，填写 [pilot-confirmation-record-actual.md](D:/projects-test/cgc-pms/docs/training/pilot-confirmation-record-actual.md)。
2. 真实签收
   依据真实现场记录形成“通过 / 不通过 / 保留待确认”正式签收，并填写 [pending-items-signoff-actual.md](D:/projects-test/cgc-pms/docs/training/pending-items-signoff-actual.md)。
3. 正式回写
   仅对真实签收明确允许回写的事项，按 [confirmation-writeback-sop.md](D:/projects-test/cgc-pms/docs/training/confirmation-writeback-sop.md) 执行正式回写。

补充约束：
- 如缺真实数据，不允许更新 actual 文件中的待回填占位。
- 如只有口头说明、没有可追溯证据，不允许写成已确认结果。
- 如未完成真实签收，不允许进入正式回写。

## 5. 建议接手角色

建议由以下角色接手：
- 实施负责人：负责组织现场、控范围、处理阻塞。
- 记录人：负责填写真实记录与真实签收文件。
- 系统管理员：负责提供账号、角色、密码相关真实证据。
- 流程管理员：负责提供审批模板与审批人映射真实证据。
- 业务代表：负责现场确认业务口径，参与签收。
- 文档回写责任人：仅在签收允许回写后执行正式回写。

## 6. 建议执行顺序

1. 先读 [external-input-checklist.md](D:/projects-test/cgc-pms/docs/training/external-input-checklist.md)，确认外部资料是否齐备。
2. 再按 [pilot-execution-task-card.md](D:/projects-test/cgc-pms/docs/training/pilot-execution-task-card.md) 组织一次真实现场执行。
3. 现场完成后，填写 [pilot-confirmation-record-actual.md](D:/projects-test/cgc-pms/docs/training/pilot-confirmation-record-actual.md)。
4. 基于真实记录填写 [pending-items-signoff-actual.md](D:/projects-test/cgc-pms/docs/training/pending-items-signoff-actual.md)。
5. 仅对允许回写事项，按 [pilot-backfill-result-package.md](D:/projects-test/cgc-pms/docs/training/pilot-backfill-result-package.md) 和 [confirmation-writeback-sop.md](D:/projects-test/cgc-pms/docs/training/confirmation-writeback-sop.md) 完成正式归档与回写。

- 仓库内准备工作已收口，后续外部执行入口已明确。
- 外部执行人能直接从必读文件继续，不需要补建新的流程机制。
- real actual 文件仍保持真实结果驱动，未发生假填、预填或越权回写。

- 已形成真实现场记录。
- 已形成真实签收结论。
- 已按 SOP 完成允许回写事项的正式回写。
