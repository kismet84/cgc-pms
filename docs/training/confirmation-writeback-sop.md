# 确认结果回写标准作业说明

## 1. 文档定位

本文件是“确认完成后如何回写正式文档”的 SOP，不是已完成回写结果。

使用边界：
- 只定义允许回写什么、谁回写、回写到哪里、回写后如何复核。
- 不替代现场记录单和签收单。
- 不把未确认事项、口头说法或敏感信息写成正式事实。

## 2. 回写责任

| 角色 | 责任 |
|------|------|
| 记录人 | 提供来源记录位置、签收结论和拟回写内容 |
| 文档回写责任人 | 根据签收结论更新正式文档 |
| 审核人 | 复核回写内容是否与签收单一致 |
| 实施负责人 | 对跨角色冲突项做最终协调 |

## 3. 允许回写条件

仅当以下条件同时满足时，才允许回写：
- 来源事项在 `pending-items-signoff.md` 中签收结论为“通过”。
- 能定位到对应现场记录位置。
- 已明确正式回写目标文件。
- 回写内容是“确认后的最小稳定口径”，不是现场讨论过程。
- 回写内容不包含真实账号、真实密码、真实模板名等敏感信息。

## 4. 禁止回写条件

出现以下任一情况时，禁止回写：
- 签收结论为“不通过”或“保留待确认”。
- 只有口头结论，没有现场记录或签收依据。
- 现有手册、确认单、签收单三者口径不一致。
- 生产经理能力边界仍处于待确认状态。
- 需要确认的内容尚未转换成稳定表述，却试图写成“系统已支持”或“现场已确认”。

## 5. 回写目标文件

| 回写事项 | 目标文件 | 回写原则 |
|------|------|------|
| 登录提示、密码发放方式、密码重置口径 | `docs/manuals/end-user-manual.md` | 只写用户可执行或可获知的最终口径 |
| 角色开通、账号准备、审批模板入口、审批人映射口径 | `docs/manuals/admin-manual.md` | 只写管理员侧稳定规则，不写现场草稿 |
| 培训组织顺序、试点培训前置说明 | `docs/training/README.md` 或角色化培训脚本 | 只写培训使用边界和顺序 |
| 生产经理新增已确认能力边界 | `docs/manuals/production-manager-capability-boundary.md` 或相关正式手册 | 只写已签收通过的能力口径 |

## 6. 回写步骤

### 第一步：核对来源

输入：
- `pilot-confirmation-record.md`
- `pending-items-signoff.md`

输出：
- 可回写事项列表

完成条件：
- 每个事项都能对应现场记录和签收结论。

### 第二步：确定目标文件

输入：
- 可回写事项列表

输出：
- 每个事项的唯一目标文件

完成条件：
- 不存在一个事项同时写入多个文件却口径不同的情况。

### 第三步：转换为正式口径

输入：
- 可回写事项列表
- 目标文件现有内容

输出：
- 拟回写文案

完成条件：
- 文案只保留最终确认口径，不保留“待确认”“据说”“现场口头说明”等过程性措辞。

### 第四步：回写后复核

输入：
- 已更新目标文件
- 签收单

输出：
- 复核结论

完成条件：
- 审核人确认回写内容没有超出签收范围，也没有把未确认事项写成事实。

## 7. 回写后复核要求

- 必须核对“来源记录位置、签收结论、目标文件表述”三者一致。
- 必须确认未把“保留待确认”事项转写进正式手册。
- 必须确认未补充任何现场未确认的新承诺。
- 若回写涉及生产经理能力边界，必须再次核对 `production-manager-capability-boundary.md` 与正式手册口径一致。

## 8. 失败处理

| 失败场景 | 处理方式 |
|------|------|
| 发现目标文件内容超出签收范围 | 退回文档回写责任人修改，并重新复核 |
| 发现待确认事项被写成事实 | 立即撤回该表述，恢复为未确认口径 |
| 同一事项在不同文件表述不一致 | 暂停回写，交由实施负责人统一口径 |
| 生产经理能力边界未签收通过却被要求写入正式手册 | 拒绝回写，保留在确认单与签收单中继续跟踪 |

## 9. 完成定义

本 SOP 对应的回写完成，必须同时满足：
- 只有签收为“通过”的事项被回写；
- 每个回写项都有明确目标文件；
- 回写后已完成复核；
- 未确认内容没有被写成正式事实。
