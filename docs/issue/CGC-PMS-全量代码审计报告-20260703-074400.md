## 🔴 第1章：后端安全审计

### 1.1 SQL 注入风险 — ✅ 未发现问题

**结论**: 项目在所有 Mapper 接口和自定义查询中均使用 MyBatis-Plus 的 `#{}` 参数化占位符，未发现 `${}` 字符串拼接。DAO 层全部 @Select/@Update/@Delete/@Insert 注解均使用参数化查询。

### 1.2 文件上传/下载安全 — ⚠️ 2项低危

| ID | 严重度 | 文件 | 行号 | 描述 | 建议 |
|----|:-----:|------|:----:|------|------|
| VUL-001 | 🟢 低危 | `backend/src/main/java/com/cgcpms/file/service/FileTypeValidator.java` | 19-23 | 文件类型白名单含 .txt/.csv，魔术字节检测仅检查无 null 字节。可上传不含 null 字节的任意文本内容（如 HTML/JS），有潜在 XSS 风险 | 限制 text/plain 文件响应头为 `Content-Type: text/plain; charset=utf-8` 且 `Content-Disposition: attachment` |
| VUL-002 | 🟢 低危 | `backend/src/main/java/com/cgcpms/file/controller/FileController.java` | 28 | 文件大小限制为 50MB，在速率限制（20次/分钟）下仍可能被利用进行存储耗尽攻击 | 考虑降低默认文件大小限制 |

### 1.3 认证/授权 — ✅ 良好

- 所有 48 个 Controller 类均有 `@PreAuthorize` 注解 ✅
- 使用 `BCryptPasswordEncoder` 进行密码编码 ✅
- Token 仅通过 HttpOnly Cookie 传递 ✅
- Token 刷新时 old refresh token 加入黑名单，防止重放攻击 ✅

仅 `OperationAuditAspect` 使用 `SpelExpressionParser` 解析注解中的编译时常量表达式（非用户输入），无注入风险。
