=== Key Design Decisions ===

1. NOT extending BaseEntity: sys_dict_type and sys_dict_data tables (V5 migration) do NOT have
   created_by, updated_by, deleted_flag, or remark columns. Extending BaseEntity would cause
   MyBatis-Plus @TableLogic (deleted_flag WHERE clause) and MetaObjectHandler (createdBy/updatedBy
   fill) to generate SQL referencing non-existent columns.

2. Explicit tenantId set in create(): MetaObjectHandler auto-fill requires @TableField annotations
   present on BaseEntity. Since entities don't extend BaseEntity, tenantId is set explicitly via
   entity.setTenantId(UserContext.getCurrentTenantId()) before insert.

3. NO caching: No @Cacheable or @EnableCaching anywhere in dict package — direct DB queries only.

4. Permission codes: system:dict:list (menu 504 in V6), system:dict:add/edit/delete (buttons
   721/722/723 in V39). Controllers use @PreAuthorize("hasRole('ADMIN') or hasAuthority('system:dict:xxx')").

5. Tenant isolation: Every LambdaQueryWrapper includes .eq(TenantId, UserContext.getCurrentTenantId()).
   Every getById/update/delete does post-check on entity.getTenantId().
