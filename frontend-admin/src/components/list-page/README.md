# list-page 公共组件目录

本目录放列表页可复用的轻量 UI 组件，只解决通用结构和交互，不承载业务字段定义、查询参数拼装或页面专属口径。

## 文件说明

- `LgPageHeader.vue`：列表页头部标题区。
- `LgSearchBar.vue`：轻量关键词搜索栏，适合“关键词 + 少量扩展筛选”的单行场景。
- `LgFilterPanel.vue`：带卡片外壳的筛选面板，适合字段较多、需要更明确筛选区块的列表页。
- `LgToolbar.vue`：列表页工具栏容器。
- `LgKpiCard.vue`：列表页上方 KPI 卡片。
- `LgEmptyState.vue`：空状态提示。
- `ColumnSettingsButton.vue`：列表列显隐设置按钮。
- `index.ts`：统一导出入口。

## 什么时候用 `LgSearchBar`

- 页面筛选很轻，只需要关键词输入。
- 扩展筛选字段不多，放在同一行即可。
- 不需要单独的卡片式筛选区块。

## 什么时候用 `LgFilterPanel`

- 列表页筛选项较多，页面需要一个独立筛选面板。
- 页面希望把筛选区和表格区明确分层。
- 业务页自己掌握字段布局，只复用公共外壳、关键词和查询/重置按钮。

## `LgFilterPanel` 使用边界

- 默认结构是“上方字段区 + 下方关键词/动作区”的筛选卡片，不负责业务字段本身。
- 只提供筛选面板布局、关键词输入、查询/重置按钮、`filters` 插槽。
- 业务字段通过 `filters` slot 自行传入。
- 不要把业务查询参数、字段 schema、配置驱动表单、动态字段注册塞进公共组件。
- 不要把页面专属校验、联动逻辑、接口请求写进公共组件。
