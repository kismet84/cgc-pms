## 正式版基线说明

本文件已按正式开发交付口径整理，并统一以下架构基线：

1. 一期数据库默认采用 **MySQL 8.0**，审批规则、动态表单和节点规则使用 MySQL JSON 字段。
2. 后端审批引擎内部采用**事务内领域服务直接调用**，不通过 HTTP 调用自身接口。
3. PC Web 是一期完整管理后台；移动端以 uni-app 为现场主端，企业微信和微信小程序作为轻入口。
4. 移动端正式业务开发节奏后移：第 6 周启动基础工程，第 8 周左右进入真实业务功能。
5. 所有业务数据以 `project_id` 为根，涉及合同的业务以 `contract_id` 为纲，成本以业务事实归集，付款与成本口径分离。

## 1. 测试目标

1. 验证项目、合同、采购、分包、成本、付款、结算全过程闭环。
2. 验证审批引擎支持顺序审批、会签、或签、转办、加签、撤回、驳回重提。
3. 验证成本自动归集、付款控制、结算追溯等核心规则。
4. 验证前后端 JSON 契约、错误码、权限和数据范围。
5. 验证 PC Web、移动端、企业微信、小程序等关键路径。
6. 验证上线部署、备份、监控、日志、回滚可执行。

## 3. 核心业务验收矩阵

| 链路 | 验收点 |
|---|---|
| 项目主线 | 项目可新建、成员可配置，核心业务均关联 `project_id` |
| 合同中心 | 合同可新建、审批、签订，采购/分包/付款/结算均引用 `contract_id` |
| 采购验收 | 采购申请通过后生成订单，验收通过后生成材料成本 |
| 分包计量 | 已签分包合同可计量，计量通过后生成分包成本 |
| 成本管理 | 成本来源可通过 `source_type/source_id` 追溯 |
| 付款管理 | 未签合同、未验收、未计量、超合同、超比例可拦截或升级 |
| 结算管理 | 结算从合同、变更、验收、计量、付款、成本汇总并可反查 |
| 资料归档 | 合同、验收、付款、结算资料可归档和预览 |

## 4. 审批引擎验收矩阵

| 功能 | 验收标准 |
|---|---|
| 顺序审批 | 当前人同意后进入下一节点 |
| 会签 | 支持全部通过、比例通过、任意驳回等规则 |
| 或签 | 一人处理后其他任务自动取消 |
| 转办 | 原任务转办后保留记录，新任务可处理 |
| 加签 | 支持前加签、后加签、并行加签 |
| 撤回 | 记录保留，待办取消，业务回草稿或待提交 |
| 驳回重提 | 新增轮次，旧轮次记录保留 |
| 节点实例 | 审批记录能追溯到具体节点实例 |
| 可用动作 | 前端按钮全部由后端返回 |
| 并发控制 | `taskVersion` 冲突时提示刷新 |
| 幂等控制 | 同一幂等键不会重复执行 |

## 5. API 契约验收

| 验收项 | 标准 |
|---|---|
| 统一响应 | 所有接口返回 `code/message/traceId/data` |
| ID 类型 | JSON 中所有 ID 均为字符串 |
| 金额类型 | 金额均为字符串，后端 BigDecimal |
| 枚举一致 | 前后端枚举值完全一致 |
| 日期格式 | 日期和日期时间符合约定 |
| 错误码 | 业务错误返回明确 code 和 message |
| 分页 | `pageNo/pageSize/total/records` 正确 |
| 幂等 | 动作接口重复提交不产生重复记录 |
| 版本冲突 | 任务版本不一致提示刷新 |

## 9. 运维与回滚

| 对象 | 运维策略 |
|---|---|
| MySQL | 每日全量备份，关键上线前手动备份 |
| MinIO | 定期备份对象存储数据 |
| Redis | 非核心缓存可重建，必要时开启持久化 |
| 应用服务 | 镜像版本管理，可回滚上一版本 |
| 数据库脚本 | 使用 Flyway / Liquibase 管理迁移 |
| 日志 | 关键业务、审批、接口、异常日志集中采集 |
| 监控 | 应用、数据库、中间件、业务指标监控 |
| 告警 | 服务不可用、慢 SQL、队列积压、磁盘不足、上传失败率高 |

### 回滚原则

1. 先回滚应用，再评估数据库回滚。
2. 高风险数据库变更必须提前准备反向脚本。
3. 上线前必须保留数据库和配置备份。
4. 生产问题优先止血，再分析根因。

## V1.2 增量：异常路径与边界测试矩阵

### 1. 审批引擎专项边界测试

| 场景 | 预期结果 |
|---|---|
| 两个审批人同时处理同一任务 | 只有一个成功，另一个返回 `WF_TASK_VERSION_CONFLICT` |
| 同一请求重复提交同一 `idempotencyKey` | 不重复写记录，返回首次处理结果 |
| 或签一人通过后其他人再点击同意 | 返回任务已取消或已处理 |
| 会签中一人驳回 | 按 rejectRule 结束或继续等待 |
| 驳回后重新提交 | `currentRound + 1`，旧记录保留 |
| 撤回已有人处理过的流程 | 已处理记录保留，未处理任务取消 |
| 非审批人处理任务 | 返回无权限 |
| 任务状态不是 PENDING | 禁止处理 |

### 2. 成本生成专项测试

| 场景 | 预期结果 |
|---|---|
| 材料验收审批通过 | 自动生成材料成本 |
| 同一验收单重复触发审批通过事件 | 不重复生成成本 |
| 成本生成失败 | 审批事务回滚或业务状态不生效 |
| 成本来源单据被作废 | 成本记录进入冲销或作废流程，不物理删除 |
| 分包计量审批通过 | 自动生成分包成本 |
| 签证变更审批通过 | 调整动态成本 |

### 3. 付款控制异常测试

| 场景 | 预期结果 |
|---|---|
| 未签合同发起付款 | 禁止提交，零星采购除外 |
| 材料未验收发起付款 | 禁止提交或强制升级，按规则配置 |
| 分包未计量发起付款 | 禁止提交 |
| 超合同金额付款 | 必须先走合同变更 |
| 超付款比例 | 自动升级审批或预警 |
| 质保金未到期释放 | 禁止或强制公司领导审批 |

### 4. 附件与弱网测试

| 场景 | 预期结果 |
|---|---|
| 附件上传中断 | 可重新上传，不产生脏业务附件 |
| 审批提交时附件不存在 | 禁止提交或提示补充附件 |
| 移动端离线保存草稿 | 本地草稿可恢复 |
| 弱网重复点击提交 | 后端幂等，只生成一次业务单据或审批记录 |
| 大文件上传失败 | 返回明确错误，不影响已保存草稿 |
