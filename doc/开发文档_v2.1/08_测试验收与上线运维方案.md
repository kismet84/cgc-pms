# 测试验收与上线运维方案

## 1. 测试目标

测试目标不是只验证页面能否点击，而是验证系统是否真正满足全过程闭环：

```text
合同可引用
审批可追溯
成本可自动生成
付款可被控制
结算可反查
权限可隔离
异常可阻断
重复提交可幂等
并发处理可控制
```

## 2. 测试分层

| 层级 | 内容 | 工具建议 |
|---|---|---|
| 单元测试 | 工具类、金额计算、审批规则、成本生成 | JUnit 5 / Mockito |
| 接口测试 | REST API、错误码、权限、分页筛选 | Postman / Apifox / REST Assured |
| 集成测试 | 审批 + 业务回调 + 成本生成 | Spring Boot Test |
| 前端测试 | 组件、表单、按钮、路由 | Vitest / Playwright |
| E2E 测试 | 合同到付款闭环 | Playwright |
| 性能测试 | 台账查询、审批并发、文件上传 | JMeter / k6 |
| 安全测试 | 权限绕过、越权、文件类型、注入 | OWASP ZAP / 手工测试 |
| UAT | 甲方业务验收 | 业务脚本 |

## 3. P0 测试矩阵

### 3.1 审批引擎专项测试

| 场景 | 正常路径 | 异常/边界 |
|---|---|---|
| 提交审批 | 草稿单据提交后生成实例、节点、任务、记录 | 重复提交、业务已删除、模板不存在、审批人为空 |
| 顺序审批 | A 同意后进入 B | 非当前审批人处理、任务已处理、版本冲突 |
| 会签 | 多人全部同意后通过 | 任一驳回、部分超时、重复处理 |
| 或签 | 任一人同意后通过，其他任务取消 | 全部驳回、取消任务再处理 |
| 驳回重提 | currentRound + 1，旧记录保留 | 非驳回状态重提、业务版本不一致 |
| 撤回 | 未处理任务取消，业务回到可编辑 | 已有下一节点处理后是否允许撤回按规则控制 |
| 转办 | 原任务转办，新任务生成 | 转给无权限人员、转办后原审批人再次处理 |
| 加签 | 前加签/后加签/并行加签记录正确 | 加签人为空、重复加签、加签任务驳回 |
| availableActions | 后端按角色和状态返回按钮 | 前端伪造动作，后端必须拦截 |
| 幂等 | 同一 idempotencyKey 重复提交只执行一次 | 不同 key 重复请求按业务规则判断 |
| 并发 | 两人同时处理同一任务仅一个成功 | 另一个返回 `WF_TASK_VERSION_CONFLICT` |

### 3.2 合同管理测试

| 场景 | 正常路径 | 异常/边界 |
|---|---|---|
| 新建合同 | 保存草稿成功 | 合同编号重复、合作方黑名单、金额为负 |
| 合同清单 | 多明细金额汇总 | 数量为 0、单价为负、合计不一致 |
| 付款条件 | 比例和金额维护 | 付款比例超过 100%、质保金为负 |
| 提交审批 | 进入审批中 | 缺少附件、缺少付款条件、超目标成本 |
| 审批通过 | 合同状态变已签订 | 重复审批回调不得重复生成锁定成本 |
| 作废合同 | 未发生后续业务可作废 | 已有订单/付款/结算不得作废 |

### 3.3 材料采购与成本测试

| 场景 | 正常路径 | 异常/边界 |
|---|---|---|
| 采购申请 | 提交审批通过 | 超预算自动升级、重复采购预警 |
| 采购订单 | 关联合同生效 | 超合同剩余额度、黑名单供应商 |
| 零星采购 | 可不关联合同 | 超零星采购阈值必须审批升级 |
| 材料验收 | 验收通过生成成本 | 无订单验收、超量验收、质量不合格 |
| 成本生成 | 生成 cost_item | 重复回调不重复生成，来源唯一约束生效 |
| 入库出库 | 库存流水正确 | 出库数量超过库存 |

### 3.4 分包与成本测试

| 场景 | 正常路径 | 异常/边界 |
|---|---|---|
| 分包任务 | 从合同引用生成任务 | 未签合同禁止下达 |
| 进度填报 | 分包进度可记录 | 进度超过 100% 预警 |
| 分包计量 | 计量通过生成分包成本 | 超合同工程量、质量未验收、安全处罚未处理 |
| 成本幂等 | 同一计量单只生成一次 | 重复审批回调、并发回调 |

### 3.5 付款测试

| 场景 | 正常路径 | 异常/边界 |
|---|---|---|
| 付款申请 | 基于合同和依据发起 | 未签合同、无验收/无计量、超付款比例 |
| 付款审批 | 按金额分级审批 | 超合同金额必须先变更 |
| 质保金释放 | 到期后可申请 | 未到期不得释放或强制升级 |
| 财务回写 | 生成付款记录 | 重复回写、金额大于审批金额 |
| 已付款汇总 | 回写合同已付款金额 | 多笔付款累计比例正确 |

## 4. 数据权限测试

| 场景 | 验收标准 |
|---|---|
| 项目经理 | 只能查看授权项目 |
| 成本经理 | 可查看成本数据，但不能修改合同审批结果 |
| 财务人员 | 可处理付款和实际支付，不能修改验收事实 |
| 分包单位 | 只能查看自己相关任务和计量 |
| 管理层 | 可查看全公司汇总 |
| 无权限用户 | 直接访问接口返回权限不足 |

## 5. 文件附件测试

| 场景 | 验收标准 |
|---|---|
| 上传合同附件 | 附件与合同正确关联 |
| 上传验收照片 | 照片与验收单正确关联 |
| 文件类型限制 | 非允许类型被拒绝 |
| 文件大小限制 | 超大小被拒绝 |
| 附件预览 | PDF/图片可预览 |
| 附件删除 | 已审批资料不得随意删除 |
| 上传失败 | 前端提示明确，可重试 |
| MinIO 不可用 | 后端返回可识别错误，不影响数据库事务一致性 |

## 8. 运维监控

| 监控项 | 说明 |
|---|---|
| 应用存活 | `/actuator/health` |
| JVM | CPU、内存、GC、线程 |
| MySQL | 连接数、慢 SQL、锁等待、磁盘 |
| Redis | 内存、连接数、命中率 |
| MinIO | 存储容量、上传失败率 |
| 审批 | 待办积压、超时任务、失败任务 |
| 文件 | 上传失败、预览失败 |
| 日志 | ERROR 数量、traceId 检索 |

## 9. 回滚方案

| 类型 | 回滚方式 |
|---|---|
| 前端发布 | Nginx 切回上一版本静态资源 |
| 后端发布 | Docker 镜像回滚上一版本 |
| 数据库脚本 | 上线前备份，禁止无回滚高危变更 |
| 配置错误 | 配置中心或环境变量回滚 |
| 文件存储 | 保留对象版本或定期备份 |

## 10. UAT 验收脚本

必须至少覆盖以下端到端流程：

```text
项目创建 → 合作方创建 → 合同新建 → 合同审批通过 → 采购订单 → 材料验收 → 成本生成 → 付款申请 → 付款审批 → 财务回写 → 结算汇总

项目创建 → 分包合同 → 分包任务 → 分包计量 → 成本生成 → 分包付款 → 分包结算

合同变更 → 审批通过 → 当前合同金额调整 → 付款比例重新计算 → 结算引用变更金额
```
