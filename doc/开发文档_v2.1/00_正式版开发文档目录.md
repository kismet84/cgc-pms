## 3. 统一架构基线

```text
业务主线：项目为根、合同为纲、执行为据、成本为果、付款为流、结算为终
系统架构：模块化单体优先，后期可演进微服务
数据库：MySQL 8.0，一期不默认 PostgreSQL
ID 策略：默认后端统一生成雪花 ID，数据库不强制 AUTO_INCREMENT
前端：Vue 3 + TypeScript + Vite + Ant Design Vue + Pinia + VxeTable + ECharts
后端：Java 21 + Spring Boot + MyBatis-Plus + MySQL + Redis + MinIO
审批：统一自研轻量审批引擎，业务模块通过回调 Handler 接入
移动端：第 6 周启动基础工程，第 8 周左右进入真实业务功能
桌面壳：一期延期，仅保留 Tauri 优先、Electron 备选
```

## 4. 开工前必须完成的 8 个动作

| 序号 | 动作 | 验收标准 |
|---:|---|---|
| 1 | 冻结技术栈 | 前后端仓库创建，依赖版本写入 README |
| 2 | 初始化数据库迁移 | Flyway 可执行，基础表和字典数据可一键初始化 |
| 3 | 创建前端脚手架 | 登录页、Layout、菜单、合同台账 POC 可运行 |
| 4 | 创建后端脚手架 | 登录、RBAC、统一响应、异常、日志、Swagger 可运行 |
| 5 | 审批引擎 POC | 跑通提交、顺序、会签、或签、驳回重提、撤回、并发冲突 |
| 6 | 业务回调接口 | 合同、材料验收、分包计量至少接入 Handler 样例 |
| 7 | 成本幂等生成 | `cost_item` 来源唯一约束和重复生成测试通过 |
| 8 | 测试矩阵确认 | P0 用例覆盖正常、异常、边界、并发、幂等、回滚 |
