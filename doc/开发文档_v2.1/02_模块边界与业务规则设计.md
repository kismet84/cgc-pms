# 模块边界与业务规则设计

## 1. 设计目标

本文件用于明确系统各模块的职责边界，避免重复建表、重复录入和业务口径混乱。所有开发、测试、验收均以此作为模块边界判断依据。

## 2. 模块边界总表

| 模块 | 负责什么 | 不负责什么 | 关键字段 |
|---|---|---|---|
| 项目管理 | 项目主数据、项目成员、项目目标成本、项目视图 | 不维护合同条款和付款数据 | `project_id` |
| 合同管理 | 合同主数据、合同金额、付款条件、合同清单、合同变更 | 不执行采购、验收、计量 | `contract_id` |
| 合作方管理 | 供应商、分包商、租赁商、服务商统一主数据 | 不维护业务合同金额 | `partner_id` |
| 材料设备 | 采购申请、采购订单、验收、入库、出库、库存 | 不重复维护采购合同主数据 | `order_id`, `receipt_id` |
| 分包管理 | 分包任务、进度、计量、评价、分包结算 | 不重复创建分包合同主表 | `sub_task_id`, `sub_measure_id` |
| 成本管理 | 成本归集、动态成本、偏差分析、预警 | 不作为业务事实录入口 | `cost_item_id`, `source_type`, `source_id` |
| 付款管理 | 付款申请、付款审批、付款依据、付款记录 | 不代表成本发生 | `pay_request_id`, `pay_record_id` |
| 变更签证 | 现场签证、合同变更、索赔、成本影响 | 不直接替代合同审批 | `var_order_id`, `ct_change_id` |
| 结算管理 | 总包、分包、采购、租赁结算汇总和归档 | 不重复录入前端业务数据 | `settlement_id` |
| 审批流程 | 流程模板、实例、节点、任务、记录 | 不直接持有业务主表字段 | `wf_instance_id`, `wf_task_id` |
| 文件资料 | 文件上传、附件关联、目录归档 | 不判断业务规则 | `file_id`, `business_id` |

## 3. 项目管理模块

### 3.1 定位

项目是所有业务数据的根节点。所有合同、采购、分包、成本、付款、结算、资料均必须归属项目。

### 3.2 核心功能

| 功能 | 说明 |
|---|---|
| 项目台账 | 查询、筛选、新建、编辑、归档 |
| 项目成员 | 维护项目经理、商务经理、成本经理、材料员、资料员等 |
| 项目目标 | 目标成本、计划工期、关键节点 |
| 项目全景 | 合同、成本、付款、进度、风险、资料统一视图 |
| 项目关闭校验 | 未结算、未付款、未归档、未关闭问题自动阻断 |

### 3.3 关闭前校验

项目关闭前必须校验：

```text
是否存在未结算合同
是否存在未关闭采购订单
是否存在未完成分包计量
是否存在未支付付款申请
是否存在未归档合同资料
是否存在未闭合签证变更
是否存在未关闭质量安全问题
```

## 4. 合同管理模块

### 4.1 定位

合同管理是系统合同数据中心。所有合同数据只维护一套，其他模块通过 `contract_id` 引用合同。

### 4.2 合同类型

```text
总包合同
分包合同
采购合同
租赁合同
服务合同
补充协议
其他合同
```

### 4.3 状态拆分

| 字段 | 示例值 | 说明 |
|---|---|---|
| `approval_status` | 草稿、审批中、已通过、已驳回、已撤回 | 审批状态 |
| `contract_status` | 未签订、已签订、履约中、已结算、已归档、已作废 | 合同业务状态 |

### 4.4 合同台账字段

合同台账默认显示字段和可选字段如下：

| 字段 | 默认显示 | 说明 |
|---|---:|---|
| 合同编号 | 是 | 主查询字段 |
| 合同名称 | 是 | 支持模糊搜索 |
| 合同类型 | 是 | 总包、分包、采购等 |
| 项目名称 | 是 | 关联项目 |
| 合作方 | 是 | 乙方或供应商 |
| 合同金额含税 | 是 | 原始合同金额 |
| 当前合同金额 | 是 | 原合同金额 + 已生效变更 |
| 已付款金额 | 是 | 财务实际支付汇总 |
| 付款比例 | 是 | 已付款 / 当前合同金额 |
| 签订日期 | 是 | 合同签署日期 |
| 合同状态 | 是 | 业务状态 |
| 审批状态 | 是 | 审批状态 |
| 变更金额 | 否 | 列设置可打开 |
| 结算状态 | 否 | 列设置可打开 |
| 归档状态 | 否 | 列设置可打开 |
| 风险等级 | 否 | 列设置可打开 |

### 4.5 参考页面差异整改

参考 HTML 页面当前主要展示合同编号、合同名称、合同类型、合作方、金额、签订日期、状态等基础字段。正式开发必须补齐：

```text

归档状态
风险等级
列设置
固定列
导出
批量操作
点击下钻
```

## 5. 材料设备管理模块

### 5.1 采购订单三类场景

| 场景 | 规则 |
|---|---|
| 有合同采购 | 必须关联采购合同 |
| 零星采购 | 可不关联合同，但必须标记零星采购原因和金额阈值 |
| 框架协议采购 | 关联框架合同，记录本次订单金额和累计使用金额 |

### 5.2 材料验收规则

| 控制点 | 规则 |
|---|---|
| 无采购订单 | 原则上不允许验收 |
| 到货数量大于订单数量 | 触发超量预警 |
| 质量不合格 | 禁止入库，进入退场或复检 |
| 资料不全 | 可暂存，不形成正式成本 |
| 验收通过 | 自动生成材料成本 |
| 已验收金额 | 作为付款申请上限之一 |

## 6. 分包管理模块

### 6.1 定位

分包管理负责分包履约过程，不重复维护分包合同主数据。分包合同来源于合同管理模块，分包管理通过 `contract_id` 引用。

### 6.2 分包计量控制规则

| 控制点 | 规则 |
|---|---|
| 未签分包合同 | 禁止提交计量 |
| 质量未验收 | 禁止计量通过 |
| 安全处罚未处理 | 付款时扣减或挂起 |
| 计量金额超合同 | 禁止通过，必须先变更 |
| 计量通过 | 自动生成分包成本 |

## 7. 成本管理模块

### 7.1 定位

成本管理负责成本归集、计算、分析和预警，不作为重复录入业务数据的入口。

### 7.2 成本来源

| 成本来源 | 来源单据 | 成本生成时点 |
|---|---|---|
| 合同锁定成本 | 合同审批 | 合同审批通过 |
| 材料成本 | 材料验收单 | 验收审批通过 |
| 分包成本 | 分包计量单 | 计量审批通过 |
| 机械成本 | 设备租赁/使用记录 | 使用确认通过 |
| 劳务成本 | 劳务考勤/工资 | 工资确认通过 |
| 变更签证成本 | 签证/变更单 | 审批通过 |
| 付款金额 | 财务付款记录 | 不生成成本，只作为资金流 |

### 7.3 幂等要求

成本自动生成必须满足幂等：同一个来源单据重复回调、重复提交、系统重试时，不允许重复生成成本。

建议唯一约束：

```sql
CREATE UNIQUE INDEX uk_cost_source
ON cost_item(source_type, source_id);
```

如按明细生成成本：

```sql
CREATE UNIQUE INDEX uk_cost_source_item
ON cost_item(source_type, source_id, source_item_id, cost_type);
```

## 8. 付款管理模块

### 8.1 付款分层

| 层级 | 归属 | 说明 |
|---|---|---|
| 付款条件 | 合同管理 | 合同约定的付款节点、比例、质保金 |
| 付款申请 | 业务模块发起 | 基于验收、计量、合同节点等依据 |
| 付款审批 | 审批引擎 | 按金额、规则、风险进行审批 |
| 实际支付 | 财务或财务接口 | 回写付款金额、日期、凭证 |

### 8.2 付款控制规则

```text
未签合同不得付款，零星采购除外
未完成验收的材料不得付款
未完成计量确认的分包不得付款
超合同金额付款必须先走合同变更
超付款比例必须预警或升级审批
质保金未到期不得释放
实际付款数据以财务系统为准
```

## 9. 结算管理模块

结算管理从以下数据汇总：

```text
合同
合同清单
合同变更
采购订单
材料验收
材料出入库
分包计量
现场签证
付款记录
成本明细
发票资料
验收资料
竣工资料
```

结算阶段禁止重新录入一套不可追溯的数据。
