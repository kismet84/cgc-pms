# 开发计划与里程碑

## 1. 计划原则

本计划按真实阻塞依赖链重排，不再只按模块平铺。

核心依赖链：

```text
脚手架与环境
→ 登录鉴权 / RBAC
→ 审批引擎 POC
→ 项目 / 合作方主数据
→ 合同管理
→ 采购订单 / 分包计量
→ 材料验收 / 计量确认
→ 成本生成
→ 付款申请
→ 结算
→ 驾驶舱与预警
```

## 2. 总体里程碑

| 阶段 | 周期 | 目标 |
|---|---:|---|
| 第 0 阶段 | 第 1 周 | 技术栈冻结、脚手架、数据库 migration、前端 POC 启动 |
| 第 1 阶段 | 第 1-4 周 | 基础平台、审批引擎 POC、项目/合作方、合同台账 POC |
| 第 2 阶段 | 第 4-6 周 | 合同中心完整闭环：新建、清单、付款条件、附件、审批 |
| 第 3 阶段 | 第 5-8 周 | 采购申请、采购订单、材料验收、材料成本生成 |
| 第 4 阶段 | 第 7-10 周 | 分包任务、分包进度、分包计量、分包成本生成 |
| 第 5 阶段 | 第 10-12 周 | 付款申请、付款审批、付款记录、发票 |
| 第 6 阶段 | 第 12-15 周 | 结算汇总、归档、预警、驾驶舱 |

| 第 8 阶段 | 第 19 周+ | 移动端增强、财务接口、BI、OCR、AI 风险识别 |

## 3. 前 4 周详细计划

### 第 1 周：开工准备与骨架搭建

| 工作项 | 负责人 | 交付物 | 验收标准 |
|---|---|---|---|
| 技术栈冻结 | 架构师 | 技术选型确认单 | Vue3/Java21/MySQL8 写入 README |
| 后端脚手架 | 后端 | Spring Boot 工程 | 启动成功，统一响应、异常、Swagger 可用 |
| 前端脚手架 | 前端 | Vue3 工程 | 登录页、Layout、路由、菜单框架可运行 |
| 数据库 migration | 后端/DBA | Flyway 脚本 | 本地一键初始化成功 |
| Redis/MinIO 配置 | 后端/运维 | docker-compose | 本地可上传文件、缓存字典 |
| 参考页组件拆分 | 前端 | 组件结构图 | Layout + 合同台账组件设计完成 |

### 第 2 周：RBAC 与审批 POC 启动

| 工作项 | 交付物 | 验收标准 |
|---|---|---|
| 用户/角色/菜单 | 基础权限接口 | 不同角色看到不同菜单 |
| 按钮权限 | 权限指令/组件 | 无权限按钮不可见，接口仍强校验 |
| 数据权限 | 项目数据权限模型 | 项目成员只能看授权项目 |
| 审批表结构 | `wf_*` migration | 模板、实例、节点、任务、记录可建表 |
| 审批提交 POC | 提交接口 | 可生成实例、节点、任务、记录 |
| 合同台账 POC | Vue 页面 | Mock 数据驱动，非硬编码表格 |

### 第 3 周：审批 POC 完整流转 + 主数据

| 工作项 | 交付物 | 验收标准 |
|---|---|---|
| 顺序审批 | 审批动作接口 | 同意后进入下一节点 |
| 会签/或签 | 流程规则 | 会签/或签任务状态正确 |
| 驳回重提 | `currentRound` | 旧轮次保留，新轮次生成 |
| 撤回/转办/加签 | 动作接口 | 任务和记录正确 |
| 项目 CRUD | 项目接口和页面 | 项目可创建、编辑、查询 |
| 合作方 CRUD | 合作方接口和页面 | 黑名单字段可维护 |

### 第 4 周：合同台账与合同新建基础

| 工作项 | 交付物 | 验收标准 |
|---|---|---|
| 合同台账接口 | 查询、筛选、分页、列设置 | 支持 15+ 字段返回 |
| 合同台账页面 | Vue 组件化页面 | 与参考页一致度 80% 以上 |
| 新建合同分步表单 | 7 步表单 | 基本信息、合同方、金额、清单、付款条件、履约节点、附件 |
| 合同提交审批 | 接入审批引擎 | 合同可提交审批，状态进入审批中 |
| 审批 availableActions | 后端返回按钮 | 前端不自行判断审批按钮 |
| 审批并发测试 | 测试用例 | taskVersion 冲突可复现并处理 |

## 4. 优先级矩阵

| 优先级 | 时间 | 事项 | 理由 |
|---|---:|---|---|
| P0 | 第 1 周 | 确定前端技术栈并搭建脚手架 | 没有框架后续页面无法开始 |
| P0 | 第 1 周 | 后端脚手架 + MySQL migration | 后端开发前置条件 |
| P0 | 第 1-2 周 | 审批引擎独立 POC | 所有业务审批前置依赖 |
| P0 | 第 1-2 周 | 登录 + RBAC + 菜单权限 | 后台基础能力 |
| P1 | 第 2-3 周 | 项目 + 合作方 CRUD | 合同中心前置数据 |
| P1 | 第 3-4 周 | 合同台账页面组件化 | 核心页面和列表交互规范 |
| P1 | 第 4-5 周 | 新建合同 + 审批提交 | 合同中心入口 |
| P2 | 第 5-7 周 | 采购订单 → 材料验收 → 成本生成 | 打通执行到成本链路 |
| P2 | 第 7-10 周 | 分包计量 → 成本生成 | 第二类核心成本来源 |
| P2 | 第 10-12 周 | 付款申请 + 财务回写 | 付款闭环 |
| P3 | 第 12 周+ | 驾驶舱、预警、结算 | 依赖前面数据沉淀 |

## 5. 移动端节奏

移动端正式业务开发不得过早启动。

| 周期 | 移动端安排 |
|---|---|
| 第 1-4 周 | 不启动正式业务开发，只做原型、交互规范、接口契约评审 |
| 第 5-6 周 | 启动基础工程：登录、Token、项目上下文、附件上传、离线草稿框架 |
| 第 7-10 周 | 做轻量审批：我的待办、审批详情、同意、驳回、转办、加签、项目速览 |
| 第 11-16 周 | 做现场高频业务：材料验收、拍照上传、现场签证、质量整改、扫码查询 |
| 第 17-20 周 | 做付款、结算、成本状态查询和消息通知 |
| 第 21 周以后 | 离线同步压测、多端兼容、弱网恢复、UAT 优化 |

## 6. 阶段验收标准

### 第 1 阶段验收

```text
前端脚手架可运行
后端脚手架可运行
MySQL 可一键初始化
登录和 RBAC 可用
审批 POC 跑通完整流转
合同台账 POC 可展示
```

### 第 2 阶段验收

```text
合同可新建、编辑、提交审批
合同审批通过后状态正确
合同付款条件可维护
合同附件可上传和预览
合同台账支持列设置和筛选
```

### 第 3 阶段验收

```text
采购申请可审批
采购订单可生效
材料验收可审批
验收通过后自动生成材料成本
重复回调不重复生成成本
```

### 第 4 阶段验收

```text
分包任务可从合同引用
分包计量可提交审批
计量通过后自动生成分包成本
超合同计量被拦截或要求变更
```

### 第 5 阶段验收

```text
付款申请可基于合同、验收、计量依据发起
未验收/未计量付款被拦截
超付款比例触发预警或升级审批
实际付款可回写
```

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 审批引擎复杂度高 | 阻塞所有业务流程 | 第 1-2 周独立 POC |
| 数据库脚本不可执行 | 后端开发延误 | 第 1 周完成 Flyway 初始化 |
| 前端参考页停留在静态 HTML | 页面返工 | 第 1-4 周完成 Vue 组件化 POC |
| 成本重复生成 | 经营数据错误 | 增加来源唯一约束和幂等测试 |
| 移动端过早启动 | 大量 Mock 和返工 | 按第 6 周后置启动 |
| 付款规则复杂 | 财务风险 | 后端强校验和异常升级 |
