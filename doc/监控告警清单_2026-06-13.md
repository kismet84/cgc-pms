## 1. 监控概览

### 1.1 监控目标

| 目标 | 说明 |
|------|------|
| **服务存活** | 确保核心服务（后端应用、MySQL、Redis、MinIO）7×24 可用 |
| **资源饱和度** | 预防连接池耗尽、JVM 内存溢出等容量问题 |
| **业务健康度** | 及时发现接口异常、登录攻击、审批/上传/批处理业务阻断 |
| **快速响应** | 故障发生后 5 分钟内感知，30 分钟内响应 |

### 1.2 一期范围

| 包含 | 不包含 |
|------|--------|
| ✅ 11 项监控指标定义（含告警阈值） | ❌ Prometheus server 部署 |
| ✅ Docker Healthcheck 配置建议 | ❌ Grafana 仪表盘搭建 |
| ✅ 简易 PowerShell 健康检查脚本 | ❌ Alertmanager 告警路由 |
| ✅ 日志关键字匹配简易告警方案 | ❌ 指标时序数据库（如 VictoriaMetrics） |
| ✅ 二期工具选型建议 | ❌ 钉钉/企微/邮件告警通道 |

### 1.3 架构概览

```

│                   监控层次                        │
```

## 2. 监控项清单

| # | 监控项 | 检查方式 | 告警阈值 | 严重级别 | 说明 |
|---|--------|---------|---------|---------|------|
| 1 | **后端应用存活** | `GET /api/actuator/health` | 连续 3 次非 200 | 🔴 P0-Critical | 应用进程存活检测；Actuator 端点需 JWT 认证，监控脚本需携带有效 token |
| 2 | **MySQL 可用性** | TCP 连接 `localhost:3306`（prod）/ `3307`（dev） | 连接超时 > 5s | 🔴 P0-Critical | 数据库不可用导致所有业务中断；Docker 已内置 `mysqladmin ping` healthcheck |
| 3 | **Redis 可用性** | TCP 连接 `localhost:6379` | 连接超时 > 3s | 🟡 P1-Warning | 缓存不可用时业务不中断，但性能下降；Docker 已内置 `redis-cli ping` healthcheck |
| 4 | **MinIO 可用性** | `GET http://localhost:9000/minio/health/live` | HTTP 状态非 200 | 🟡 P1-Warning | 文件上传/下载受影响；Docker 已内置 `mc ready local` healthcheck |
| 5 | **MySQL 连接池使用率** | `GET /api/actuator/metrics/hikaricp.connections.active` | active / maximum > 80% | 🟡 P1-Warning | 连接池即将耗尽，新请求阻塞；prod 配置 max=50，dev max=20 |
| 6 | **接口 5xx 错误率** | 日志关键字 `"SYSTEM_ERROR"` 或 HTTP 500 | 1 分钟内 > 10 次 | 🟡 P1-Warning | 后端服务异常频发；注意：`AuthorizationDeniedException` 已被正确映射为 403（`AUTH_FORBIDDEN`），不计入 5xx |
| 7 | **登录失败次数** | 日志关键字 `"Login failed"` 或日志级别 `WARN` + 登录路径 | 1 分钟内 > 20 次 | 🔵 P2-Info | 可疑暴力破解行为；需排查是否正常用户输错密码 |
| 8 | **审批失败数** | 日志关键字 `"Business exception"` + 审批相关上下文 | 1 小时内 > 5 次 | 🟡 P1-Warning | 审批流程异常可能导致业务阻断（提交/同意/驳回/撤回失败） |
| 9 | **文件上传失败数** | 日志关键字 `"MinIO"` 或 `"File upload failed"` | 1 小时内 > 3 次 | 🟡 P1-Warning | MinIO 连接异常或文件大小/类型校验失败；白名单支持 18 种扩展名，上限 50MB |
| 10 | **预警批处理失败** | 日志关键字 `"AlertEvaluationService"` + `ERROR` 级别 | 连续 2 次执行失败 | 🟡 P1-Warning | 8 类预警规则定时评估（每 30 分钟），连续失败将遗漏成本超支/合同到期等风险 |
| 11 | **JVM 堆内存使用率** | `GET /api/actuator/metrics/jvm.memory.used` + `jvm.memory.max` | used / max > 85% | 🟡 P1-Warning | 堆内存不足将导致 OOM 崩溃；建议结合 GC 频率综合判断 |

> **严重级别说明**：
> - 🔴 **P0-Critical**：服务完全不可用，需立即响应（5 分钟内）
> - 🟡 **P1-Warning**：服务降级或性能劣化，需 30 分钟内响应
> - 🔵 **P2-Info**：安全事件或异常趋势，需 2 小时内排查

### 3.3 现有服务 Healthcheck 说明

| **MySQL** | `mysqladmin ping -h localhost -u root -p$MYSQL_ROOT_PASSWORD` | 10s | 5s | 5 |

| **MinIO** | `mc ready local` | 10s | 5s | 5 |
| **Backend** | `curl -f http://localhost:8080/api/actuator/health` | 15s | 5s | 3 |

### 3.4 关于 Actuator 端点认证

**现状**：`/actuator/health` 端点不在 SecurityConfig `WHITELIST` 中（仅 `/auth/login`、`/auth/refresh`、`/swagger-ui/**` 等为匿名端点），因此需要 JWT 认证。

**一期方案**（简易脚本）：监控脚本携带有效 token 访问 `/api/actuator/health`。

**二期建议**：在 `application-prod.yml` 中暴露 health 端点到单独管理端口（无需认证）：

```yaml
management:
  server:
    port: 8081          # 独立管理端口
  endpoints:
    web:
      exposure:
        include: health,metrics
```

同时在 SecurityConfig 中放行管理端口：

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .securityMatcher("/**")  // 主应用端口
        // ... 现有配置 ...
        ;

    return http.build();
}
```

> ⚠️ **注意**：管理端口仅对内网暴露，切勿绑定到 `0.0.0.0`。

## 4. 简易健康检查脚本

### 4.1 PowerShell 脚本（Windows）

脚本路径建议：`scripts/health-check.ps1`

```powershell
<#
.SYNOPSIS
    CGC-PMS 简易健康检查脚本（一期）
.DESCRIPTION
    检查所有核心服务健康状态，输出汇总报告，失败项高亮。
    适用于 Windows 开发/测试环境。生产环境请替换为 Prometheus + Grafana。
#>

param(
    [string]$BaseUrl = "http://localhost:8080/api",
    [string]$MySqlHost = "localhost",
    [int]$MySqlPort = 3306,
    [string]$RedisHost = "localhost",
    [int]$RedisPort = 6379,
    [string]$MinioUrl = "http://localhost:9000",
    [string]$JwtToken = ""    # 可选：携带 JWT 访问 /actuator/health
)

$ErrorActionPreference = "Continue"
$results = @()
$passCount = 0
$failCount = 0
$warnCount = 0

Write-Host "`n========================================" -ForegroundColor Cyan
Write-Host "  CGC-PMS 健康检查报告" -ForegroundColor Cyan
Write-Host "  时间: $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')" -ForegroundColor Cyan
Write-Host "========================================`n" -ForegroundColor Cyan

# ──────────────────────────────────────────
# 1. 后端应用存活
# ──────────────────────────────────────────
Write-Host "[1/11] 后端应用存活 ..." -NoNewline
try {
    $headers = @{}
    if ($JwtToken) { $headers["Authorization"] = "Bearer $JwtToken" }
    $resp = Invoke-RestMethod -Uri "$BaseUrl/actuator/health" -Method Get -Headers $headers -TimeoutSec 5
    if ($resp.status -eq "UP") {
        Write-Host " ✅ UP" -ForegroundColor Green
        $results += @{ Item="后端应用存活"; Status="PASS"; Detail="status=UP" }
        $passCount++
    } else {
        Write-Host " ⚠️  DEGRADED" -ForegroundColor Yellow
        $results += @{ Item="后端应用存活"; Status="WARN"; Detail="status=$($resp.status)" }
        $warnCount++
    }
} catch {
    Write-Host " ❌ DOWN" -ForegroundColor Red
    $results += @{ Item="后端应用存活"; Status="FAIL"; Detail="$($_.Exception.Message)" }
    $failCount++
}

# ──────────────────────────────────────────
# 2. MySQL 可用性
# ──────────────────────────────────────────
Write-Host "[2/11] MySQL 可用性 (${MySqlHost}:${MySqlPort}) ..." -NoNewline
try {
    $tcp = New-Object System.Net.Sockets.TcpClient
    $connect = $tcp.BeginConnect($MySqlHost, $MySqlPort, $null, $null)
    $success = $connect.AsyncWaitHandle.WaitOne(5000)
    if ($success) {
        $tcp.EndConnect($connect)
        $tcp.Close()
        Write-Host " ✅ REACHABLE" -ForegroundColor Green
        $results += @{ Item="MySQL 可用性"; Status="PASS"; Detail="TCP ${MySqlHost}:${MySqlPort} OK" }
        $passCount++
    } else {
        $tcp.Close()
        throw "Connection timeout"
    }
} catch {
    Write-Host " ❌ UNREACHABLE" -ForegroundColor Red
    $results += @{ Item="MySQL 可用性"; Status="FAIL"; Detail="$($_.Exception.Message)" }
    $failCount++
}

# ──────────────────────────────────────────
# 3. Redis 可用性
# ──────────────────────────────────────────
Write-Host "[3/11] Redis 可用性 (${RedisHost}:${RedisPort}) ..." -NoNewline
try {
    $tcp = New-Object System.Net.Sockets.TcpClient
    $connect = $tcp.BeginConnect($RedisHost, $RedisPort, $null, $null)
    $success = $connect.AsyncWaitHandle.WaitOne(3000)
    if ($success) {
        $tcp.EndConnect($connect)
        $tcp.Close()
        Write-Host " ✅ REACHABLE" -ForegroundColor Green
        $results += @{ Item="Redis 可用性"; Status="PASS"; Detail="TCP ${RedisHost}:${RedisPort} OK" }
        $passCount++
    } else {
        $tcp.Close()
        throw "Connection timeout"
    }
} catch {
    Write-Host " ❌ UNREACHABLE" -ForegroundColor Red
    $results += @{ Item="Redis 可用性"; Status="FAIL"; Detail="$($_.Exception.Message)" }
    $failCount++
}

# ──────────────────────────────────────────
# 4. MinIO 可用性
# ──────────────────────────────────────────
Write-Host "[4/11] MinIO 可用性 (${MinioUrl}) ..." -NoNewline
try {
    $resp = Invoke-RestMethod -Uri "$MinioUrl/minio/health/live" -Method Get -TimeoutSec 5
    Write-Host " ✅ LIVE" -ForegroundColor Green
    $results += @{ Item="MinIO 可用性"; Status="PASS"; Detail="HTTP 200" }
    $passCount++
} catch {
    Write-Host " ❌ DOWN" -ForegroundColor Red
    $results += @{ Item="MinIO 可用性"; Status="FAIL"; Detail="$($_.Exception.Message)" }
    $failCount++
}

# ──────────────────────────────────────────
# 5. MySQL 连接池使用率（需 Actuator + JWT）
# ──────────────────────────────────────────
Write-Host "[5/11] MySQL 连接池使用率 ..." -NoNewline
try {
    $headers = @{}
    if ($JwtToken) { $headers["Authorization"] = "Bearer $JwtToken" }
    $metrics = Invoke-RestMethod -Uri "$BaseUrl/actuator/metrics/hikaricp.connections.active" `
        -Method Get -Headers $headers -TimeoutSec 5
    $active = $metrics.measurements[0].value
    $maxMetrics = Invoke-RestMethod -Uri "$BaseUrl/actuator/metrics/hikaricp.connections.max" `
        -Method Get -Headers $headers -TimeoutSec 5
    $max = $maxMetrics.measurements[0].value
    if ($max -gt 0) {
        $usage = [math]::Round($active / $max * 100, 1)
        if ($usage -gt 80) {
            Write-Host " ⚠️  ${usage}% (HIGH)" -ForegroundColor Yellow
            $results += @{ Item="MySQL 连接池使用率"; Status="WARN"; Detail="$active/$max = ${usage}%" }
            $warnCount++
        } else {
            Write-Host " ✅ ${usage}%" -ForegroundColor Green
            $results += @{ Item="MySQL 连接池使用率"; Status="PASS"; Detail="$active/$max = ${usage}%" }
            $passCount++
        }
    } else {
        Write-Host " ⚠️  N/A (max=0)" -ForegroundColor Yellow
        $results += @{ Item="MySQL 连接池使用率"; Status="WARN"; Detail="Cannot read max pool size" }
        $warnCount++
    }
} catch {
    Write-Host " ⚠️  SKIPPED (no JWT token or metrics unavailable)" -ForegroundColor Yellow
    $results += @{ Item="MySQL 连接池使用率"; Status="SKIP"; Detail="$($_.Exception.Message)" }
    $warnCount++
}

# ──────────────────────────────────────────
# 6-10. 日志分析类（建议配合外部日志系统）
# ──────────────────────────────────────────
Write-Host "[6/11]  接口 5xx 错误率        → 需日志聚合系统（ELK/Loki），脚本仅给出 grep 命令模板" -ForegroundColor Gray
Write-Host "[7/11]  登录失败次数          → 需日志聚合系统" -ForegroundColor Gray
Write-Host "[8/11]  审批失败数            → 需日志聚合系统" -ForegroundColor Gray
Write-Host "[9/11]  文件上传失败数        → 需日志聚合系统" -ForegroundColor Gray
Write-Host "[10/11] 预警批处理失败        → 需日志聚合系统" -ForegroundColor Gray
$results += @{ Item="接口 5xx 错误率"; Status="SKIP"; Detail="需日志聚合系统" }
$results += @{ Item="登录失败次数"; Status="SKIP"; Detail="需日志聚合系统" }
$results += @{ Item="审批失败数"; Status="SKIP"; Detail="需日志聚合系统" }
$results += @{ Item="文件上传失败数"; Status="SKIP"; Detail="需日志聚合系统" }
$results += @{ Item="预警批处理失败"; Status="SKIP"; Detail="需日志聚合系统" }

# ──────────────────────────────────────────
# 11. JVM 堆内存使用率（需 Actuator + JWT）
# ──────────────────────────────────────────
Write-Host "[11/11] JVM 堆内存使用率 ..." -NoNewline
try {
    $headers = @{}
    if ($JwtToken) { $headers["Authorization"] = "Bearer $JwtToken" }
    $usedMetrics = Invoke-RestMethod -Uri "$BaseUrl/actuator/metrics/jvm.memory.used" `
        -Method Get -Headers $headers -TimeoutSec 5
    $maxMetrics = Invoke-RestMethod -Uri "$BaseUrl/actuator/metrics/jvm.memory.max" `
        -Method Get -Headers $headers -TimeoutSec 5
    # 查找 "heap" 标签的指标
    $heapUsed = 0; $heapMax = 0
    foreach ($m in $usedMetrics.availableTags[0].values) {
        if ($m -eq "heap") {
            $heapUsed = ($usedMetrics.measurements | Where-Object { $_.statistic -eq "VALUE" }).value
            break
        }
    }
    # 简化：取第一个 measurement
    $heapUsed = $usedMetrics.measurements[0].value
    $heapMax = $maxMetrics.measurements[0].value
    if ($heapMax -gt 0) {
        $usage = [math]::Round($heapUsed / $heapMax * 100, 1)
        if ($usage -gt 85) {
            Write-Host " ⚠️  ${usage}% (HIGH)" -ForegroundColor Yellow
            $results += @{ Item="JVM 堆内存使用率"; Status="WARN"; Detail="$([math]::Round($heapUsed/1MB,0))MB / $([math]::Round($heapMax/1MB,0))MB = ${usage}%" }
            $warnCount++
        } else {
            Write-Host " ✅ ${usage}%" -ForegroundColor Green
            $results += @{ Item="JVM 堆内存使用率"; Status="PASS"; Detail="$([math]::Round($heapUsed/1MB,0))MB / $([math]::Round($heapMax/1MB,0))MB = ${usage}%" }
            $passCount++
        }
    }
} catch {
    Write-Host " ⚠️  SKIPPED (no JWT token or metrics unavailable)" -ForegroundColor Yellow
    $results += @{ Item="JVM 堆内存使用率"; Status="SKIP"; Detail="$($_.Exception.Message)" }
    $warnCount++
}

# ──────────────────────────────────────────
# 汇总报告
# ──────────────────────────────────────────
Write-Host "`n========================================" -ForegroundColor Cyan
Write-Host "  汇总报告" -ForegroundColor Cyan
Write-Host "========================================" -ForegroundColor Cyan

$results | ForEach-Object {
    $icon = switch ($_.Status) {
        "PASS" { "✅" }
        "WARN" { "⚠️ " }
        "FAIL" { "❌" }
        "SKIP" { "⏭️ " }
    }
    $color = switch ($_.Status) {
        "PASS" { "Green" }
        "WARN" { "Yellow" }
        "FAIL" { "Red" }
        "SKIP" { "Gray" }
    }
    Write-Host "  $icon $($_.Item) — $($_.Detail)" -ForegroundColor $color
}

Write-Host "`n────────────────────────────────────────" -ForegroundColor Cyan
Write-Host "  PASS: $passCount  |  WARN/SKIP: $warnCount  |  FAIL: $failCount" -ForegroundColor White
if ($failCount -gt 0) {
    Write-Host "  ⛔ 存在严重故障，请立即排查！" -ForegroundColor Red
} elseif ($warnCount -gt 0) {
    Write-Host "  ⚠️  部分项目需关注" -ForegroundColor Yellow
} else {
    Write-Host "  ✅ 所有检查通过" -ForegroundColor Green
}
Write-Host "────────────────────────────────────────`n" -ForegroundColor Cyan

# 退出码：FAIL > 0 时返回 1
if ($failCount -gt 0) { exit 1 } else { exit 0 }
```

### 4.2 日志关键字 grep 命令模板

以下命令可用于配合 cron / 任务计划程序做简易日志分析告警：

```bash
# Linux / Mac (需日志文件路径)
# 6. 接口 5xx 错误率（1 分钟内 > 10 次）
tail -10000 /var/log/cgc-pms/backend.log | grep "SYSTEM_ERROR" | wc -l

# 7. 登录失败次数（1 分钟内 > 20 次）
tail -5000 /var/log/cgc-pms/backend.log | grep -c "Login failed"

# 8. 审批失败数（1 小时内 > 5 次）
tail -50000 /var/log/cgc-pms/backend.log | grep -c "Business exception.*workflow"

# 9. 文件上传失败数（1 小时内 > 3 次）
tail -50000 /var/log/cgc-pms/backend.log | grep -c "File upload failed\|MinIO"

# 10. 预警批处理失败（连续 2 次）
tail -2000 /var/log/cgc-pms/backend.log | grep "AlertEvaluationService" | grep "ERROR" | wc -l
```

```powershell
# Windows (PowerShell)
# 6-10. 日志关键字统计（需日志文件路径）
$logPath = "D:\projects-test\cgc-pms\backend\logs\application.log"

# 5xx 错误
($lines | Select-String "SYSTEM_ERROR").Count

# 登录失败
($lines | Select-String "Login failed").Count

# 审批异常
($lines | Select-String "Business exception").Count

# 文件上传
($lines | Select-String "File upload|MinIO").Count

# 预警批处理
($lines | Select-String "AlertEvaluationService.*ERROR").Count
```

|------|------|------|
| 健康检查 | PowerShell 脚本 + Windows 任务计划程序 | 每 5 分钟执行一次，失败时写 Windows Event Log |
| 日志告警 | PowerShell 日志关键字扫描 + 邮件通知 | 每 1 分钟扫描，触发阈值时发邮件 |
| Dashboard | 无（二期 Grafana） | — |

### 5.2 二期建议：Prometheus + Grafana

| 组件 | 选型 | 说明 |
|------|------|------|
| **指标采集** | Prometheus + Micrometer | Spring Boot Actuator 已内置 Micrometer，只需添加 `micrometer-registry-prometheus` 依赖 |
| **可视化** | Grafana | Spring Boot 官方 Dashboard（ID: 19004），含 JVM/HTTP/HikariCP/Logback 面板 |
| **告警路由** | Alertmanager + 钉钉/企微 Webhook | 分级告警：P0 电话/短信，P1 企微群，P2 邮件 |
| **日志聚合** | Loki + Grafana（或 ELK） | 替代 grep 脚本，支持全文检索 + 告警规则 |
| **基础设施监控** | Node Exporter + cAdvisor | 主机 CPU/内存/磁盘 + 容器资源 |
