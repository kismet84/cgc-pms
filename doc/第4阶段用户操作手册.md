# 第 4 阶段用户操作手册

> 项目：建筑工程总包项目全过程管理系统 (cgc-pms)
> 版本：V1.0.0-SNAPSHOT
> 日期：2026-06-12
> 适用角色：系统管理员、项目经理、商务经理、成本经理、材料员、财务、分包管理员

---

## 1. 概述

第 4 阶段新增了组织架构管理、项目成员管理、字典管理、库存管理（仓库 + 库存台账 + 出入库 + 采购申请）、发票管理、消息中心和待办中心（已办 + 抄送）功能模块。本手册按菜单路径逐一说明各页面的操作步骤。

---

## 2. 组织架构

**菜单路径**：侧边栏 → 组织架构

组织架构页面采用"三 Tab"布局，集中管理公司、部门、岗位数据。

### 2.1 公司管理

左侧为公司列表，支持按公司编号、公司名称、状态（启用/停用）筛选。

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增公司 | 点击列表上方「+ 新建公司」按钮 → 填写公司编号、公司名称、备注 → 点击确定 | `org:add` |
| 编辑公司 | 在目标公司行点击「编辑」按钮 → 修改信息 → 点击确定 | `org:edit` |
| 删除公司 | 在目标公司行点击「删除」按钮 → 二次确认后删除 | `org:delete` |
| 启用/停用 | 编辑公司时将状态切换为「启用」或「停用」 | `org:edit` |

> **说明**：公司编码（company_code）全局唯一，重复编码无法保存。删除为逻辑删除，数据不会从数据库物理清除。

### 2.2 部门管理

右侧上方为部门树（自引用树形结构），展示某个公司下的部门层级。筛选公司后自动加载该公司的部门树。

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增部门 | 点击「+ 新增部门」按钮 → 选择所属公司、上级部门、填写部门编码和名称 → 确定 | `org:add` |
| 编辑部门 | 在树节点上点击「编辑」按钮 → 修改信息 → 确定 | `org:edit` |
| 删除部门 | 在树节点上点击「删除」按钮 → 二次确认 | `org:delete` |
| 展开/折叠 | 点击树节点左侧箭头展开或折叠子部门 | 无需额外权限 |

> **说明**：部门树支持无限层级嵌套。`parent_id` 为空的部门视为根部门。

### 2.3 岗位管理

右侧下方为岗位列表，支持按岗位编码、岗位名称、状态筛选。岗位代表 HR 头衔，不包含权限语义（权限由 RBAC 角色控制系统）。

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增岗位 | 点击「+ 新增岗位」按钮 → 填写编码、名称、状态 → 确定 | `org:add` |
| 编辑岗位 | 在目标岗位行点击「编辑」按钮 → 修改 → 确定 | `org:edit` |
| 删除岗位 | 在目标岗位行点击「删除」按钮 → 二次确认 | `org:delete` |

> **说明**：岗位编码全局唯一（租户内）。岗位仅作为用户信息字段使用，不影响审批流程中的角色判断。

## 3. 项目成员管理

**菜单路径**：侧边栏 → 项目管理 → 项目列表 → 点击某个项目 → 项目总览 → 项目成员（Tab）

项目成员管理页面用于为指定项目分配参与人员，并定义其在项目中的角色。

### 3.1 成员列表

进入项目成员页面后，显示该项目下所有已分配的成员列表，包含：用户名、真实姓名、项目角色、岗位名称、加入日期、退出日期、状态。

角色代码对照：

| 代码 | 角色名称 | 颜色标识 |
|------|----------|----------|
| PM | 项目经理 | 蓝色 |
| CM | 商务经理 | 绿色 |
| CSTM | 成本经理 | 橙色 |
| MAT | 材料员 | 紫色 |
| SUBC | 分包管理员 | 青色 |
| FIN | 财务 | 红色 |
| OTH | 其他 | 灰色 |

### 3.2 操作步骤

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增成员 | 点击「+ 新增成员」 → 选择用户 → 选择角色代码 → 填写岗位名称、加入日期（可选）→ 确定 | `project:member:add` |
| 编辑成员 | 在目标成员行点击「编辑」按钮 → 修改角色/岗位/日期 → 确定 | `project:member:edit` |
| 删除成员 | 在目标成员行点击「删除」按钮 → 二次确认 | `project:member:delete` |

> **说明**：同一用户在同一项目中只能被添加一次，重复添加会被唯一约束拦截。成员状态 `ACTIVE` 表示在职，`INACTIVE` 表示已退出。

## 4. 字典管理

**菜单路径**：侧边栏 → 系统设置 → 字典管理

字典管理页面采用"左主右从"布局：左侧为字典类型列表，右侧为选中类型的数据项列表。

### 4.1 字典类型管理

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增字典类型 | 左侧列表上方点击「新建」 → 填写字典编码（如 `CONTRACT_TYPE`）和字典名称 → 确定 | `system:dict:add` |
| 编辑字典类型 | 在目标类型行点击「编辑」 → 修改 → 确定 | `system:dict:edit` |
| 删除字典类型 | 在目标类型行点击「删除」 → 二次确认（级联删除该类型下所有数据项） | `system:dict:delete` |

### 4.2 字典数据项管理

点击左侧某个字典类型（如"合同类型"），右侧自动加载该类型下的所有数据项。

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增数据项 | 右侧列表上方点击「新建」 → 填写字典标签、字典值、排序号、状态 → 确定 | `system:dict:add` |
| 编辑数据项 | 在目标数据项行点击「编辑」 → 修改 → 确定 | `system:dict:edit` |
| 删除数据项 | 在目标数据项行点击「删除」 → 二次确认 | `system:dict:delete` |

> **说明**：字典数据以租户维度隔离。系统预置了一些字典类型（如合同类型、付款方式等），可通过此页面扩展。

## 5. 库存管理

库存管理包含 4 个子模块：仓库管理、库存台账、出入库管理、采购申请。

### 5.1 仓库管理

**菜单路径**：侧边栏 → 库存管理 → 仓库管理

仓库管理页面维护各个项目下的物理仓库信息。

#### 5.1.1 功能概览

| 字段 | 说明 |
|------|------|
| 仓库编号 | 自定义编号，如 `WH-001` |
| 仓库名称 | 仓库描述名称 |
| 所属项目 | 仓库关联的项目 |
| 状态 | 启用 / 停用 |
| 创建时间 | 仓库创建时间 |

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增仓库 | 点击「+ 新建仓库」 → 选择所属项目，填写编号和名称 → 确定 | `inventory:warehouse:add` |
| 编辑仓库 | 在目标仓库行点击「编辑」 → 修改 → 确定 | `inventory:warehouse:edit` |
| 删除仓库 | 在目标仓库行点击「删除」 → 二次确认 | `inventory:warehouse:delete` |
| 筛选 | 选择所属项目、输入编号/名称、选择状态进行组合筛选 | `inventory:warehouse:list` |

> **说明**：仓库停用后不影响已发生的出入库记录查询，但不能继续执行新的入库/出库操作。

### 5.2 库存台账

**菜单路径**：侧边栏 → 库存管理 → 库存台账

库存台账页面用于查询某个仓库中某类物料的当前库存余额和历史流水明细。

#### 5.2.1 页面布局

- **上半部分**：筛选区（选择仓库 + 物料），展示当前库存余额卡（可用数量）。
- **下半部分**：库存流水明细表（带分页），展示出入库的每一笔记录。

#### 5.2.2 操作步骤

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 查询库存 | 选择仓库（必选）→ 可选物料（筛选特定物料）→ 自动加载余额和流水 | `inventory:stock:list` |
| 翻页查看流水 | 点击表格下方的分页控件换页 | `inventory:stock:list` |

#### 5.2.3 流水类型说明

| 类型 | 含义 | 标签颜色 |
|------|------|----------|
| IN | 入库（库存增加） | 绿色 |
| OUT | 出库（库存减少） | 红色 |
| ADJUST | 库存调整（盘盈/盘亏校正） | 橙色 |

> **说明**：库存余额表（`mat_stock`）使用乐观锁（`@Version`），并发扣减时自动重试或提示冲突。流水表（`mat_stock_txn`）记录每次变动的来源业务类型和来源业务 ID，支持追溯到具体业务单据。

### 5.3 出入库管理

**菜单路径**：侧边栏 → 库存管理 → 出入库管理

出入库管理页面提供入库和出库两个操作 Tab。

#### 5.3.1 入库操作

1. 切换到「入库」Tab。
2. 选择目标仓库（仅显示启用状态的仓库）。
3. 选择物料（从物料字典中选择已启用的物料）。
4. 输入入库数量（必须为正数）。
5. 点击「确认入库」按钮。

入库完成后自动生成一条 `txn_type = 'IN'` 的库存流水记录，并更新对应仓库-物料的库存余额。

#### 5.3.2 出库操作

1. 切换到「出库」Tab。
2. 选择目标仓库、物料。
3. 输入出库数量（必须为正数，系统自动校验库存是否充足）。
4. 点击「确认出库」按钮。

出库完成后生成 `txn_type = 'OUT'` 的流水记录，扣减库存余额。

| 操作 | 所需权限 |
|------|----------|
| 入库 | `inventory:transaction:add` |
| 出库 | `inventory:transaction:add` |

> **说明**：出入库操作不做审批流程，直接生效。如需审批控制的物料进出，请使用采购订单流程（Phase 2 已实现）。

### 5.4 采购申请

**菜单路径**：侧边栏 → 库存管理 → 采购申请

采购申请页面用于物资需求的提交和审批管理。申请审批通过后可转为采购订单（`CONVERTED` 状态）。

#### 5.4.1 申请列表

列表字段：申请编号（`PR-yyyyMMdd-XXX` 自动生成）、所属项目、审批状态、业务状态、创建时间。

| 审批状态 | 说明 |
|----------|------|
| DRAFT | 草稿 |
| APPROVING | 审批中 |
| APPROVED | 已通过 |
| REJECTED | 已驳回 |
| WITHDRAWN | 已撤回 |

| 业务状态 | 说明 |
|----------|------|
| DRAFT | 草稿 |
| APPROVED | 已通过，可转采购订单 |
| CONVERTED | 已转采购订单 |

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新建采购申请 | 点击「+ 新建采购申请」→ 选择所属项目 → 填写备注 → 确定 | `purchase:request:add` |
| 编辑申请 | 草稿状态下点击「编辑」 → 修改 → 确定 | `purchase:request:edit` |
| 删除申请 | 草稿状态下点击「删除」 → 二次确认 | `purchase:request:delete` |
| 填写申请明细 | 点击申请编号进入详情 → 在明细区域添加物料、数量、期望到货日期 → 批量保存 | `purchase:request:edit` |
| 提交审批 | 点击「提交审批」按钮 → 确认提交 | `purchase:request:submit` |

#### 5.4.2 审批流程

提交后自动启动采购申请审批流程（模板编号 `TPL-PURCHASE-REQUEST-001`），审批节点如下：

| 节点 | 审批角色 | 超时时间 |
|------|----------|----------|
| 1. 项目经理 | PROJECT_MANAGER | 48 小时 |
| 2. 商务经理 | COMMERCIAL_MANAGER | 48 小时 |
| 3. 成本/采购 | COST_MANAGER | 48 小时 |

三级顺序审批全部通过后，申请状态变更为 `APPROVED`，可在采购订单模块将其转为正式采购单。

> **说明**：审批流程支持转办（`allow_transfer = 1`）和加签（`allow_add_sign = 1`）。草稿状态（`DRAFT`）下可编辑和删除，审批中（`APPROVING`）不可编辑或删除。

## 6. 发票管理

**菜单路径**：侧边栏 → 发票管理

发票管理页面维护发票信息，关联付款申请和付款记录，支持核验操作。

### 6.1 发票列表

列表字段：发票号码、发票类型、发票金额、税率、税额、开票日期、核验状态、备注、创建时间。

| 发票类型 | 说明 |
|----------|------|
| VAT_SPECIAL | 增值税专用发票 |
| VAT_NORMAL | 增值税普通发票 |
| OTHER | 其他发票 |

| 核验状态 | 说明 | 标签颜色 |
|----------|------|----------|
| PENDING | 待核验 | 默认/灰色 |
| VERIFIED | 已认证 | 绿色 |
| ABNORMAL | 异常 | 红色 |

### 6.2 操作步骤

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 新增发票 | 点击「+ 新增发票」 → 填写发票号码、类型、金额、税率、税额、开票日期、关联付款记录（可选）→ 确定 | `invoice:add` |
| 编辑发票 | 点击目标行「编辑」 → 修改 → 确定 | `invoice:edit` |
| 删除发票 | 点击目标行「删除」 → 二次确认 | `invoice:delete` |
| 核验发票 | 点击目标行「核验」 → 选择核验状态（`VERIFIED` / `ABNORMAL`）→ 确认 | `invoice:verify` |
| 发票登记 | 点击「发票登记」按钮快速录入发票信息 | `invoice:add` |
| 按付款记录筛选 | 在筛选区选择付款记录，查看关联的所有发票 | `invoice:query`（隐含） |

> **说明**：发票号码在租户内全局唯一（`uk_pi_tenant_invoice_no`），重复号码无法保存。核验操作不走审批链，直接切换 `verify_status` 字段。发票通过 `pay_application_id` 和 `pay_record_id` 关联到付款流程。

## 7. 消息中心

**菜单路径**：侧边栏 → 消息中心

消息中心用于查看系统推送的站内消息通知。导航栏右上角的通知铃铛图标（`NotificationBell` 组件）同步显示未读数量。

### 7.1 消息列表

列表字段：通知标题、内容、业务类型、通知类型、已读状态、创建时间。

| 通知类型 | 说明 |
|----------|------|
| INFO | 信息通知 |
| WARNING | 警告通知 |
| ERROR | 错误通知 |

| 业务类型 | 说明 |
|----------|------|
| WORKFLOW_APPROVAL | 审批通过通知 |
| WORKFLOW_REJECT | 审批驳回通知 |
| WORKFLOW_CC | 抄送通知 |
| ALERT | 预警通知 |
| SYSTEM | 系统通知 |

### 7.2 操作步骤

| 操作 | 步骤 | 所需权限 |
|------|------|----------|
| 查看消息列表 | 进入消息中心即展示当前用户的所有消息（分页） | `notification:view` |
| 筛选未读 | 开启「仅看未读」开关，只显示未读消息 | `notification:view` |
| 标记已读 | 点击某条消息的「标记已读」按钮 | `notification:edit` |
| 全部已读 | 点击「全部已读」按钮，当前用户所有未读消息全部标记为已读 | `notification:edit` |

### 7.3 SSE 实时推送

消息中心支持 SSE（Server-Sent Events）实时推送。前端通过 `/api/notifications/stream` 端点订阅消息流：

1. 页面加载时自动建立 SSE 连接。
2. 当有新通知产生时，后台通过 SseEmitter 推送到当前用户。
3. 页面右上角铃铛角标实时更新未读数。

> **说明**：SSE 基于 HttpOnly Cookie 认证，与系统登录状态一致。若连接中断，前端自动重建订阅。

## 8. 待办中心

**菜单路径**：侧边栏 → 审批管理 → 我的待办

待办中心合并了三级 Tab：我的待办、我的已办、我的抄送。

### 8.1 我的待办

展示当前用户作为审批人的所有待处理任务列表。

| 字段 | 说明 |
|------|------|
| 业务类型 | 合同审批 / 付款申请 / 签证变更 / 材料验收 / 采购申请 等 |
| 标题 | 业务单据的审批标题 |
| 发起人 | 提交审批的用户 |
| 到达时间 | 任务到达当前节点的时间 |

操作：点击某条待办，跳转到对应的审批详情页（`/approval/:instanceId`），可执行同意 / 驳回 / 转办 / 加签等操作。

### 8.2 我的已办

展示当前用户已完成审批的历史任务列表。包含审批结果（同意 / 驳回）、审批意见、审批时间等信息。

### 8.3 我的抄送

展示当前用户被抄送的审批实例列表。抄送人无需审批操作，仅供知悉。

| 字段 | 说明 |
|------|------|
| 标题 | 审批实例标题 |
| 业务类型 | 业务单据类型 |
| 状态 | 审批实例当前状态 |
| 抄送时间 | 该抄送记录的创建时间 |
| 已读标记 | 未读 / 已读 |

一次性可标记单条或批量已读。

> **说明**：抄送数据存储于独立的 `wf_cc` 表，不侵入审批引擎核心表（`wf_instance` / `wf_task` / `wf_record`）。通知中心同步会收到一条 `WORKFLOW_CC` 类型的消息。
