## 2. 正式版定版结论

| 事项 | 定版结论 |
|---|---|
| 系统主线 | 项目为根、合同为纲、执行为据、成本为果、付款为流、结算为终 |
| 架构形态 | 一期采用模块化单体，后期可演进微服务 |
| 数据库 | 一期默认 MySQL 8.0 |
| JSON 配置 | 审批规则、动态表单、条件规则使用 MySQL JSON |
| 审批调用 | 同一后端应用内部采用事务内领域服务直接调用，不通过 HTTP 调自己 |
| PC 前端 | Vue 3 + TypeScript + Vite + 企业级组件库 |
| 移动主端 | uni-app 原生 App / H5 / 小程序路线优先 |
| 内部轻入口 | 企业微信侧承接内部待办、消息、轻审批 |
| 外部轻入口 | 微信小程序承接供应商、分包商轻协作 |
| 桌面壳 | 一期暂缓；确有托盘、文件系统、打印、外设需求时优先 Tauri |
| 移动节奏 | 第 6 周启动基础工程，第 8 周左右进入真实业务功能 |

## 4. 文档维护规则

1. 本套文档作为 V1.0 正式开发基线。
2. 数据库、审批、API、移动端节奏、技术栈发生变更时，必须同步更新相关文档。
3. API 字段变更必须同步更新 OpenAPI、前端类型定义和测试用例。
4. 数据库字段变更必须同步更新迁移脚本和数据库文档。
5. 审批流程变更必须同步更新审批模板、API 契约、测试用例。
