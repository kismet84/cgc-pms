审查对象：`D:\projects-test\cgc-pms`
审查方式：全仓库只读审查，未修改代码
技术栈：Spring Boot / Java 21、Vue 3 / TypeScript / Vite、MyBatis-Plus、Flyway、MySQL、Redis、MinIO

- `backend`: `.\mvnw.cmd test-compile -q` 通过

## 总体结论

发现多处高置信度问题，主要集中在：

- 后端数据权限缺失：核心业务接口缺少 `tenantId`、项目范围、业务归属校验。
- 工作流通用入口权限过宽：可能绕开合同模块自身权限和状态机。
- 文件访问存在 IDOR 风险：按文件 ID 获取预签名 URL/删除文件，未校验业务归属。
- 前后端接口契约不一致：合同台账前端调用的接口后端不存在。
- 前端 token 存储和 401 刷新队列存在安全与稳定性风险。

## 按审查维度汇总

### 1. 安全性检查

发现高风险问题：

- 多租户/项目级数据权限缺失。
- 文件访问存在按 ID 越权风险。
- 工作流通用提交入口权限过宽。
- Refresh token 可在账号禁用后继续换发新 token。
- 前端 token 存储在 JS 可读存储中，XSS 后果较重。

未发现高置信度 SQL 注入问题。当前查询主要使用 MyBatis-Plus Lambda Wrapper，未看到明显字符串拼接 SQL 注入路径。

### 2. 逻辑与边界

发现问题：

- 合同子资源未校验 `itemId/termId` 与路径 `contractId` 的归属关系。
- `Promise.all` 未 await。
- `localStorage` JSON 解析缺少容错。
- Refresh 逻辑未校验用户状态。

### 3. 性能影响

发现问题：

- 用户分页角色查询存在 N+1。
- 审批详情节点任务查询存在 N+1。
- 文件列表生成预签名 URL 会对每个文件逐个调用 MinIO 签名逻辑，文件量大时可能变慢，建议分页或批量限制。

### 4. 规范与可读性

整体风格较一致，Spring Controller/Service 分层清晰，Vue 组件组织也基本统一。主要规范风险是：

- 权限控制分散在 Controller 注解，数据范围未形成统一抽象。
- 前后端接口路径契约不一致，缺少契约测试或 OpenAPI 生成约束。
- 部分前端页面仍含 mock KPI/状态统计数据，容易和真实接口混淆。

### 5. 回归风险

高回归风险点：

- 修复数据权限时会影响合同、项目、合作方、文件、审批等多个模块，需要统一测试矩阵。
- 修复 token 存储方式若改为 Cookie，会影响 CORS、CSRF、登录刷新、移动端兼容。
- 补齐 `/contracts/ledger`、`/contracts/kpi` 或调整前端路径会影响合同台账首页。

### 6. 渲染性能

未发现明显重复渲染导致的高置信度严重问题。注意点：

- 合同台账使用 `computed` 动态生成表格列，列设置变动会触发表格重渲染，当前数据规模下可接受。
- 右侧统计图使用静态 mock 数据，后续接真实数据时应避免每次输入筛选都重建大对象或重复请求。

### 7. 网络请求

发现问题：

- 合同台账前端请求后端不存在的接口。
- 401 并发刷新失败时请求队列可能挂起。
- 合同详情加载多个接口，当前 `Promise.all` 未 await，错误处理不完整。

### 8. 算法复杂度

未发现明显 O(n²) 算法热点。主要是 N+1 数据访问问题，本质上会随记录数线性放大数据库请求数量。

### 9. 资源加载

未发现明显图片、脚本未按需加载问题。前端路由组件使用动态 import，符合按路由懒加载习惯。

## 建议优先级

1. 先修复数据权限与 IDOR：合同、项目、合作方、文件、审批实例。
2. 禁用或加固 `/workflow/submit` 通用入口。
3. 修复前后端合同接口契约不一致。
4. 修复 Refresh token 禁用账号续期问题。
5. 修复前端 401 队列挂起和 `Promise.all` 未 await。
6. 优化 N+1 查询。
7. 完善 token 存储策略和 CSP/CSRF 方案。

## 建议人工验证风险点

- 生产环境是否在网关/CDN 设置了 CSP、`X-Frame-Options` / `frame-ancestors`、`X-Content-Type-Options` 等安全头。
- MinIO bucket 权限是否私有，预签名 URL 是否会被日志、前端错误上报或 Referer 泄露。
- `application-prod.yml` 中 Swagger UI 虽关闭，但安全白名单仍包含 `/v3/api-docs/**`，需确认生产是否真的不可访问。
- `.sisyphus` 下工作记录是否应提交，避免泄露内部决策和开发过程信息。

## 结语

本次审查发现的关键问题不是语法或单点逻辑错误，而是权限边界与契约一致性问题。建议先建立统一的数据权限校验机制和接口契约测试，再逐步修复业务模块中的具体入口。
