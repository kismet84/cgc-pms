# 建筑工程总包项目审批流程设计方案

建筑工程总包项目全过程管理系统

审批流程设计方案

适用对象：建筑工程总包单位 / 项目部 / 公司管理层

| 设计主线：项目为根，合同为纲，执行为据，成本为果，付款为流，结算为终。 |
| --- |

# 一、审批设计总原则

审批体系围绕项目、合同、材料、分包、成本、付款和结算之间的业务关系设计。所有合同、采购、分包、成本、付款、结算均应关联 project_id，涉及合同的业务应关联 contract_id。

合同先审批，执行后发生：分包、采购、租赁等涉及合同的业务，原则上必须先完成合同审批。

业务审批与付款审批分离：材料验收、分包计量、设备租赁确认属于业务事实确认；付款申请属于资金支付审批。

成本审核嵌入关键节点：合同、采购、分包计量、签证变更、付款申请和结算审批都应经过成本校验。

金额分级审批：小额事项走项目级审批，中额事项走公司职能部门审批，大额或超预算事项走公司领导审批。

异常事项强制升级：超合同金额、超目标成本、无合同付款、未验收付款、未计量付款、质保金提前释放等必须自动升级或禁止提交。

# 二、统一审批引擎设计

## 2.1 通用审批状态

| 草稿 → 已提交 → 审批中 → 已通过 / 已驳回 / 已撤回 → 已生效 / 已作废 |
| --- |

| 审批对象 | 审批通过后的结果 |
| --- | --- |
| 合同审批 | 合同生效，生成合同台账，允许后续采购、分包、付款引用 |
| 采购申请 | 允许生成采购订单 |
| 采购订单 | 锁定采购金额，允许后续验收和入库 |
| 材料验收 | 形成材料成本依据 |
| 分包计量 | 形成分包成本依据 |
| 签证变更 | 调整合同金额或动态成本 |
| 付款申请 | 形成应付审批结果，推送财务付款 |
| 实际付款 | 回写已支付金额 |
| 结算审批 | 锁定最终结算金额，进入归档 |

## 2.2 通用审批动作

| 审批动作 | 说明 |
| --- | --- |
| 同意 | 当前节点通过，进入下一节点 |
| 驳回 | 退回发起人，修改后可重新提交 |
| 退回上一步 | 退回上一个审批节点，不直接退回发起人 |
| 撤回 | 发起人在下一节点未处理前可撤回 |
| 加签 | 当前审批人临时增加其他审批人 |
| 转办 | 当前审批人将任务转给同权限人员 |
| 抄送 | 审批完成或关键节点通知相关人员 |
| 作废 | 已通过但未发生后续业务时可作废 |

## 2.3 审批分级规则

金额阈值建议做成系统参数，便于不同企业或项目灵活配置。

| 审批等级 | 建议金额范围 | 审批范围 |
| --- | --- | --- |
| L1 项目级 | ≤ 5 万 | 项目内部审批 |
| L2 部门级 | 5 万 - 50 万 | 项目 + 公司职能部门 |
| L3 公司级 | 50 万 - 200 万 | 项目 + 部门 + 分管领导 |
| L4 重大事项 | ＞ 200 万或超预算 | 公司领导 / 总经理 / 董事长 |

| 升级条件 | 处理方式 |
| --- | --- |
| 超目标成本 | 增加成本负责人、公司领导审批 |
| 超合同金额 | 必须先走合同变更审批 |
| 无合同付款 | 原则上禁止，零星采购除外 |
| 未验收付款 | 禁止提交或强制总经理审批 |
| 未计量付款 | 禁止提交或强制总经理审批 |
| 超付款比例 | 增加商务、财务、公司领导审批 |
| 质保金提前释放 | 强制公司领导审批 |
| 合同相对方为黑名单单位 | 禁止提交 |

# 三、总体审批架构

| 业务单据发起   ↓ 校验 project_id   ↓ 判断是否涉及合同 → 涉及合同则校验 contract_id   ↓ 业务规则校验   ↓ 判断是否超预算 / 超合同 / 超付款比例   ↓ 标准流程审批或自动升级   ↓ 项目审批 → 职能部门审批 → 成本审核 → 财务审核   ↓ 按金额触发公司领导审批   ↓ 审批通过   ↓ 业务状态生效   ↓ 生成后续业务数据 / 成本 / 付款 / 结算依据 |
| --- |

# 四、核心审批流程设计

## 4.1 合同审批流程

适用于总包合同、分包合同、采购合同、租赁合同、服务合同和补充协议。

| 合同管理员起草 → 项目经理审核 → 商务经理审核 → 成本经理审核 → 法务/合同管理员审核 → 财务审核付款条款 → 分管领导审批 → 总经理审批（按金额触发） → 合同生效 → 合同台账锁定 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 项目经理 | 合同是否符合项目实际需求 |
| 商务经理 | 清单、计价方式、结算方式是否合理 |
| 成本经理 | 是否超目标成本、是否影响动态成本 |
| 法务/合同管理员 | 合同条款、风险条款、违约责任 |
| 财务人员 | 付款比例、发票、税率、质保金 |
| 公司领导 | 大额合同、重大风险、超预算事项 |

审批通过后动作：

生成合同台账

锁定合同金额

生成合同付款条件

允许采购订单、分包计量、付款申请引用

合同金额进入合同锁定成本

## 4.2 采购申请审批流程

适用于材料、设备、周转材料、零星物资采购需求。

| 材料员发起采购申请 → 施工员/专业工程师确认需求 → 项目经理审核 → 材料负责人审核 → 成本经理审核预算 → 商务经理审核合同或价格依据 → 分管领导审批（按金额触发） → 采购申请通过 → 生成采购订单 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 关联项目 | 必须关联 project_id |
| 材料预算 | 超预算自动升级 |
| 采购合同 | 有合同则引用合同，无合同则按零星采购或发起合同审批 |
| 重复采购 | 根据库存、未到货订单、需求计划校验 |
| 急采 | 必须填写原因，并抄送项目经理和成本经理 |

## 4.3 采购订单审批流程

采购申请通过后生成采购订单。订单审批重点是价格、数量、供应商、交付时间和合同关联。

| 采购员生成采购订单 → 材料负责人审核 → 成本经理审核价格和预算 → 项目经理审核 → 商务经理审核合同关联 → 分管领导审批（按金额触发） → 订单生效 → 允许材料验收和入库 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 合同采购 | 订单必须关联采购合同 |
| 框架协议采购 | 订单关联框架合同，记录本次订单金额 |
| 零星采购 | 可不关联合同，但必须标记零星采购 |
| 超合同金额采购 | 禁止提交，必须先走合同变更 |

## 4.4 材料验收审批流程

材料验收确认材料真实到场、质量合格、数量准确，是材料成本形成依据，但不是付款审批。

| 材料员发起到货验收 → 施工员/专业工程师确认使用部位 → 质检员确认质量 → 仓管员确认数量并入库 → 项目经理审核 → 成本系统生成材料成本记录 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 无采购订单 | 原则上不允许验收 |
| 到货数量大于订单数量 | 触发超量预警 |
| 检测不合格 | 禁止入库，进入退场/复检流程 |
| 资料不全 | 可暂存，不允许形成正式成本 |
| 已验收 | 可作为后续付款依据 |

## 4.5 分包计量审批流程

分包计量是分包成本形成的核心依据。分包管理通过 contract_id 引用分包合同，计量确认后形成分包成本。

| 分包单位/分包管理员提交计量 → 专业工程师确认完成量 → 质量员确认质量合格 → 安全员确认无重大安全问题 → 商务经理审核计量规则 → 成本经理审核成本影响 → 项目经理审批 → 分管领导审批（按金额触发） → 计量确认 → 生成分包成本 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 未签分包合同 | 禁止提交计量 |
| 超合同工程量 | 触发合同变更或签证流程 |
| 质量未验收 | 不允许计量通过 |
| 安全文明处罚未处理 | 付款时扣减或挂起 |
| 计量金额超合同金额 | 禁止通过，必须先变更 |

## 4.6 现场签证 / 合同变更审批流程

适用于现场签证、设计变更、工程量变更、价格调整和补充协议。

| 现场人员发起签证/变更 → 项目工程负责人审核事实 → 商务经理审核计价依据 → 成本经理审核成本影响 → 项目经理审批 → 业主/监理确认（外部流程） → 合同管理员更新合同变更 → 分管领导审批（按金额触发） → 变更生效 → 调整合同金额和动态成本 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 无现场依据 | 禁止提交 |
| 无影像资料 | 退回补充 |
| 涉及合同金额变化 | 必须形成合同变更记录 |
| 影响目标成本 | 自动推送成本管理 |
| 业主未确认 | 可作为预计成本，不作为最终结算依据 |
| 超金额阈值 | 自动升级公司领导审批 |

## 4.7 付款申请审批流程

付款审批是资金控制流程，应关联 project_id、contract_id、付款对象、付款依据、已计量金额、已验收金额、已申请金额、已审批金额和已支付金额。

| 业务模块发起付款申请 → 系统自动校验合同、验收、计量、付款比例 → 项目经理审核付款必要性 → 商务经理审核合同和付款依据 → 成本经理审核成本与动态成本 → 财务审核发票、税率、已付金额 → 分管领导审批 → 总经理审批（按金额或异常触发） → 付款申请通过 → 推送财务系统付款 → 财务回写实际付款 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 未签合同 | 禁止付款，零星采购除外 |
| 材料未验收 | 禁止付款 |
| 分包未计量 | 禁止付款 |
| 超合同金额 | 必须先走合同变更 |
| 超付款比例 | 自动升级审批 |
| 发票缺失 | 可审批但不允许实际支付，或按企业规则控制 |
| 质保金未到期 | 禁止释放或总经理特批 |

## 4.8 结算审批流程

结算审批是最终收口流程，应基于合同、计量、验收、签证、付款、成本明细自动汇总。

| 分包/采购/总包结算发起 → 系统汇总合同、计量、验收、签证、付款、成本数据 → 专业负责人确认完成内容 → 商务经理审核结算金额 → 成本经理审核成本影响 → 项目经理审批 → 财务审核应付/已付/未付 → 公司领导审批（按金额触发） → 结算确认 → 归档 |
| --- |

| 控制点 / 审批角色 | 规则 / 审批重点 |
| --- | --- |
| 数据来源 | 应从前端业务自动汇总，不重复录入 |
| 结算金额 | 与合同、变更、签证、计量、付款联动校验 |
| 资料缺失 | 不允许归档或触发预警 |
| 结算确认 | 锁定最终结算金额，后续修改需走调整审批 |

# 五、付款类型与付款依据

| 付款类型 | 付款依据 | 发起模块 |
| --- | --- | --- |
| 材料付款 | 采购合同、采购订单、验收单、入库单、发票 | 材料设备管理 / 付款管理 |
| 分包付款 | 分包合同、分包计量单、质量安全确认、发票 | 分包管理 |
| 设备租赁付款 | 租赁合同、设备使用记录、租赁结算单 | 设备管理 |
| 劳务付款 | 劳务合同、考勤、实名制、工资表 | 劳务管理 |
| 质保金释放 | 合同质保条款、质保期到期证明 | 合同管理 / 财务 |

| 付款申请金额 ≤ 合同金额 - 已审批付款金额 付款申请金额 ≤ 合同约定付款比例对应金额 付款申请金额 ≤ 已验收金额或已计量金额 |
| --- |

# 七、审批与数据状态联动

| 业务对象 | 状态流转 |
| --- | --- |
| 合同 | 草稿 → 审批中 → 已审批 → 已签订 → 履约中 → 已结算 → 已归档 |
| 采购订单 | 草稿 → 审批中 → 已生效 → 部分到货 → 已到货 → 已结算 → 已关闭 |
| 分包计量 | 待提交 → 审批中 → 已确认 → 已生成成本 → 已参与付款 → 已结算 |
| 付款申请 | 草稿 → 审批中 → 已审批 → 待付款 → 已付款 → 已冲销/作废 |
| 结算 | 草稿 → 审批中 → 已确认 → 已归档 |

# 八、关键防重叠设计

| 容易混淆事项 | 正确边界 |
| --- | --- |
| 合同审批不替代采购审批 | 合同确认主体、金额、条款；采购审批确认需求、数量、价格、供应商和交付。 |
| 分包合同审批不替代分包计量审批 | 分包合同确认合同边界和金额；分包计量确认实际完成量。 |
| 材料验收审批不等于付款审批 | 材料验收形成成本依据；付款申请形成资金支付依据。 |
| 成本审核不是重复录入成本 | 成本经理只审核和校验，成本数据来自合同、验收、计量、签证、设备、劳务和财务付款。 |

# 十、最终建议

| 合同审批 → 采购/分包/租赁等执行业务审批 → 验收/计量审批 → 成本归集 → 付款申请审批 → 财务付款确认 → 结算审批 → 项目归档 |
| --- |

系统中最重要的控制点如下：

| 无项目，不提交； 无合同，不付款； 未验收，不付款； 未计量，不付款； 超合同，先变更； 超预算，强升级； 付款后，财务回写； 结算后，数据锁定。 |
| --- |

这样设计后，合同、材料采购、分包、成本、付款、结算之间既有关联，又不会互相重复。合同管依据，采购和分包管执行，成本管归集，付款管资金，结算管收口。
