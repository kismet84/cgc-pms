## 实现机制

数据隔离通过以下方式实现：

1. **Service 层 LambdaQueryWrapper** — 所有列表查询统一追加 `.eq(Entity::getTenantId, UserContext.getCurrentTenantId())`
2. **单条操作校验** — `selectById` 后校验 `tenantId` 一致性，不一致则返回 404（不泄露存在性）
3. **文件权限** — FileController 按 tenantId 鉴权，预签名 URL 和删除操作均受保护

## 已知限制

| 限制 | 说明 |
|------|------|
| 大部分业务 API 仅 Admin 可访问 | 当前非 admin 用户（除审批工作流外）不能访问任何业务 API，因此跨租户数据泄露风险极低 |

多租户数据隔离机制生效正确：
- ✅ 列表查询不返回非授权租户数据

- ✅ 文件预签名 URL、删除接口受权限控制
- ✅ 通知不跨租户泄露
