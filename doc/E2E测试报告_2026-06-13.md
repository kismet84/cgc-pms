## 5. 执行方式

### 基本命令

```bash
cd frontend-admin

# 列出所有测试（不执行）
npx playwright test --list

# 执行全量 E2E 测试
npx playwright test

# 执行指定 spec
npx playwright test e2e/login.spec.ts

# 查看 HTML 报告
npx playwright show-report
```

### 前置条件

E2E 测试依赖后端 API 可用。Playwright 配置了 `webServer` 自动启动前端开发服务器（`pnpm dev`），在非 CI 环境下复用已有实例（`reuseExistingServer: true`）。

执行全量 E2E 前必须确保：

1. 后端服务运行中（默认 `http://localhost:8080`）
2. 数据库已通过 Flyway 迁移（V1-V40）
3. 管理员用户 `admin` / `admin123` 可用

### CI 模式

在 CI 环境中（`process.env.CI` 为真）：

- 单 worker 串行执行（`workers: 1`）
- 失败自动重试 2 次（`retries: 2`）
- `forbidOnly: true` 阻止 `test.only` 提交
- webServer 不复用已有实例（完整重启）

### 认证辅助函数

所有 9 个 spec 文件共享相同的 `loginAsAdmin` 辅助函数：

```typescript
async function loginAsAdmin(page: Page) {
  await page.goto('/login');
  await page.fill('input[placeholder="请输入用户名"]', 'admin');
  await page.fill('input[placeholder="请输入密码"]', 'admin123');
  await page.click('button[type="submit"]');
  await page.waitForURL((url) => !url.pathname.includes('/login'), { timeout: 15000 });
}
```

该函数在 `test.beforeEach` 中调用，确保每个测试用例拥有独立的已认证会话。

### 测试设计原则

1. **优雅降级**：测试在缺少前置数据时通过 `console.log` + `return` 退出，不强制报错
2. **无硬编码延迟**：使用 `waitForSelector`、`waitForURL` 等待页面状态，唯一例外为 Ant Design tabs 切换后的 300ms paint cycle
3. **选择器稳健**：优先使用文本内容（`has-text`）和语义化 class，避免依赖动态生成的 data 属性
4. **截图可追溯**：关键操作后截图保存至 `e2e/screenshots/`，命名含模块和操作描述

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './e2e',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: [['html'], ['list']],
  timeout: 30000,
  use: {
    baseURL: 'http://localhost:5173',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
  ],
  webServer: {
    command: 'pnpm dev',
    url: 'http://localhost:5173',
    reuseExistingServer: !process.env.CI,
    timeout: 120000,
  },
});
```
